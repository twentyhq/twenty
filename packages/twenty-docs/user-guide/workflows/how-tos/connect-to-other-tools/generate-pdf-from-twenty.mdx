title: Generate a PDF from Twenty
description: Create a workflow to generate and attach a PDF (such as a quote) to a record.
---

Automatically generate or fetch a PDF and attach it to a record in Twenty. This is commonly used to create quotes, invoices, or reports that are linked to Companies, Opportunities, or other objects.

## Overview

This workflow uses a **Manual Trigger** so users can generate a PDF on demand for any selected record. A **Logic Function** calls `createAttachment` from `twenty-client-sdk/files`, which downloads the PDF, uploads it to Twenty, and links it to the record in a single call.

## Prerequisites

Before setting up the workflow:

1. **Set up a PDF generation service** (optional): If you want to dynamically generate PDFs (e.g., quotes), use a service like Carbone, PDFMonkey, or DocuSeal to create the PDF and get a download URL.

<Note>
Logic functions run with an app access token that is injected automatically, so you don't need to create or paste an API key.
</Note>

## Step-by-Step Setup

### Step 1: Configure the Trigger

1. Go to **Workflows** and create a new workflow
2. Select **Manual Trigger**
3. Choose the object you want to attach PDFs to (e.g., **Company** or **Opportunity**)

<Tip>
With a Manual Trigger, users can run this workflow using a button that appears on the top right once a record is selected, to generate and attach a PDF.
</Tip>

### Step 2: Add a Logic Function

1. Add a **Code** action (logic function)
2. Create a new function with the code below
3. Configure the input parameters

#### Input Parameters

| Parameter | Value |
|-----------|-------|
| `companyId` | `{{trigger.object.id}}` |

<Note>
If attaching to a different object (Person, Opportunity, etc.), rename the parameter accordingly (e.g., `personId`, `opportunityId`) and update the `target` in the logic function.
</Note>

#### Logic Function Code

```typescript
import { createAttachment } from 'twenty-client-sdk/files';

export const main = async (params: { companyId: string }) => {
  // Replace with your PDF URL
  // This could be from a PDF generation service or a static URL
  const pdfUrl = 'https://your-pdf-service.com/generated-quote.pdf';

  return await createAttachment({
    source: { url: pdfUrl },
    filename: 'quote.pdf',
    target: {
      objectNameSingular: 'company',
      recordId: params.companyId,
    },
  });
};
```

`createAttachment` downloads the file from the URL, uploads it to Twenty, and creates an Attachment linked to the record. If you already have the file contents (e.g., from a previous step or generated in code), pass them directly instead of a URL:

```typescript
source: { data: pdfBytes, contentType: 'application/pdf' }
```

### Step 3: Customize for Your Use Case

#### To attach to a different object

Set `objectNameSingular` to the object you want to attach to:

| Object | `objectNameSingular` |
|--------|----------------------|
| Company | `company` |
| Person | `person` |
| Opportunity | `opportunity` |
| Custom Object | `yourCustomObject` |

Update the function parameter (e.g., `personId`) and the `recordId` accordingly.

#### To use a dynamic PDF URL

If using a PDF generation service, you can:
1. First make an HTTP Request action to generate the PDF
2. Pass the returned PDF URL to the logic function as a parameter

```typescript
import { createAttachment } from 'twenty-client-sdk/files';

export const main = async (
  params: { companyId: string; pdfUrl: string; filename: string },
) => {
  return await createAttachment({
    source: { url: params.pdfUrl },
    filename: params.filename,
    target: {
      objectNameSingular: 'company',
      recordId: params.companyId,
    },
  });
};
```

### Step 4: Test and Activate

1. Save the workflow
2. Navigate to a Company record
3. Click the **â‹®** menu and select your workflow
4. Check the **Attachments** section on the record to verify the PDF was attached
5. Activate the workflow

- **Carbone** - Template-based document generation
- **PDFMonkey** - Dynamic PDF creation from templates
- **DocuSeal** - Document automation platform
- **Documint** - API-first document generation

Each service provides an API that returns a PDF URL, which you can then pass to the logic function.

## Uploading to other file fields

`createAttachment` covers the common case of attaching a file to a record. To upload into any FILES field (e.g., a custom field on your own object), use `uploadFile` with the field's universal identifier:

```typescript
import { uploadFile } from 'twenty-client-sdk/files';

const uploadedFile = await uploadFile({
  source: { url: 'https://example.com/report.pdf' },
  filename: 'report.pdf',
  fieldMetadataUniversalIdentifier: MY_FILES_FIELD_UNIVERSAL_IDENTIFIER,
});
// Set the field value to [{ fileId: uploadedFile.id, label: 'report.pdf' }]
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "uploadFile() failed to download" | Check the PDF URL is accessible and returns a valid PDF |
| "uploadFile() failed" | Verify the workflow has permission to write Attachments |
| "createAttachment() failed" | Ensure `objectNameSingular` matches your target object's API name |
