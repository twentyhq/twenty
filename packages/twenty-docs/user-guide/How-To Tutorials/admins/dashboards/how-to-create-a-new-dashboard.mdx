## What You'll Accomplish

By the end of this guide, you'll have created a custom dashboard that displays key metrics and information relevant to your business needs.

## Before You Start

Make sure you have:
- Admin permissions in your Twenty workspace
- Data in your CRM (contacts, companies, opportunities, etc.)
- Clear idea of what metrics you want to track

## Understanding Dashboards

Dashboards in Twenty are customizable layouts that can display:
- **Widgets**: Visual components showing data, charts, or information
- **Tabs**: Different sections within a dashboard for organization
- **Layouts**: Flexible grid system for arranging widgets
- **Filters**: Dynamic filtering to show relevant data

## Step-by-Step Instructions

### Step 1: Access Dashboard Creation

1. Navigate to any record page (Person, Company, Opportunity, etc.)
2. Look for the **Page Layout** section or dashboard area
3. Click the **Edit** button or **Customize Layout** option
4. Select **Create New Dashboard** from the options

<img src="/images/how-to/dashboards/step-1.png" alt="Access dashboard creation" />

### Step 2: Configure Dashboard Basics

Set up your dashboard foundation:

1. **Dashboard Name**: Enter a descriptive name
   - Example: "Sales Performance Dashboard" or "Customer Overview"
2. **Description**: Add details about the dashboard's purpose
3. **Visibility**: Choose who can see this dashboard
   - Private: Only you can see it
   - Team: Your team members can access it
   - Workspace: All workspace members can view it

<img src="/images/how-to/dashboards/step-2.png" alt="Configure dashboard basics" />

### Step 3: Add Your First Tab

Organize your dashboard with tabs:

1. **Tab Name**: Give your first tab a clear name
   - Examples: "Overview", "Sales Metrics", "Activity Summary"
2. **Tab Description**: Explain what this tab will contain
3. **Default Tab**: Set whether this should be the default view
4. **Tab Icon**: Choose an icon to represent this tab (optional)

<img src="/images/how-to/dashboards/step-3.png" alt="Add first tab" />

### Step 4: Add Widgets to Your Dashboard

Start adding content to your dashboard:

1. **Click Add Widget**: Look for the + button or "Add Widget" option
2. **Choose Widget Type**: Select from available widget types:
   - **Graph Widget**: Charts and data visualizations
   - **Fields Widget**: Display specific record fields
   - **Timeline Widget**: Show activity history
   - **Tasks Widget**: Display related tasks
   - **Notes Widget**: Show associated notes
   - **Files Widget**: Display attached files
   - **iFrame Widget**: Embed external content

<img src="/images/how-to/dashboards/step-4.png" alt="Add widgets" />

### Step 5: Configure Your First Widget

Let's create a graph widget as an example:

1. **Select Graph Widget**: Choose this widget type
2. **Widget Title**: Enter a descriptive title
   - Example: "Opportunities by Stage"
3. **Data Source**: Choose what data to display
   - Select "Opportunities" object
4. **Chart Type**: Pick visualization type
   - Bar chart, pie chart, line chart, etc.
5. **Grouping**: Choose how to group the data
   - Example: Group by "Stage" field
6. **Filters**: Add any filters to limit the data shown
   - Example: Only show opportunities from this year

<img src="/images/how-to/dashboards/step-5.png" alt="Configure graph widget" />

### Step 6: Position and Size Your Widget

Customize the widget layout:

1. **Drag to Position**: Move the widget to desired location
2. **Resize**: Drag corners to adjust widget size
3. **Grid Alignment**: Widgets snap to grid for clean alignment
4. **Preview**: See how the widget looks with real data

<img src="/images/how-to/dashboards/step-6.png" alt="Position and size widget" />

### Step 7: Add More Widgets

Continue building your dashboard:

1. **Add Fields Widget**: Show key information
   - Configure to display important record fields
   - Choose which fields to include
   - Set display format and layout

2. **Add Timeline Widget**: Show recent activity
   - Configure to show recent interactions
   - Set time range for activities
   - Choose activity types to include

3. **Add Tasks Widget**: Display action items
   - Show tasks related to the current record
   - Filter by assignee or due date
   - Set task status filters

<img src="/images/how-to/dashboards/step-7.png" alt="Add more widgets" />

### Step 8: Create Additional Tabs (Optional)

Organize complex dashboards with multiple tabs:

1. **Click Add Tab**: Create a new tab
2. **Tab Configuration**: Set name, description, and settings
3. **Add Widgets**: Populate the new tab with relevant widgets
4. **Tab Order**: Arrange tabs in logical sequence

<img src="/images/how-to/dashboards/step-8.png" alt="Create additional tabs" />

### Step 9: Set Permissions and Access

Control who can view and edit your dashboard:

1. **View Permissions**: Set who can see the dashboard
2. **Edit Permissions**: Control who can modify the layout
3. **Default Dashboard**: Set as default for specific record types
4. **Team Settings**: Configure team-wide dashboard preferences

<img src="/images/how-to/dashboards/step-9.png" alt="Set permissions" />

### Step 10: Save and Test Your Dashboard

Finalize your dashboard:

1. **Save Dashboard**: Click save to preserve your configuration
2. **Exit Edit Mode**: Switch to view mode to see the final result
3. **Test Functionality**: Verify all widgets display correctly
4. **Check Responsiveness**: Test on different screen sizes
5. **Gather Feedback**: Share with team members for input

<img src="/images/how-to/dashboards/step-10.png" alt="Save and test dashboard" />

## Dashboard Best Practices

### Design Principles
**Keep It Simple**: Don't overcrowd dashboards with too many widgets
**Logical Flow**: Arrange widgets in order of importance
**Consistent Sizing**: Use consistent widget sizes for clean appearance
**Color Coordination**: Use colors that align with your brand

### Performance Optimization
**Limit Data Range**: Use filters to reduce data load
**Optimize Queries**: Choose efficient data groupings
**Monitor Load Times**: Test dashboard performance regularly
**Cache Considerations**: Understand how data refreshes

### User Experience
**Clear Labels**: Use descriptive titles and labels
**Intuitive Navigation**: Organize tabs logically
**Mobile Friendly**: Test on mobile devices
**Regular Updates**: Keep dashboards current and relevant

## Troubleshooting Common Issues

### Widget Not Displaying Data
- Check data source configuration
- Verify field selections and filters
- Ensure you have permission to access the data
- Test with different date ranges

### Dashboard Loading Slowly
- Reduce the number of widgets
- Optimize data filters and queries
- Check internet connection
- Contact support for performance issues

### Permission Problems
- Verify your admin permissions
- Check workspace-level settings
- Ensure team members have appropriate access
- Review object-level permissions

### Layout Issues
- Clear browser cache and refresh
- Try different browser or device
- Check for conflicting browser extensions
- Test in incognito/private mode
