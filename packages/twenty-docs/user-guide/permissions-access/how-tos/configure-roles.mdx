## Understanding Roles

Roles define what users can do in your workspace:
- View, edit, or delete records
- Access specific objects and fields
- Configure workspace settings
- Perform actions like import/export

## Creating a New Role

### Steps
1. Go to **Settings → Roles**
2. Click **+ Create Role**
3. Enter a name (e.g., "Sales Rep", "Manager")
4. Configure permissions
5. Click **Save**

## Configuring Permissions

### Object Permissions

**Set global object access**:
1. In the role, go to **All Objects**
2. Toggle permissions:
   - See Records
   - Edit Records
   - Delete Records
   - Destroy Records

**Set object-specific overrides**:
1. Click **Object-Level Permissions**
2. Select an object
3. Configure specific permissions
4. These override global settings

### Field Permissions

**Configure field access**:
1. Select an object
2. Click on a field
3. Set permission level:
   - See Field
   - Edit Field
   - No Access

### Settings Permissions

**Grant settings access**:
1. In the role, find **Settings Permissions**
2. Toggle **Settings All Access** for full access
3. Or enable specific settings:
   - API key generation
   - Workspace preferences
   - Role assignment
   - Data model configuration
   - Security settings
   - Workflow management

### Action Permissions

**Control workspace actions**:
1. Find **Action Permissions**
2. Toggle **Application All Access** for full access
3. Or enable specific actions:
   - Send Email
   - Import CSV
   - Export CSV

## Role Examples

### Sales Representative
- **Objects**: Full access to People, Companies, Opportunities
- **Settings**: No access
- **Actions**: Send Email, Export CSV

### Sales Manager
- **Objects**: Full access to all standard objects
- **Settings**: View reports, manage team views
- **Actions**: All actions enabled

### Admin
- **Objects**: Full access to everything
- **Settings**: Full access
- **Actions**: Full access

### Read-Only User
- **Objects**: See Records only
- **Settings**: No access
- **Actions**: No import/export

## Assigning Roles

### Assign to Existing Member
1. Go to **Settings → Roles**
2. Select the role
3. Click **Assignment** tab
4. Click **+ Assign to member**
5. Select the member
6. Confirm

### Set Default Role
1. Go to **Settings → Roles**
2. Find **Default Role** option
3. Select the role for new members

## Best Practices

### Role Design
- Create roles based on job functions
- Start restrictive, add permissions as needed
- Document what each role can do
- Review roles periodically

### Security
- Limit admin access to necessary people
- Use field-level permissions for sensitive data
- Audit permissions regularly
- Remove access promptly when needed
