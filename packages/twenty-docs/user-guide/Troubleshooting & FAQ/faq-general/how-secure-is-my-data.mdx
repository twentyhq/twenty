Data security is a top priority at Twenty. We implement comprehensive security measures to protect your business data, maintain privacy, and ensure compliance with industry standards. This guide explains our security practices and what you can do to keep your data safe.

- Hosted on leading cloud platforms with SOC 2 compliance
- Multi-region deployment with automatic failover

**Data Centers**:
- Tier IV data centers with 24/7 physical security
- Biometric access controls and surveillance
- Redundant power and cooling systems
- Fire suppression and environmental monitoring
- Compliance with ISO 27001 standards

### Application Security

**Secure Development Practices**:

## Data Protection

### Encryption Standards

**Data at Rest**:
- AES-256 encryption for all stored data
- Encrypted database storage with key rotation
- Secure file storage with client-side encryption
- Encrypted backups and disaster recovery data
- Hardware security modules (HSMs) for key management

**Data in Transit**:
- TLS 1.3 encryption for all communications
- Certificate pinning for enhanced security
- Encrypted API communications
- Secure WebSocket connections
- End-to-end encryption for sensitive operations

### Data Isolation

**Multi-Tenant Security**:
- Complete data isolation between workspaces
- Logical separation with access controls
- Encrypted tenant-specific encryption keys
- Isolated backup and recovery processes
- No cross-tenant data access or sharing

**Database Security**:
- Row-level security and access controls
- Encrypted database connections
- Regular security patches and updates
- Database activity monitoring and logging
- Automated backup encryption and testing

## Access Control and Authentication

### User Authentication

**Strong Authentication Methods**:

**Session Management**:
- Secure session tokens with expiration

### Authorization and Permissions

- Granular permission system
- Object-level and field-level access controls
- Principle of least privilege enforcement
- Regular access reviews and audits
- Automated permission inheritance

**API Security**:
- API key authentication with scoped permissions
- Rate limiting and throttling
- Request signing and validation
- Audit logging for all API access
- Webhook signature verification

## Frequently Asked Questions

### Data Location and Sovereignty

**Where is my data stored?**
Your data is stored in secure, enterprise-grade data centers in your selected region. We provide options for data residency to meet local compliance requirements.

**Can I control where my data is stored?**
Yes, enterprise customers can specify data residency requirements. We offer multiple geographic regions to meet compliance and performance needs.

### Data Access and Control

**Who can access my data?**
Only authorized personnel with legitimate business needs can access customer data, and all access is logged and monitored. We maintain strict access controls and regular audits.

**Can Twenty employees see my data?**
Twenty employees can only access customer data in specific circumstances (such as providing support with explicit permission) and all access is logged and audited.

### Backup and Recovery

**How often is my data backed up?**
We perform continuous backups with point-in-time recovery capabilities. Backups are encrypted and stored in geographically distributed locations.

**Can I restore deleted data?**
Depending on your plan and the type of data, we may be able to restore recently deleted information. Contact support immediately if you need data recovery assistance.

### Compliance and Certifications

**What compliance standards does Twenty meet?**
Twenty complies with SOC 2 Type II, GDPR, CCPA, and other relevant standards. We're working toward additional certifications including ISO 27001.

**Do you provide compliance documentation?**
Yes, we provide compliance documentation, audit reports, and data processing agreements to enterprise customers upon request.

## Reporting Security Issues

### How to Report

**Security Vulnerabilities**:
- Email: security@twenty.com
- Include detailed information about the vulnerability
- Provide steps to reproduce if possible
- Allow reasonable time for response and resolution

**Suspected Security Incidents**:
- Contact support immediately
- Provide details about the suspected incident
- Preserve any relevant evidence
- Follow your organization's incident response procedures

### What to Expect

**Response Timeline**:
- Initial acknowledgment within 24 hours
- Regular updates on investigation progress
- Coordinated disclosure for confirmed vulnerabilities
- Post-resolution summary and recommendations

Security is an ongoing commitment at Twenty. We continuously invest in security improvements, stay current with emerging threats, and work to exceed industry standards for data protection. Your trust is essential to our success, and we take that responsibility seriously.
