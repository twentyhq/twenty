## About Sales Pipelines

A sales pipeline in Twenty is a Kanban view of your Opportunities object, where each column represents a stage in your sales process.

## Default Pipeline Stages

Twenty comes with default opportunity stages:
- **New**: Fresh opportunities just identified
- **Screening**: Initial qualification
- **Meeting**: Engaged in discussions
- **Proposal**: Proposal submitted
- **Customer**: Deal closed won

## Customizing Your Pipeline

### Edit Stage Names
1. Go to **Settings â†’ Data Model**
2. Select **Opportunities**
3. Find the **Stage** field
4. Click to edit stage options
5. Rename stages to match your process

### Add New Stages
1. Navigate to the Stage field settings
2. Click **+ Add option**
3. Enter the new stage name
4. Choose a color for the stage
5. Drag to position it in order

### Remove Stages
1. Open the Stage field settings
2. Find the stage to remove
3. Click **Delete**
4. Reassign any existing opportunities first

## Pipeline Metrics

### Stage Counts
Each column header shows:
- Number of opportunities in that stage
- Total value of opportunities (if enabled)

### Weighted Pipeline
Calculate expected revenue by stage probability:
- Assign probability percentages to each stage
- View weighted totals in pipeline reports

## Working with Opportunities

### Creating Opportunities
1. Click **+ New** in the Opportunities view
2. Or click **+** in a specific stage column
3. Fill in opportunity details
4. The opportunity appears in the appropriate stage

### Moving Through Stages
Drag and drop opportunities between stages:
- Grab the opportunity card
- Drag to the new stage column
- Drop to update the stage

### Opportunity Fields
Key fields for pipeline management:
- **Amount**: Deal value
- **Close Date**: Expected close
- **Company**: Related company
- **Owner**: Assigned salesperson

## Best Practices

### Pipeline Health
- **Keep it current**: Update stages as deals progress
- **Review regularly**: Weekly pipeline reviews
- **Clean up**: Archive or close stale opportunities

### Stage Design
- **5-7 stages**: Optimal for visibility
- **Clear criteria**: Define what qualifies for each stage
- **Linear flow**: Design stages in logical progression

### Forecasting
- **Realistic probabilities**: Base on historical data
- **Regular updates**: Keep close dates accurate
- **Stage discipline**: Move deals promptly through stages
