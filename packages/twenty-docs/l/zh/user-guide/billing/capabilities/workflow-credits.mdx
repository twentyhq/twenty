## 概览

在 Twenty 中，积分为您的工作流程自动化提供支持。 每个工作流程操作会根据其复杂性消耗积分。

## 积分分配

积分以您的计费周期为准，而非您的计划：

| 计费周期 | 积分      |
| ---- | ------- |
| 每月   | 500万/月  |
| 年度计划 | 5000万/年 |

<Note>
  每月500万积分旨在让您无需担心成本即可运行自动化。 对于使用标准操作的大多数工作流程，这已经绰绰有余。 仅在运行高级代码节点或由 AI 驱动的功能时，您才需要额外积分。
</Note>

## 积分消耗

不同的操作会消耗不同数量的积分：

| 操作类型                      | 信用使用情况   |
| ------------------------- | -------- |
| **基础操作** (搜索、更新、创建记录)     | 极少       |
| **复杂操作** (代码节点、外部 API 调用) | 更多积分     |
| **人工智能提示** (即将推出)         | 根据使用情况而定 |

当工作流执行时，积分会实时扣除。

## 监控使用情况

跟踪您的积分消耗：

1. 前往**设置 → 结算**
2. 查看您当前的使用量和剩余积分
3. 监控趋势，以便在需要时规划额外积分

## 购买额外的积分

需要更多积分吗？

1. 前往**设置 → 结算**
2. 点击相应选项购买额外的积分包
3. 选择您需要的数量
