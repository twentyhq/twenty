## 連接選項

### Google 帳戶（Gmail 和 Google 日曆）

1. 前往 **設置 → 帳戶**
2. 點擊 **新增帳戶**
3. 選擇 **繼續使用 Google**
4. 授權 Twenty 訪問您的 Gmail 和 Google 日曆
5. 您的電子郵件和日曆事件將自動開始同步

### Microsoft 帳戶（Outlook 和 Microsoft 日曆）

1. 前往 **設置 → 帳戶**
2. 點擊 **新增帳戶**
3. 選擇 **繼續使用 Microsoft**
4. 授權 Twenty 訪問您的 Outlook 和 Microsoft 日曆
5. 您的電子郵件和日曆事件將自動開始同步

### SMTP/CalDAV Setup (Other Providers)

對於其他電子郵件和日曆提供商：

1. 前往 **設置 → Releases → Lab** 啟用此功能
2. 返回 **設置 → 帳戶**
3. Configure SMTP settings for email
4. 配置日曆的 CalDAV 設置
5. 測試連接

### 多個郵箱

- **無限帳戶**：每位用戶可連接多個電子郵件帳戶
- **帳戶管理**：在不同郵箱之間切換
- **同步設置**：為每個郵箱配置不同設置

<Warning>
只能連接實際郵箱（例如，擁有自己收件箱的 support@domain.com）。 
只能連接實際郵箱（例如，擁有自己收件箱的 support@domain.com）。 無法將轉寄至其他郵箱的電子郵件別名連接到 Twenty。

## 電子郵件配置

### 郵件可見性

選擇您的電子郵件的不同可見級別：

- **僅元數據**：僅共享基本信息（發件人、收件人、日期、時間）
- **主題和元數據**：共享主題行以及元數據
- **所有電子郵件內容**：共享整個電子郵件內容，包括附件

### 聯繫人自動創建

- **停用**：不自動創建聯繫人
- **對於已發送和接收的郵件**：為所有外部電子郵件互動創建聯繫人
- **僅對於已發送的郵件**：僅為您發送的郵件創建聯繫人
- **注意**：內部郵件（同一域）從不同步以維護隱私

### 控制哪些電子郵件與消息文件夾選擇同步（實驗功能）

控制哪些電子郵件文件夾與 Twenty 同步：

1. 前往 **設置 → Releases → Lab** 並啟用 **消息文件夾**
2. 返回 **設置 → 帳戶** 並選擇您連接的電子郵件帳戶
3. 選擇要同步的文件夾：
   - **收件箱**：首要收入電子郵件
   - **已發送**：您已發送的外發郵件
   - **自定義文件夾**：您希望包括的任何特定文件夾
   - **排除文件夾**：跳過如垃圾郵件、垃圾筒或個人文件夾

這使您可以精確控制哪些電子郵件出現在您的 CRM 中而不需同步全部。

**同步內容**：

- **外部電子郵件**：選定文件夾中的所有外部聯繫人電子郵件
- **內部電子郵件**：不同步（相同域名的電子郵件保持私密）
- **附件**：2026 年第一季度推出

**注意**：我們不提供選擇性同步的抄送電子郵件地址。 **注意**：我們不提供選擇性同步的抄送電子郵件地址。 取而代之，使用上述消息文件夾功能來實現相同級別的控製，以決定哪些電子郵件與 Twenty 同步。

## 日曆配置

### 事件可見性

選擇在您的工作空間中對其他用戶可見的內容：

- **全部**：整個事件詳細信息將與您的團隊共享
- **元數據**：僅將日期和參加者與您的團隊共享

### 會議的聯繫人自動創建

- **是**：自動為會議與 CRM 中不在的參加者創建聯繫人
- **否**：僅將會議鏈接到現有聯繫人

### 控制哪些事件同步

- **會議導入**：自動導入日曆事件
- **聯繫人鏈接**：將會議鏈接到人員和公司記錄

**同步內容**：

- **會議**：具有外部參加者的日曆事件
- **聯繫人鏈接**：事件自動鏈接至 CRM 記錄
- **團隊事件**：共享日曆的可見性

## 同步頻率

**每5分鐘更新**：電子郵件和日曆數據在首次導入後每5分鐘自動同步。
