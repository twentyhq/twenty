Below are workflow examples you could roll out to automate repetitive tasks to reduce manual work and keep your data up-to-date.

## 數據管理自動化

### 自動標記個人電子郵件

**問題**：您的團隊需要識別個人與商業電子郵件地址，以便更好地瞄準和合規。

**Solution**: Create a workflow that automatically updates an "is personal email" field whenever an email address is added or updated.

**觸發器**：記錄已更新（人員對象，電子郵件字段）

**操作**：

- 編寫代碼以檢查電子郵件域是否與常見的個人服務提供商（gmail.com，yahoo.com等）匹配
- 更新記錄以設置"是否為個人電子郵件"標記

### Lead Assignment - Round Robin

**問題**：需要公平地分配新潛在客戶給銷售團隊，確保工作負擔均衡。

**解決方案**：利用輪流分配系統自動分配新潛在客戶給銷售代表。

**觸發器**：記錄已創建（人員對象）

**操作**：

- 搜索記錄以找到最後分配的代表
- 編寫代碼以確定輪換中的下一位代表
- 更新記錄以將潛在客戶分配給選擇的代表
- 發送電子郵件通知分配的代表

### 領先分配 - 基於地區

**問題**：應根據地理區域或公司特徵分配潛在客戶。

**解決方案**：基於位置，公司規模或行業將潛在客戶導向適當的銷售代表。

**觸發器**：記錄已創建（人員或公司對象）

**操作**：

- 編寫代碼根據位置/行業規則確定領土
- 搜索記錄以查找領土所有者
- 更新記錄以分配潛在客戶
- 創建記錄以進行後續任務

## 銷售流程自動化

### 機會階段管理 - 關閉已贏得

**問題**：當交易完成時，需要在不同的記錄和團隊成員間執行多個手動更新。

**Solution**: Automatically handle all post-win activities when an opportunity moves to "Closed Won".

**Trigger**: Record is Updated (Opportunities object, Stage field = "Closed Won")

**操作**：

- 更新記錄以將公司類型從"潛在"更改為"客戶"
- 創建記錄以分配給客戶經理的入職任務
- 向客戶成功團隊發送電子郵件通知
- HTTP請求以更新外部計費系統

### 機會階段管理 - 關閉已失去續簽

**問題**：當續簽機會失敗時，需要更新客戶狀態以便正確的賬戶管理。

**解決方案**：當續簽交易失敗時，自動更新客戶狀態。

**觸發器**：記錄已更新（機會對象，階段="關閉已失去"並且類型="續簽"）

**操作**：

- 更新記錄以將公司類型從"客戶"更改為"流失客戶"
- 創建記錄以進行流失分析任務
- 發送電子郵件提醒給客戶成功經理
- 更新記錄以添加流失日期和原因

### 陳舊機會警報

**問題**：機會未更新導致交易變冷，預測變得不可靠。

**解決方案**：當機會最近未更新時發送自動提醒。

**觸發器**：按計畫表（每天）

**操作**：

- 搜索記錄以查找在X天內未更新的機會
- 編寫代碼以格式化包含機會詳情的提醒消息
- 發送電子郵件給機會持有者和經理
- 如無回應，創建後續任務記錄

## 生產力自動化

### 每週任務回顧

**問題**：團隊成員失去對即將到來的任務和截止日期的追蹤。

**解決方案**：發送自動的每週電子郵件提醒，其中包含任務摘要。

**觸發器**：按計畫表（每週一上午8點）

**操作**：

- 搜索記錄以查找本週由分配者的到期任務
- 編寫代碼按人員格式化任務列表
- 向每位團隊成員發送電子郵件，附上他們的任務回顧
- 向經理發送電子郵件，附上團隊概覽

### 會議後續自動化

**問題**：會議的關鍵操作項目被遺忘或延遲。

**解決方案**：當會議被安排或完成時，自動創建後續任務。

**觸發器**：記錄已創建（活動對象，類型="會議"）

**操作**：

- 創建記錄以進行會前準備任務
- 創建記錄以進行會後後續任務
- 向參會者發送電子郵件提醒
- 更新記錄以將任務鏈接到會議

### 跨對象字段同步

**問題**：您需要能夠輕鬆訪問相關記錄中的信息（例如，商機記錄上的主要聯繫人的電子郵件）。

**解決方案**：自動同步相關對象之間的字段，直到可用嵌套字段。

**觸發器**：記錄已更新（機會對象，聯絡人字段）

**操作**：

- 搜索記錄以查找鏈接人的詳細信息
- 更新記錄以將電子郵件地址複製到機會
- 更新記錄以將電話號碼複製到機會
- 更新記錄以複製公司信息

## 數據驗證與清理

### 電話號碼標準化

**問題**：電話號碼以不同格式輸入，這使得它們難以用於撥打電話或發消息。

**解決方案**：每當輸入新的電話號碼時，自動將其格式化為標準格式。

**觸發器**：記錄已更新（人員對象，電話字段）

**操作**：

- 編寫代碼以解析和格式化電話號碼
- 更新記錄以使用標準化的電話格式
- 更新記錄以添加缺失的國碼
