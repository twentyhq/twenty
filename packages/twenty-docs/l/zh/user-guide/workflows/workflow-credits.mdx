## 按方案分配點數

根據您的訂閱方案，工作流程點數按工作空間分配：

- **試用期**：500萬點數
- **專業方案**：每月1000萬點數
- **組織方案**：每月2000萬點數

## 如何運作點數消耗

點數是在工作流程執行時消耗的，而不是在您創建它們時。 每個工作流程操作的點數消耗取決於其複雜程度：

### 按操作類型消耗點數

- **基本內部操作**：非常低的點數消耗
  - 搜索記錄
  - 創建記錄
  - 更新記錄
  - 刪除記錄
  - 表單操作

- **複雜操作**：更高的點數消耗
  - 代碼操作 (JavaScript執行)
  - 向外部服務發出HTTP請求

- **AI功能**：顯著的點數消耗（即將推出）
  - AI提示和處理將根據使用情況消耗點數

### 實時扣除

工作流程執行時點數會實時扣除。 這意味著：

- 草稿工作流程不消耗點數
- 只有激活並運行的工作流程使用您的點數分配
- 失敗的工作流程仍會為已完成的步驟消耗點數

## 管理點數

### 檢查點數使用情況

1. 前往**設定 → 計費**
2. 查看您當前的點數消耗和剩餘餘額
3. 監控使用模式以優化您的工作流程

### 購買額外點數

如果需要超出方案分配的更多點數：

1. 前往**設定 → 計費**
2. 點擊選項以購買額外點數。 有不同規模的套餐可供選擇。
3. 點數將添加到您的當前餘額中

## 最佳做法

### 高效的工作流程設計

- **開始簡單**：從基礎動作開始，然後逐步添加複雜性
- **在草稿中測試**：在激活前徹底測試工作流程，以避免因錯誤浪費點數
- **最小化HTTP請求**：使用高效的搜索標準並儘可能合併操作
- **批次處理**：有效使用批量操作和迭代器操作
- **錯誤處理**：實施適當的錯誤處理，以防止不必要的重試
- **手動觸發優化**：對於手動觸發，選擇“批量”可用性，以在一次工作流程運行中處理多個記錄（詳情請參閱[工作流程功能](/l/zh/user-guide/workflows/workflow-features)）

### 操作優化

- 在可能的情況下使用基本內部操作（較低的點數消耗）
- 優化代碼操作以提高效率
- 對於非緊急過程考慮手動觸發
- 批量操作以減少單個操作調用

### 點數管理

- **定期監控**：每週在設定 → 計費中檢查您的點數使用情況
- **使用追蹤**：跟踪消耗模式以避免意外的超支
- **按計劃行事**：如果您經常需要更多點數，請考慮升級您的方案
- **工作流程審查**：定期審查高消耗的工作流程並停用未使用的
- **策略規劃**：基於點數可用性規劃工作流程的部署

有關詳細的計費信息和點數管理，請訪問Twenty工作空間中的**設定 → 計費**。
