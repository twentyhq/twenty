title: 工作流程积分
description: 了解工作流程积分的消耗与管理。

在 Twenty 中，工作流程积分为您的自动化提供支持。 了解其如何运作，有助于优化成本并有效管理您的自动化预算。

## 积分分配

工作流程积分基于您的计费周期分配，而非您的计划：

| 计费周期     | 积分          |
| -------- | ----------- |
| **月度订阅** | 每月 500 万积分  |
| **年度订阅** | 每年 5000 万积分 |

<Note>
  每月 500 万积分对于标准自动化而言已相当充足。 大多数团队在典型的工作流程使用中不会超出此限制。 额外积分主要用于高级 Code 操作和由 AI 驱动的工作流程。
</Note>

## 积分消耗工作原理

工作流程执行时消耗积分，而不是创建时。 每个工作流程操作根据其复杂性消耗积分：

### 按操作类型消耗积分

* **基本内部操作**：非常低的积分消耗
  * 搜索记录
  * 创建记录
  * 更新记录
  * 删除记录
  * 表单操作

* **复杂操作**：更高的积分消耗
  * 代码操作（JavaScript 执行）
  * 向外部服务发送HTTP请求

* **AI 功能**：更高的积分消耗
  * AI 代理操作会根据所用 AI 模型消耗积分
  * 更复杂的提示词和更长的输出会消耗更多积分

* **延迟操作**：积分消耗极少
  * Delay 节点在执行时会消耗**1 积分**
  * 在等待期间**不会消耗积分**
  * 5 分钟的延迟与 5 天的延迟消耗的积分相同

### 实时扣减

工作流程执行时，积分实时扣减。 这意味着：

* 草稿工作流程不消耗积分
* 仅激活并运行中的工作流程使用您的积分额度
* 失败的工作流程仍为已完成的步骤消耗积分

## 管理积分

### 检查积分使用情况

1. 前往**设置 → 结算**
2. 查看您当前的积分消耗和剩余余额
3. 监控使用模式以优化工作流程

### 购买额外的积分

如果您需要超出计划分配的更多积分：

1. 前往**设置 → 结算**
2. 点击选项以购买额外的积分。 提供不同大小的套餐。
3. 积分将添加到您当前的余额

## 最佳实践

* **批处理**：有效利用批量操作和迭代器操作
* **手动触发优化**：对于手动触发，选择 `Bulk` 可用性以在单个工作流程运行中处理多个记录
* 优化代码操作以提高效率
* 进行批量操作以减少个别动作调用
