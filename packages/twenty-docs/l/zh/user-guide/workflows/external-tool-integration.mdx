Below are workflow examples you could roll out to connect Twenty with the rest of your stack.

## Data Ingestion Use Cases

### 網頁表單提交

**問題**: 您需要從網站表單、著陸頁或聯絡表中捕獲線索並直接導入 Twenty。

**解決方案**: 使用 Webhook 觸發器自動從表單提交創建記錄。

**設置**:

- 創建具有 Webhook 觸發器的工作流程
- 配置 Webhook 以期望接收表單數據（姓名、電子郵件、公司等）
- 將 Webhook 方法設置為 POST
- 在觸發器中定義預期的數據結構

**觸發器**: Webhook（由您的表單發送的 POST 請求）

**操作**:

- 搜索記錄以檢查人員/公司是否已存在
- 分支：如果存在 → 更新記錄，如果不存在 → 創建記錄
- 創建記錄以為業務代表分配跟進任務
- 發送電子郵件通知給銷售團隊

### 產品數據同步

**問題**: 您的銷售團隊需要對儲存在您的數據倉庫中的產品使用情況、計費或功能採用數據的可見性。

**解決方案**: 定期將產品數據同步到 Twenty，為銷售提供客戶的背景信息。

**觸發器**: 按排程（每日或每週）

**操作**:

- 向數據倉庫 API 發送 HTTP 請求
- 編寫代碼以處理和格式化數據
- 使用迭代器功能進行接下來的步驟
- 搜索記錄以查找匹配的公司記錄
- 更新記錄以新增產品使用指標
- 當使用量低於阈值時創建任務記錄

### 通話記錄員的會議筆記

**問題**: 銷售電話中的重要見解常常丟失或未能妥善記錄在 CRM 中。

**解決方案**: 自動從通話錄音系統創建筆記和動作項目。

**觸發器**: Webhook（來自通話錄音平台）

**操作**:

- 編寫代碼以提取會議摘要和行動項目
- 搜索記錄以找到相關的機會或聯絡人
- 創建會議摘要的筆記記錄
- 基於行動項目的後續任務創建記錄
- 發送電子郵件給參與者，附上摘要和後續步驟

### 數據增強

**問題**: 您的聯絡人和公司記錄缺少重要的人口和公司信息。

**解決方案**: 使用外部數據供應商自動增強記錄。

**觸發器**: 記錄被創建（人員或公司對象）

**操作**:

- 向增強 API 發送 HTTP 請求
- 編寫代碼以處理增強響應
- 使用迭代器功能進行接下來的步驟
- 使用額外的公司/聯絡人信息更新記錄
- 當標識出高價值潛在客戶時創建銷售任務記錄
- 若增強揭示關鍵購入信號，則發送電子郵件警報

## 數據分發用例

### 新聞稿訂戶管理

**問題**: 您希望使用專業的電子郵件工具向您的 CRM 數據中的特定細分發送營銷電子郵件。

**解決方案**: 在需要時將訂戶列表匯出到您的電子郵件行銷平台。

**設置**: 在 Twenty 中創建一個檢視來顯示所有的新聞稿接收者

**觸發器**: 手動啟動（當沒有選擇記錄時）

**操作**:

- 使用新聞稿檢視條件搜尋記錄
- 編寫代碼以格式化電子郵件平台的電子郵件地址
- 通過 HTTP 請求將訂戶新增到您的電子郵件行銷工具
- Create Record for campaign tracking
- Send Email confirmation to marketing team

### 電子郵件序列觸發器

**問題**: 您希望根據 CRM 活動觸發複雜的電子郵件序列，使用專用的電子郵件自動化工具。

**解決方案**: 當特定事件發生時，將新潛在客戶或顧客發送到您的電子郵件自動化平台。

**觸發器**: 紀錄被創建（符合特定條件的人員對象）

**操作**:

- 編寫代碼以確定適當的電子郵件序列
- 通過 HTTP 請求將聯絡人新增到電子郵件自動化平台
- Update Record to track sequence enrollment
- Create Record for follow-up task to monitor engagement

### 潛在客戶得分集成

**問題**: 您需要運用 CRM 數據與外部信號結合的高級潛在客戶評分。

**解決方案**: 將潛在客戶數據發送到外部評分工具或在工作流程中實施評分邏輯。

**選項 1 - 外部工具**:

**觸發器**: 紀錄被更新（人員對象）

**操作**:

- 通過 HTTP 請求將潛在客戶數據發送到評分平台
- 編寫代碼以處理評分回應
- 更新紀錄以保留潛在客戶評分
- 如果評分超出閾值，則創建業務任務的紀錄

**選項 2 - 內部邏輯**:

**觸發器**: 紀錄被更新（人員對象）

**操作**:

- 編寫代碼與評分算法（公司規模、行業、行為）
- 使用計算得分更新紀錄
- 發送電子郵件警報給高評比分的業務代表

### 發票生成

**問題**: 當交易結束時，您的計費系統需要更新客戶和交易資訊。

**解決方案**: 當機會贏得時，自動將交易數據發送到您的發票系統。

**觸發器**: 紀錄被更新（機會對象，階段 = "關閉並贏得"）

**操作**:

- 搜索紀錄以獲取完整的客戶信息
- 編寫代碼以格式化計費系統的數據
- 通過 HTTP 請求在計費平台中創建客戶
- 通過 HTTP 請求生成發票
- 更新紀錄以儲存發票參考
- 將發票詳情發送給財務團隊

## 高級集成模式

### 雙向同步

**問題**: 您需要將 Twenty 和另一個系統的數據雙向同步。

**Solution**: Combine scheduled workflows with webhook triggers for real-time sync.

**從 Twenty 到外部系統**:

**觸發器**: 紀錄被更新（任何相關對象）

**操作**:

- 通過 HTTP 請求更新外部系統
- 更新紀錄以跟蹤同步狀態和時間戳記

**從外部系統到 Twenty**:

**觸發器**: Webhook（來自外部系統）

**操作**:

- 搜索紀錄以找到匹配的紀錄
- 使用外部系統的新數據更新紀錄
- 如果需要，創建紀錄以解決衝突任務

### 多步驟數據處理

**問題**: 來自外部來源的數據需要進行複雜的處理，然後才能在 Twenty 中使用。

**解決方案**: 使用代碼操作進行數據轉換和驗證。

**觸發器**: Webhook 或按計劃

**操作**:

- 編寫代碼以驗證輸入數據格式
- 編寫代碼以轉換數據結構
- 編寫代碼以應用業務規則和計算
- 搜索紀錄以檢查重複項
- 使用處理過的數據創建或更新紀錄
- 如果檢測到數據質量問題，則發送電子郵件警報

## 實施提示

- 安全地將 API 金鑰儲存在“設置”→“API 和 Webhooks”中
- 使用 HTTPS 對所有外部 API 請求
- 注意 API 限速 - 儘可能使用計劃的工作流程
- 當不需要實時處理時，考慮“按計劃”批量更新
- Remember the 100 concurrent workflow limit per workspace - use "Bulk" availability for manual triggers when processing multiple records (see [Workflow Features](/l/zh/user-guide/workflows/workflow-features) for details)
- Test with sample data before activating workflows
