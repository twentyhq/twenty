## 什麼是關係字段？

關係字段將一個對象的記錄連接到另一個對象的記錄。 例如： 例如：

- **人員** → **公司**（每個人員都在一家公司工作）
- **機會** → **人員**（每筆交易都有一個聯繫人）
- **任務** → **機會**（每個任務都與特定的交易相關）

## 創建關係字段

### 1. 添加字段

前往 **設置 → 數據模型 → [您的對象]** 並點擊 **添加字段**。

### 2. 選擇關係類型

- **字段類型**：選擇 "關係"
- **目標對象**：選擇要連接的對象
- **關係**：目前僅支持**一對多**的關係。 Make sure to create the relationship in the right direction. Make sure to create the relationship in the right direction.

### 3. 配置字段名稱

您需要為關係的雙方設置名稱：

- **源字段名稱**：字段會在當前對象上以此名稱出現
- **目標字段名稱**：反向字段會在目標對象上以此名稱出現

<Warning>

字段名稱一旦保存為關係後便無法編輯，因為這會影響到 API 的結構。 謹慎選擇。
 謹慎選擇。
</Warning>

## 與團隊成員相關的關係

您可以創建與任何對象的關係，包括 **工作空間成員**（您的 Twenty 團隊用戶）。 這對於創建所有權字段非常有用：

- **賬戶負責人**：將公司鏈接到管理它的團隊成員
- **交易負責人**：將機會分配給特定的銷售人員

當您創建與 `工作空間成員` 的關係時，您會在下拉選擇中看到您的團隊成員的姓名，方便分配所有權和責任。

## 最佳實踐

- **在創建之前計劃您的關係**
- **雙方字段名稱使用清晰、描述性的名稱**
- **使用樣本數據測試關係**在完整實施之前
