## About Fields

字段類似於電子表格中的列。 它們存儲不同類型的數據，如文本、數字或日期。 字段可以是標準（內建）或自訂（由您創建的）。

### 標準字段

標準字段與 Twenty 緊密集成，用於處理常見的業務需求。

例如，「名字」和「姓氏」是「個人」對象中的標準字段。 它們存儲個人姓名的文本數據。

您無法刪除標準字段，但如果不需要可以將其停用。

您還可以自訂標準`SELECT`類字段的選項，例如機會階段的選項。

<img src="/images/user-guide/fields/standard-fields.png" style={{width:'100%'}}/>

### 自訂字段

可以向任何對象中添加自訂字段。 您可以存儲文本、數字、日期、下拉選項等。 使用自訂字段來跟踪特定於您業務的信息。

例如，SpaceX 的自訂字段可能是「火箭啟動狀態」，用於指示火箭是否正在運行。

## 停用字段

您可以停用某個字段以將其從應用中隱藏，而不丟失數據。 將其視為隱藏字段而非刪除字段。 將其視為隱藏字段而非刪除字段。

操作方法如下：

1. 在對象設置中找到您想要停用的字段。

2. 點擊字段旁邊的三個點 `⋮`以打開菜單。

3. 從下拉菜單中選擇「停用」。

<img src="/images/user-guide/fields/deactivate-field.png" style={{width:'100%'}}/>

停用字段會發生什麼？

1. 在應用中：字段消失，您無法向其添加新值。

2. 現有關係：如果是關係字段，現有連結會保留，但您無法創建新連結。

3. API 訪問：您仍然可以通過 API 訪問字段及其數據。

您可以重新激活標準和自訂欄位，或者永久刪除它們。

## Make Fields Unique

創建唯一字段，以確保不同記錄無法使用相同的值。 例如，電子郵件地址對每個人都是唯一的。

如果設置唯一性時出錯，請檢查數據中是否存在重複值（包括已刪除的記錄）。

## 字段配置最佳實踐

### 命名約定和限制

- 創建後關係字段名稱不能更新（影響 API 結構）
- 單數和複數名稱必須不同：我們的 GraphQL API 需要不同的名稱來進行變更
- 受保護的字段名稱：某些名稱被保留用於系統使用（例如，`Type`）

### 貨幣和電話字段

- 默認貨幣：可以通過數據模型進行配置
- 默認國家代碼：可以通過數據模型為電話欄位配置

### 選擇字段

- 可以為每個選擇字段選擇一個默認選項

### 記錄文本字段

- 每個對像有一個主要顯示字段：此字段出現在最左列，並在連接到其他對象時代表記錄。 必須是文本字段。 每個對像有一個主要顯示字段：此字段出現在最左列，並在連接到其他對象時代表記錄。 必須是文本字段。 例如，People 使用「Name」作為主要字段，因此當您將人員連接到公司時，您會在公司的視圖中看到他們的姓名。

### 關係字段

- 連接對象：關係字段將來自不同對象的記錄鏈接起來。 有關創建和管理關係的詳細信息，請參閱我們的[關係字段](/l/zh/user-guide/data-model/relation-fields)文章。
