## 概览

将您的工作区数据导出为 CSV，用于备份、报表或迁移。

**使用场景：**

* **定期备份** — 保留您的数据副本
* **外部报表** — 在 Excel、Google 表格或 BI 工具中分析数据
* **迁移** — 将数据迁移到其他系统
* **批量更新** — 通过导出、编辑并重新导入来更新记录

## 您需要了解的内容

### 导出限制

* **每次导出最多 20,000 条记录**
* 仅导出**可见列**
* 仅导出**筛选后的记录**（基于您当前视图）

<Note>对于更大的导出（20,000+ 条记录），请使用筛选器分批导出，或使用 [API](/l/zh/developers/extend/capabilities/apis)。</Note>

### 权限

导出数据需要拥有 **"Export CSV"** 权限。 如果您没有此选项，请联系您的工作区管理员。

## 步骤 1：导航到对象

前往您要导出的对象：

* **People** — 用于联系人
* **Companies** — 用于组织
* **Opportunities** — 用于商机
* **自定义对象** — 您创建的任意对象

## 步骤 2：配置您的视图

\*\*重要：\*\*导出仅包含您当前视图中可见的内容。

### 添加/移除列

1. 单击 **Options → Fields**（或单击列末尾的 **+**）
2. 勾选您要导出的字段
3. 取消勾选不需要的字段

### 筛选记录（可选）

如果您只需要数据的一个子集：

1. 单击 **Filter**
2. 添加筛选条件（例如："Created date > January 1, 2024"）
3. 仅会导出匹配的记录

### 排序记录（可选）

1. 单击列标题进行排序
2. 导出将遵循您的排序顺序

<Note>**创建专用的导出视图。** 保存一个专为导出配置的视图，这样每次就无需重新配置。</Note>

## 步骤 3：导出数据

1. 单击表格右上角的 **⋮** 图标
2. 选择 **Export view**
3. 选择 CSV 文件的保存位置
4. 等待下载完成

## 导出内容

| 包含           | 不包含      |
| ------------ | -------- |
| 所有可见列        | 隐藏列      |
| 与当前筛选条件匹配的记录 | 被过滤掉的记录  |
| 自定义字段值       | 不在视图中的字段 |
| 记录 ID        | 文件附件     |
| 关系 ID        | 图片       |

### 关系字段

关系 ID 仅在关系的\*\*“多”端\*\*导出：

* **People 导出** 包含一个 `companyId` 列（People → Company 关系）
* **Companies 导出** 不包含 `peopleIds`（Companies 是“一”端）

这意味着您可以使用 People 的导出来重新导入并保持与 Company 的链接，但您需要在导入 Companies 之后再次导入 People 以重新创建这些关系。

## 针对特定目的的导出

### 用于备份

1. 创建一个显示**所有字段**的视图
2. 移除所有筛选器以包含所有记录
3. 分别导出每种对象类型
4. 将导出文件存储在安全位置
5. 设置重复提醒（每周/每月）

### 用于外部报表

1. 只包含您进行分析所需的字段
2. 应用筛选器以聚焦相关数据
3. 考虑按您将要分析的字段进行排序

### 用于批量更新

1. 导出您要更新的记录
2. 包含唯一标识符（`email`、`domain` 或 `id`）
3. 编辑导出的文件
4. 重新导入以更新记录
   参见：[如何更新现有记录](/l/zh/user-guide/data-migration/how-tos/update-existing-records-via-import)

### 用于迁移

如果您为迁移到其他系统而进行导出：

1. **分别导出每个对象** — People、Companies、Opportunities 等。
2. **包含 ID 字段** — 有助于维护关系
3. **记录字段映射** — 记录 Twenty 字段如何映射到您的目标系统

## 处理大型数据集（20,000+ 条记录）

导出限制为 20,000 条记录。 对于更大的数据集：

### 选项 1：分批导出

1. 添加一个筛选器（例如 "Created date" 范围）
2. 导出第一批
3. 更改筛选器
4. 导出下一批
5. 在电子表格中合并文件

**用于分批的示例筛选器：**

* 按日期范围（1 月、2 月、3 月……）
* 按所有者（团队成员 A、团队成员 B……）
* 按状态（活跃、非活跃……）

### 选项 2：使用 API

API 没有记录数量限制：

1. 在 **Settings → Developers** 中获取您的 API 密钥
2. 使用 GraphQL API 查询记录
3. 在您的应用程序中处理结果

参见：[API 文档](/l/zh/developers/extend/capabilities/apis)

## 技巧与最佳实践

### 创建导出视图

保存专为导出配置的视图：

1. 配置列和筛选器
2. 单击 **View options** → **Save as new view**
3. 将其命名为 "Export - [Purpose]"

### 保护您的导出文件安全

导出的文件可能包含敏感数据：

* 存放在安全位置
* 不再需要时删除旧的导出文件
* 共享导出文件时请谨慎

### 导出前检查

<Check>显示了正确的列</Check>
<Check>筛选器设置正确（或为完整导出已移除）</Check>
<Check>您具有导出权限</Check>

## 常见问题

<AccordionGroup>
  <Accordion title="为什么我的导出缺少某些列？">
    仅导出可见列。 在导出之前通过 **Options → Fields** 添加所需的列。
  </Accordion>

  <Accordion title="为什么我的导出没有包含所有记录？">
    检查您的筛选器。 导出仅包含与您当前视图筛选条件匹配的记录。 移除筛选器以导出所有记录。
  </Accordion>

  <Accordion title="我可以导出超过 20,000 条记录吗？">
    单次导出不可以。 使用筛选器分批导出，或使用 API 处理更大的数据集。
  </Accordion>

  <Accordion title="导出文件是什么格式？">
    CSV（逗号分隔值）。 可在 Excel、Google 表格或任何电子表格应用中打开。
  </Accordion>

  <Accordion title="是否包含关系 ID？">
    包含，但仅限于关系的“多”端。 例如，People 的导出包含 `companyId`，但 Companies 的导出不包含人员 ID。
  </Accordion>

  <Accordion title="我可以自动化导出吗？">
    无法直接通过 UI 实现。 使用 API 构建自动化的导出工作流。
  </Accordion>
</AccordionGroup>
