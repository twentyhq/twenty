## 导入前验证

上传文件并完成字段映射后，Twenty 会在导入**之前**验证你的数据。 这样可以在不影响现有数据的情况下捕捉并修复错误。

## 工作原理

1. **上传**您的 CSV 文件
2. **映射**您的列到 Twenty 的字段
3. **查看**黄色高亮的潜在错误
4. 可直接在 UI 中**修复错误**
5. **确认**导入

## 错误显示

存在问题的行会被高亮为**黄色**。 您可以：

* **直接编辑单元格**以修复错误
* **删除该行**以完全跳过该行

这种行内编辑可节省时间—无需返回电子表格、修复错误并重新上传。

## 常见错误类型

### 重复值

**原因**：唯一字段（电子邮件、域名）已在 Twenty 中存在，或在您的文件中出现了两次。

**修复**：

* 在导入 UI 中编辑重复值
* 删除其中一条重复的行

关于如何实施唯一性，详见[唯一性约束](/l/zh/user-guide/data-migration/capabilities/uniqueness-constraints)。

### 格式无效

**原因**：数据与期望的格式不匹配（例如，电子邮件无效、日期格式错误）。

**修复**：编辑单元格，使其符合正确格式。

请参见[字段映射](/l/zh/user-guide/data-migration/capabilities/field-mapping)，了解每种字段类型的期望格式。

### 缺少必填字段

**原因**：必填字段为空。

**修复**：在必填字段中输入值，或删除该行。

### 未找到关系

**原因**：被引用的记录不存在（例如，公司域名尚未导入）。

**修复**：

* 先导入父级记录
* 或更正引用值

有关正确的导入顺序及如何关联记录，参见[导入关系](/l/zh/user-guide/data-migration/capabilities/import-relations)。

## 减少错误的建议

1. **下载模板**，在导入文件前查看期望的格式
2. 先在电子表格中，**清理您的数据**
3. **按正确顺序导入文件**以导入关系（公司 → 联系人 → 商机）
4. **用小批量进行测试**，再进行完整导入
5. 在上传前，**检查重复项**
6. **将您的文件大小限制为 10,000 条记录** 每个文件
