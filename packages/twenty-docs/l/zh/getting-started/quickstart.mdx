## 注册

<Steps>
  <Step title="创建您的帐户">
    访问 [app.twenty.com](https://app.twenty.com) 并使用 Google、Microsoft 或电子邮件注册。
  </Step>
  <Step title="选择试用">
    选择 **30 天**（需要信用卡）或 **7 天**（无需信用卡）。 两种选项都包含完整访问权限——不限联系人数量、电子邮件集成、自定义对象和 API。 您可随时更改套餐或计费周期。
  </Step>
  <Step title="创建您的工作区">
    通过 Stripe 完成付款确认后，您将设置工作区名称和用户个人资料。 您可以随时取消。
  </Step>
</Steps>

## 配置您的工作区

进入后，完成以下三个步骤，让 Twenty 真正成为您的工具：

### 1. 连接您的邮箱

前往 **Settings → Accounts** 并连接您的 Google 或 Microsoft 帐户。 Twenty 将导入您的电子邮件和日历事件，并根据交互自动创建联系人。 使用其他服务提供商？ 您可以在同一页面通过 SMTP 添加邮箱，或通过 CalDAV 添加日历。

<Note>从这里开始——连接邮箱可以在您自定义其他内容之前，通过真实数据立即为团队带来价值。</Note>

<VimeoEmbed videoId="1185416821" title="连接您的邮箱" />

### 2. 塑造您的数据模型

前往 **Settings → Data Model** 创建自定义对象和字段。 有几点需要了解：

* 自定义对象和字段在所有套餐中均**不限量**——无额外加价。
* **人员、公司和机会** 是显示已同步电子邮件和会议的对象。 将它们作为基础使用，并添加字段进行分类（例如，一个 `Person Type` 字段），而不是创建不会包含电子邮件历史记录的单独对象。
* 两个人不能拥有相同的电子邮件地址。 两家公司不能拥有相同的域名。
* 您可以停用不需要的标准字段/对象，并在不删除字段的情况下将其从视图中隐藏。

[数据模型参考 →](/l/zh/user-guide/data-model/overview)

<VimeoEmbed videoId="1185416793" title="塑造您的数据模型" />

### 3. 导入您的数据

使用命令菜单 (`Cmd+K` / `Ctrl+K`) 通过 CSV 导入人员、公司、机会或任何自定义对象。 请先下载示例文件，以查看期望的格式。 将文件限制在最多 10,000 条记录，并在导入前去重电子邮件/域名。

[数据迁移指南 →](/l/zh/user-guide/data-migration/overview)
