Twenty 中的一切都围绕着**对象**和**字段**构建——它们是你的数据模型的基础构件。

## 对象

对象是保存你数据的表。 Twenty 自带标准对象：

* **Companies** — 与你有业务往来的组织
* **People** — 个人联系人
* **Opportunities** — 销售流程中的交易
* **Tasks** — 你的团队需要完成的行动项
* **Notes** — 关联到记录的自由文本

你也可以为业务所需的任何内容创建**自定义对象**——项目、支持工单、产品、合同等。

## 字段

字段是每个对象上的属性。 Twenty 支持广泛的字段类型：

| 类别     | 类型                             |
| ------ | ------------------------------ |
| **基础** | 文本、数字、布尔值、日期、货币、评分、选择          |
| **复合** | 地址（街道、城市、州/省、邮编）、全名、链接、电话、电子邮件 |
| **特殊** | 关系、文件附件、JSON、操作者（创建/修改的人）      |

每个对象还会自动获得系统字段：`id`、`createdAt`、`updatedAt`、`createdBy` 和 `position`。

## 关系

对象通过关系相互连接。 一个公司拥有多个联系人，一个商机从属于一个公司，依此类推。 你可以在任意对象之间创建自定义关系，包括多对多关系。

## 强大之处

与传统 CRM 中受限于预定义对象和预定义字段不同，Twenty 允许你按照业务的实际运行方式来建模你的数据。 自定义对象与内置对象享有同等的一流待遇——包括 API 端点、视图、权限和工作流触发器。
