工作流可让您自动化重复性任务，并将 Twenty 连接到外部工具——无需编写代码（当然如果您愿意也可以编写）。

## 工作流如何运作

每个工作流由三部分组成：

1. **触发器** — 启动工作流的条件
2. **步骤** — 接下来发生的事情（一个或多个按顺序执行的操作）
3. **变量** — 在步骤之间流动的数据

## 触发器

| 触发器         | 触发时机                       |
| ----------- | -------------------------- |
| **记录事件**    | 当记录被创建、更新、删除或执行 upsert 操作时 |
| **手动**      | 当用户点击按钮时（针对单条记录、多条记录或全局）   |
| **计划**      | 按固定时间间隔（cron 语法）           |
| **Webhook** | 外部系统发送一个 HTTP POST         |

## 操作

工作流可以以任意组合串联以下内容：

* **记录操作** — 创建、更新、查找、删除或 upsert 记录
* **发送电子邮件** — 从已连接的账户发送或撰写电子邮件
* **HTTP 请求** — 调用任意外部 API
* **代码** — 运行自定义 JavaScript 以处理复杂逻辑
* **分支** — 使用 if/else 条件拆分工作流路径
* **迭代器** — 遍历数据数组
* **AI Agent** — 让 AI 代理自主处理数据
* **延迟** — 在继续之前等待
* **表单** — 在工作流中途收集用户输入

## 你可以构建的内容

* 当交易达到某个阶段时发送 Slack 警报
* 使用外部 API 的数据自动丰富新联系人
* 检测长期未跟进的机会并通知负责人
* 在 Twenty 与您的计费系统之间同步数据
* 从记录数据生成 PDF 或发票
* 对符合特定条件的入站电子邮件自动回复
