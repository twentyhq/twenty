## Obecné pokyny

**Před zahájením procesu aktualizace vždy zálohujte svou databázi** spuštěním:

```bash
docker exec -it {db_container_name_or_id} pg_dumpall -U {postgres_user} > databases_backup.sql
```

Pro obnovení ze zálohy:

```bash
cat databases_backup.sql | docker exec -i {db_container_name_or_id} psql -U {postgres_user}
```

Pokud používáte Docker Compose, postupujte takto:

1. Zastavte Twenty: `docker compose down`
2. Změňte hodnotu `TAG` v souboru `.env` vedle `docker-compose.yml`
3. Spusťte Twenty: `docker compose up -d`

Server při startu automaticky spustí všechny požadované migrační kroky upgradu. Není potřeba žádný ruční příkaz.

## Upgrade napříč verzemi (v1.22+)

Počínaje **v1.22** podporuje Twenty upgrade napříč verzemi. Můžete přejít přímo z jakékoli podporované verze na nejnovější vydání, aniž byste museli procházet každou meziverzí.

Například přechod z v1.22 přímo na v2.0 je plně podporován.

## Kontrola stavu upgradu

Příkaz `upgrade:status` vám umožní zkontrolovat aktuální stav vaší instance a migrací pracovních prostorů. Je užitečný pro ladění problémů s upgradem nebo při podávání požadavku na podporu.

Spusťte jej z kontejneru serveru:

```bash
docker exec -it {server_container_name_or_id} yarn command:prod upgrade:status
```

### Možnosti

| Přepínač                  | Popis                                                                                |
| ------------------------- | ------------------------------------------------------------------------------------ |
| `-w, --workspace-id <id>` | Omezí se na konkrétní pracovní prostor. Lze zadat vícekrát.                          |
| `-f, --failed-only`       | Skryje aktuální pracovní prostory; zobrazí pouze ty, které jsou pozadu, a neúspěšné. |

## Řešení potíží

Pokud upgrade selže u některých pracovních prostorů, server nepokročí za chybující krok. Restartování serveru (`docker compose up -d`) zkusí upgrade znovu od místa, kde se zastavil.

Pro rychlou identifikaci problémů spusťte:

```bash
docker exec -it {server_container_name_or_id} yarn command:prod upgrade:status --failed-only
```

Tímto se zobrazí pouze pracovní prostory, které jsou pozadu nebo selhaly, spolu s chybovou zprávou pro každé selhání.

## Před v1.22

Pokud je vaše instance starší než v1.22, musíte upgradovat postupně přes každou hlavní označenou verzi (z v1.6 na v1.7, poté z v1.7 na v1.8 atd.), dokud nedosáhnete v1.22. Odtud můžete přejít přímo na nejnovější verzi.
