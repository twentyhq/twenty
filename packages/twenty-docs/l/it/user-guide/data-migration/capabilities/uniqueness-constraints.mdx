## Panoramica

Twenty applica l'unicità su determinati campi per prevenire record duplicati e garantire l'integrità dei dati. Comprendere questi vincoli è essenziale per importazioni riuscite.

## Campi univoci predefiniti

| Oggetto                    | Campi univoci                            |
| -------------------------- | ---------------------------------------- |
| **Persone**                | `id`, `email`                            |
| **Aziende**                | `id`, `domain`                           |
| **Oggetti personalizzati** | Solo `id` (per impostazione predefinita) |

<Note>Il campo `id` è l'identificatore interno di Twenty, generato automaticamente per ciascun record. Utilizza il formato UUID (ad es., `c776ee49-f608-4a77-8cc8-6fe96ae1e43f`).</Note>

## Campi univoci personalizzati

Puoi definire campi univoci aggiuntivi in **Impostazioni → Modello dati**:

1. Vai a **Impostazioni → Modello dati**
2. Seleziona l'oggetto
3. Fai clic su un campo
4. Abilita **Univoco** nelle impostazioni del campo

### Casi d'uso per campi univoci personalizzati

* **ID esterni**: Memorizza ID provenienti da altri sistemi (Salesforce ID, HubSpot ID)
* **Identificativi aziendali**: numeri dei dipendenti, codici cliente
* **Informazioni di contatto alternative**: profilo LinkedIn, numero di telefono

<Note>Il nome del campo `id` è riservato all'ID interno di Twenty. Usa un nome diverso come `externalId` o `legacyId` per gli identificatori esterni.</Note>

## Comportamento dell'importazione

### Creazione di nuovi record

Se il valore di un campo univoco non esiste, viene creato un nuovo record.

### Aggiornamento dei record esistenti

Se il valore di un campo univoco corrisponde a un record esistente, quel record viene **aggiornato** con i nuovi dati.
<Warning>Per **aggiornare i record esistenti**, si consiglia di **associare un solo campo univoco**.</Warning>

### Record eliminati in modo non definitivo

<Warning>
  **I record eliminati contano ai fini dell'unicità.**

  I record eliminati in modo non definitivo (visibili in Menu Comandi → Vedi record eliminati) sono inclusi nei controlli di unicità. Se importi un record con lo stesso valore univoco di un record eliminato, il record eliminato verrà **ripristinato** con i nuovi dati.
</Warning>

## Rilevamento dei duplicati durante l'importazione

Durante la fase di convalida:

* I duplicati presenti nel tuo file sono evidenziati in giallo
* Puoi modificare o rimuovere le righe duplicate dall'interfaccia utente prima di avviare l'importazione

## Migliori Pratiche

1. **Rimuovi i duplicati** dal tuo file prima di importare
2. **Verifica la presenza di record esistenti** in Twenty prima di importare
3. **Usa ID esterni** quando esegui una migrazione da altri sistemi
4. **Includi campi univoci** se vuoi aggiornare i record esistenti
