## Informazioni sulle esecuzioni

Un'**Esecuzione** è una registrazione dell'esecuzione di un flusso di lavoro. Ogni volta che un flusso di lavoro viene attivato — che sia da un evento di record, una pianificazione, un'azione manuale o un webhook — viene creata una nuova esecuzione.

## Visualizzazione delle esecuzioni

### Dall'editor del flusso di lavoro

1. Apri il flusso di lavoro che desideri monitorare
2. Fai clic sul pannello **Esecuzioni** a destra
3. Visualizza un elenco delle esecuzioni recenti con il relativo stato

### Dalla vista Esecuzioni dei flussi di lavoro

1. Vai a **Esecuzioni dei flussi di lavoro** nella barra laterale
2. Visualizza le esecuzioni in tutti i flussi di lavoro
3. Filtra per stato, flusso di lavoro o data

## Stati delle esecuzioni

| Stato             | Descrizione                                                                                       |
| ----------------- | ------------------------------------------------------------------------------------------------- |
| **In esecuzione** | Il flusso di lavoro è in esecuzione                                                               |
| **Completato**    | Il flusso di lavoro è terminato correttamente                                                     |
| **Non riuscito**  | Il flusso di lavoro ha riscontrato un errore ed è stato interrotto                                |
| **In attesa**     | Il flusso di lavoro è in pausa (ad es., in attesa di un'azione Ritardo o dell'invio di un modulo) |

## Dettagli dell'esecuzione

Fai clic su una qualsiasi esecuzione per visualizzare:

* **Stato**: stato attuale dell'esecuzione
* **Ora di avvio**: quando è iniziata l'esecuzione
* **Durata**: quanto è durata l'esecuzione
* **Dati del trigger**: l'input che ha avviato il flusso di lavoro
* **Output dei passaggi**: dati restituiti da ciascun passaggio
* **Messaggi di errore**: se l'esecuzione non è riuscita, cosa è andato storto

## Esecuzione passo dopo passo

Ogni esecuzione mostra l'avanzamento nel flusso di lavoro:

1. Vedi quali passaggi sono stati completati correttamente
2. Identifica dove si sono verificati errori
3. Visualizza i dati scambiati tra i passaggi
4. Esegui il debug dei problemi esaminando gli input e gli output dei passaggi

## Gestione degli errori

Quando un'esecuzione non riesce:

1. Apri l'esecuzione non riuscita
2. Trova il passaggio che ha causato l'errore
3. Controlla il messaggio di errore per i dettagli
4. Problemi comuni:
   * Campi obbligatori mancanti
   * Formato dati non valido
   * Errori delle API esterne
   * Problemi di autorizzazioni

## Eseguire nuovamente i flussi di lavoro

Se un'esecuzione non riesce, puoi:

* Correggi il problema alla base e attendi il prossimo trigger
* Per i flussi di lavoro manuali, attivalo di nuovo con gli stessi dati o con dati aggiornati
* Rivedi la logica del flusso di lavoro per prevenire errori futuri

## Suggerimenti sulle prestazioni

### Gestione della cronologia delle esecuzioni

* Le esecuzioni vengono conservate per riferimento storico
* Le esecuzioni molto vecchie possono essere archiviate automaticamente
* Esporta i dati delle esecuzioni se devi conservare i registri

### Pratiche consigliate per il monitoraggio

* Controlla regolarmente le esecuzioni dopo aver attivato nuovi flussi di lavoro
* Esamina le esecuzioni non riuscite per identificare schemi ricorrenti
