## Debug con esecuzioni di flussi di lavoro

Utilizza l'interfaccia **Esecuzioni di flussi di lavoro** per eseguire il debug dei problemi:

- Accedi tramite il pannello **Esecuzioni** nell'editor del flusso di lavoro
- Clicca sulle singole esecuzioni per vedere i dati di input/output per ogni passaggio
- Controlla lo stato dell'esecuzione, i messaggi di errore e il flusso di dati tra i passaggi

## Problemi comuni e soluzioni

### Flusso di lavoro non attivato

**Problema**: Il flusso di lavoro non viene eseguito quando previsto.

**Soluzioni**:

- Verifica che il flusso di lavoro sia **Attivo** (non in modalità Bozza)
- Controlla che la configurazione del trigger corrisponda alla tua struttura dati
- Per i trigger di Record, assicurati che l'oggetto e i campi corretti siano selezionati
- Per i trigger Webhook, verifica l'URL e il formato dei dati atteso
- Per i trigger Pianificati, controlla la configurazione dell'orario

### Moduli non accessibili

**Problema**: Le azioni del modulo sono difficili da trovare o accedere nei flussi di lavoro automatizzati.

<Warning>
I moduli sono attualmente progettati solo per i trigger manuali. Per i flussi di lavoro con altri trigger (Record Creato, Aggiornato, ecc.), i moduli sono accessibili solo tramite l'interfaccia di esecuzione del flusso di lavoro, il che non è l'esperienza utente prevista. Un centro notifiche sarà rilasciato nel 2026 per supportare correttamente i moduli nei flussi di lavoro automatizzati.
</Warning>

**Soluzione alternativa**: Usa trigger manuali quando sono richiesti moduli, o ristruttura i flussi di lavoro per evitare moduli nei flussi automatizzati.

### Elevato consumo di crediti

**Problema**: I flussi di lavoro consumano più crediti del previsto.

**Cause comuni e soluzioni**:

- **Chiamate API inefficienti**: Raggruppa le chiamate API quando possibile anziché richieste singole
- **Configurazione errata del trigger manuale**: Utilizza disponibilità `Bulk` anziché `Singolo` per elaborare più record in una esecuzione del flusso di lavoro
- **Filtri mancanti**: Aggiungi logica condizionale per interrompere i flussi di lavoro quando i criteri non sono soddisfatti
- **Passaggi non necessari**: Rimuovi azioni ridondanti e ottimizza la logica del flusso di lavoro
- **Tempo reale vs. Pianificato**: Usa flussi di lavoro pianificati per processi non urgenti

### Limiti di flussi di lavoro simultanei

**Problema**: È stato raggiunto il limite di 100 flussi di lavoro simultanei per spazio di lavoro.

<Warning>
You cannot run more than 100 workflows in parallel at any given time per workspace.
</Warning>

**Soluzioni**:

- Utilizza disponibilità `Bulk` per i trigger manuali per elaborare più record in un'esecuzione
- Implementa ritardi tra le esecuzioni dei flussi di lavoro utilizzando i trigger pianificati
- Ottimizza i flussi di lavoro per eseguire più velocemente e ridurre il tempo di esecuzione simultaneo
- Considera l'elaborazione batch durante le ore non di punta

### Limitazione del rate delle API

**Problema**: Le chiamate API esterne non riescono a causa dei limiti di rate.

**Soluzioni**:

- Utilizza flussi di lavoro pianificati invece di trigger in tempo reale quando possibile
- Implementa ritardi tra le chiamate API nelle azioni di codice
- Raggruppa le richieste API quando il servizio esterno lo supporta
- Monitora le esecuzioni del flusso di lavoro per errori di limite di rate e modifica i tempi

### Problemi con l'iteratore

**Problema**: Le azioni dell'iteratore non funzionano come previsto.

**Soluzioni**:

- **Nota**: L'iteratore è attualmente in beta. Attivalo in Impostazioni → Rilasci → Lab
- Verifica che l'input sia un array di record da un passaggio precedente
- Controlla che le azioni all'interno dell'iteratore siano configurate correttamente
- Utilizza l'iteratore con trigger manuali `Bulk` per prestazioni ottimali

### Permessi mancanti

**Problema**: Impossibile accedere alla sezione dei flussi di lavoro.

<Warning>
Se non vedi la sezione dei flussi di lavoro, ciò è dovuto a un problema di permessi. Contatta l'amministratore dello spazio di lavoro per concederti l'accesso ai flussi di lavoro.
</Warning>

## Suggerimenti per l'ottimizzazione

### Pratiche migliori per le prestazioni

- **Inizia semplice** e aggiungi complessità gradualmente
- **Usa disponibilità `Bulk`** per i trigger manuali per elaborare più record in modo efficiente
- **Aggiungi filtri presto** per interrompere i flussi di lavoro quando i criteri non sono soddisfatti
- **Raggruppa le chiamate API** e usa flussi di lavoro pianificati per processi non urgenti
- **Nascondi le esecuzioni di flussi di lavoro** dalle pagine che mostrano tutti i flussi di lavoro per migliorare le prestazioni di caricamento
- **Monitora il consumo di crediti** regolarmente in Impostazioni → Fatturazione

### Prevenzione degli errori

- **Testa in modalità bozza** prima di attivare i flussi di lavoro
- **Convalida le risposte API** e implementa azioni di fallback
- **Usa nomi di passaggi descrittivi** per una manutenzione più facile
- **Documenta logica complessa** per i membri del team
