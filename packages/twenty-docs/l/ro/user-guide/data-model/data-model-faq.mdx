## Gestionarea obiectelor

<AccordionGroup>
<Accordion title="Can I reorder objects in the left navigation bar?">
Încă nu. Ordinea obiectelor în navigare este momentan fixă, însă această funcționalitate este planificată pentru o versiune viitoare.
</Accordion>

<Accordion title="Can I hide objects from the left navigation bar?">
Toate obiectele active apar în navigare. Poți dezactiva obiectele de care nu ai nevoie sub **Setări → Model de Date**.
</Accordion>

<Accordion title="Can I delete standard objects (People, Companies, etc.)?">
You can deactivate any standard objects but you cannot hard delete them.
</Accordion>
</AccordionGroup>

## Capabilitățile câmpurilor

<AccordionGroup>
<Accordion title="Can I create formula fields?">
Câmpurile cu formule vor fi disponibile în **T1 2026**. Între timp, poți folosi fluxuri de lucru pentru a calcula și actualiza automat valorile câmpurilor.
</Accordion>

<Accordion title="Can I have nested fields in my objects?">
Câmpurile imbricate vor fi disponibile în **T1 2026**. În prezent, poți folosi fluxuri de lucru pentru a prelua valorile câmpurilor din obiectele înrudite. De exemplu, pentru a afișa industria unei companii pe un document despre o Persoană, creează un câmp personalizat la Persoane și folosește un flux de lucru pentru a sincroniza valoarea.
</Accordion>

<Accordion title="Why can't I update relation field names?">
Numele câmpurilor de relație afectează structura API și nu pot fi schimbate după crearea lor. Dacă trebuie să redenumești un câmp de relație, va trebui să creezi unul nou și să-l ștergi pe cel vechi.
</Accordion>

<Accordion title="Why do I need different singular and plural names?">
Our GraphQL API uses both forms for different operations:
- `createPerson` (singular) for single record actions
- `createPeople` (plural) for bulk operations

Aceasta creează limitări atunci când formele singular și plural sunt aceleași, dar îmbunătățește experiența dezvoltatorului. </Accordion>

<Accordion title="Why are some field names protected?">
Anumite nume de câmpuri, precum `Type` sau `Application`, sunt rezervate pentru utilizarea sistemului. Alege nume alternative precum `Category` sau `Classification`.
</Accordion>
</AccordionGroup>

## Caracteristici avansate

<AccordionGroup>
<Accordion title="Can I create many-to-many relationships?">
Relațiile de tip mulți-la-mulți vor fi disponibile în **T1 2026**. În prezent, creează un obiect intermediar cu două relații de tip 1-la-mulți ca soluție temporară.
</Accordion>

<Accordion title="What are Morph Many relationships?">
Relațiile de tip Morph Many permit unui obiect să se relateze la mai multe tipuri diferite de obiecte. De exemplu, o Oportunitate ar putea să se relaționeze fie la o Persoană, fie la o Companie. Această funcționalitate este acum disponibilă.
</Accordion>

<Accordion title="Can I reorder fields in objects?">
Rearanjarea câmpurilor va fi disponibilă cu machete personalizate în **T4 2025**. În prezent, câmpurile apar în ordine alfabetică.
</Accordion>
</AccordionGroup>

## Acces și permisiuni

<AccordionGroup>
<Accordion title="Where can I see and edit my data model?">
You can access your Data Model under **Settings → Data Model**.
</Accordion>

<Accordion title="Why can't I see the Data Model under Settings?">
Contactează-ți administratorul de spațiu de lucru. Accesul la modelul de date este de obicei restricționat doar la administratori.
</Accordion>

<Accordion title="How many custom objects or fields can I create?">
You can create as many custom objects and fields as you need - the price won't change.
</Accordion>
</AccordionGroup>
