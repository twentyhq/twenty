## 객체 관리

<AccordionGroup>
  <Accordion title="사용자 정의 객체를 삭제할 수 있나요?">
    예, 사용자 정의 객체는 삭제할 수 있습니다. 먼저 비활성화할 수도 있습니다. 이렇게 하면 데이터는 보존하면서 객체와 그 데이터를 인터페이스에서 숨깁니다.
  </Accordion>

  <Accordion title="표준 객체(사람, 회사 등)를 삭제할 수 있습니까?">
    아니요, 표준 객체는 삭제할 수 없습니다. 비활성화만 할 수 있으며, 인터페이스에서는 숨겨지지만 데이터는 보존됩니다.
  </Accordion>

  <Accordion title="사용자 정의 객체는 몇 개까지 만들 수 있나요?">
    필요한 만큼 사용자 정의 객체와 필드를 만들 수 있으며, 가격은 변동하지 않습니다.
  </Accordion>

  <Accordion title="표준 객체의 이름을 바꿀 수 있나요?">
    표준 객체(사람, 회사, 기회)의 레이블은 변경할 수 있지만, API 이름은 변경할 수 없습니다. 모든 Twenty 워크스페이스에서의 일관성을 위해 API 이름은 고정되어 있습니다.
  </Accordion>

  <Accordion title="객체의 아이콘을 변경할 수 있나요?">
    예, **설정 → 데이터 모델**에서 표준 객체와 사용자 정의 객체 모두의 아이콘을 변경할 수 있습니다.
  </Accordion>

  <Accordion title="왼쪽 탐색 모음에서 객체 순서를 변경할 수 있습니까?">
    아직 안 됩니다. 탐색의 객체 순서는 현재 고정되어 있으며, 이 기능은 향후 릴리스에 계획되어 있습니다.
  </Accordion>

  <Accordion title="왼쪽 탐색 모음에서 객체를 숨길 수 있습니까?">
    모든 활성 객체는 탐색에 나타납니다. **설정 → 데이터 모델**에서 필요 없는 객체를 비활성화할 수 있습니다.
  </Accordion>
</AccordionGroup>

## 필드 기능

<AccordionGroup>
  <Accordion title="생성 후에 필드 유형을 변경할 수 있나요?">
    아니요, 생성 후에는 필드 유형을 변경할 수 없습니다. 다른 유형이 필요하다면 올바른 유형의 새 필드를 만들고 데이터를 마이그레이션한 다음 이전 필드를 비활성화하세요.
  </Accordion>

  <Accordion title="단수 및 복수 이름이 필요한 이유는 무엇입니까?">
    우리의 GraphQL API는 다양한 작업에서 두 형식을 모두 사용합니다:

    * 단일 기록 작업을 위한 `createPerson` (단수)
    * 대량 작업을 위한 `createPeople` (복수)

    단수 및 복수 형식이 동일할 때 제한이 있지만, 개발자 경험을 향상시킵니다.
  </Accordion>

  <Accordion title="왜 일부 필드 이름은 보호됩니까?">
    `Type` 또는 `Application`과 같은 특정 필드 이름은 시스템 사용을 위해 예약되어 있습니다. 대신 `Category` 또는 `Classification`과 같은 대체 이름을 선택하세요.
  </Accordion>

  <Accordion title="필드를 비활성화하면 어떻게 되나요?">
    * 필드는 인터페이스에서 숨겨집니다
    * 기존 데이터는 보존됩니다
    * API를 통해 해당 필드에 계속 액세스할 수 있습니다
    * 기존 관계는 유지되지만 새 관계를 생성할 수는 없습니다
    * 필드는 나중에 다시 활성화할 수 있습니다
  </Accordion>

  <Accordion title="필드를 필수로 설정할 수 있나요?">
    현재는 사용자 정의 필드를 필수로 설정할 수 없습니다. 모든 필드는 빈 값을 허용합니다. 필드가 비어 있을 때 알림을 보내거나 동작을 차단하는 워크플로우를 사용해 필수 필드를 강제할 수 있습니다.
  </Accordion>

  <Accordion title="고유와 필수의 차이는 무엇인가요?">
    * **고유**: 두 레코드가 이 필드에 동일한 값을 가질 수 없습니다
    * **필수**: 필드에 값이 있어야 합니다(현재 사용자 정의 필드에서는 지원되지 않음)
  </Accordion>

  <Accordion title="수식 필드를 만들 수 있습니까?">
    수식 필드는 **2026년 1분기**에 제공될 예정입니다. 현재는 워크플로를 사용하여 필드 값을 자동으로 계산하고 업데이트할 수 있습니다.
  </Accordion>

  <Accordion title="내 객체에 중첩 필드를 가질 수 있습니까?">
    중첩 필드는 **2026년 1분기**에 제공될 예정입니다. 현재로서는 관련 객체의 필드 값을 가져오려면 워크플로를 사용할 수 있습니다. 예를 들어, 회사의 산업을 사람 기록에 표시하려면 사람에 대한 사용자 정의 필드를 만들고 워크플로를 사용하여 값을 동기화하세요.
  </Accordion>

  <Accordion title="객체 내에서 필드를 정렬할 수 있습니까?">
    필드 정렬은 **2025년 4분기**에 사용자 정의 레이아웃과 함께 제공될 예정입니다. 현재 필드는 알파벳 순으로 표시됩니다.
  </Accordion>
</AccordionGroup>

## 관계

<AccordionGroup>
  <Accordion title="같은 객체를 가리키는 관계를 만들 수 있나요?">
    예! 자기 참조 관계가 지원되며, 계정 계층 구조와 같은 사용 사례에 권장됩니다. 예를 들어, 상위/하위 계정을 추적하기 위해 회사에서 회사로의 관계를 생성하세요.
  </Accordion>

  <Accordion title="다대다 관계를 만들 수 있습니까?">
    다대다 관계는 **2026년 상반기**에 제공될 예정입니다. 현재는 우회 방법으로 두 개의 1대다 관계를 가진 중간 객체를 만드세요.

    예를 들어 People과 Projects를 연결(다대다)하려면 "Project Assignments" 객체를 만들고 다음을 포함하세요:

    * People에 대한 관계(여러 할당 → 한 사람)
    * Projects에 대한 관계(여러 할당 → 한 프로젝트)
  </Accordion>

  <Accordion title="여러 객체 유형에 대한 관계(Morph 관계)란 무엇인가요?">
    이는 하나의 필드를 통해 하나의 객체가 여러 서로 다른 객체 유형과 관계를 맺을 수 있도록 합니다. 예를 들어, 노트는 동시에 사람, 회사, 기회에 첨부될 수 있습니다.

    각 노트는 동시에 한 사람, 한 회사, 한 기회에 연결됩니다.

    자세한 내용은 [관계 필드](/l/ko/user-guide/data-model/capabilities/relation-fields)에서 확인하세요.
  </Accordion>

  <Accordion title="동일한 두 객체 간에 여러 관계를 만들 수 있나요?">
    예, 동일한 두 객체 간에 여러 관계를 생성할 수 있습니다. 예를 들어, 회사는 사람에 대해 "Primary Contact"와 "Billing Contact" 관계를 모두 가질 수 있습니다.
  </Accordion>

  <Accordion title="레코드를 삭제하면 관계는 어떻게 되나요?">
    레코드를 삭제하면 관련 레코드에서 관계 링크가 제거됩니다. 관련 레코드 자체는 삭제되지 않습니다.
  </Accordion>

  <Accordion title="순환 관계를 만들 수 있나요?">
    기술적으로는 가능하지만, 순환 관계(A → B → C → A)는 혼란을 초래하고 성능 문제를 일으킬 수 있으므로 피하는 것이 좋습니다.
  </Accordion>
</AccordionGroup>

## 접속 및 권한

<AccordionGroup>
  <Accordion title="내 데이터 모델을 어디서 보고 편집할 수 있습니까?">
    **설정 → 데이터 모델**로 이동하여 모든 객체와 필드를 보고 편집하세요.
  </Accordion>

  <Accordion title="왜 설정에서 데이터 모델을 볼 수 없습니까?">
    워크스페이스 관리자에게 문의하세요. 데이터 모델 접근은 보통 관리자에게만 제한됩니다.
  </Accordion>
</AccordionGroup>

## 데이터 관리

<AccordionGroup>
  <Accordion title="보유할 수 있는 레코드 수에 제한이 있나요?">
    레코드 수에 대한 하드 제한은 없습니다. 하지만 매우 큰 데이터 세트는 일부 보기에서 성능에 영향을 줄 수 있습니다. 필터와 보기를 사용해 대규모 데이터 세트를 효과적으로 관리하세요.
  </Accordion>

  <Accordion title="사용자 정의 객체로 데이터를 가져올 수 있나요?">
    예, 사용자 정의 객체를 포함한 모든 객체에 CSV 데이터를 가져올 수 있습니다. 가져오기 프로세스는 사용자 정의 필드에 대한 필드 매핑을 지원합니다. 자세한 내용은 [CSV 파일 준비 방법](/l/ko/user-guide/data-migration/how-tos/prepare-your-csv-files)을 참조하세요.
  </Accordion>

  <Accordion title="내 데이터 모델 구성 정보를 내보낼 수 있나요?">
    현재 데이터 모델 구성에 대한 내장된 내보내기 기능은 없습니다. 워크스페이스 간에 데이터 모델을 마이그레이션해야 하는 경우 지원팀에 문의하세요.
  </Accordion>
</AccordionGroup>

## 추가 도움이 필요하십니까?

복잡한 데이터 모델 설계를 지원받으려면 [구현 서비스](/l/ko/user-guide/getting-started/capabilities/implementation-services)를 확인하세요.
