## 데이터 모델이란 무엇입니까?

데이터 모델은 CRM에서 정보가 조직되는 방식을 정의하는 구조입니다. 고객 데이터의 **청사진**이라고 생각하세요 — 한 번 설계한 다음 실제 데이터로 채웁니다.

## 핵심 개념

### 개체

**개체**는 CRM에서 데이터의 주요 범주입니다. 각 개체는 추적하려는 항목의 유형을 나타냅니다.

Twenty에는 표준 개체가 기본으로 제공됩니다:

* **People** — 개인(연락처, 리드, 파트너)
* **Companies** — 조직
* **Opportunities** — 거래 또는 판매
* **Notes** — 기록에 첨부된 노트
* **Tasks** — 기록에 연결된 할 일

비즈니스에 특화된 항목에 대해 **사용자 지정 개체**를 생성할 수도 있습니다(예: Projects, Subscriptions, Events).

### 필드

**필드**는 각 개체를 설명하는 속성 또는 특성입니다. 실제 정보를 저장합니다.

예를 들어, **People** 개체에는 다음과 같은 필드가 있습니다:

* 이름
* 이메일
* 전화
* 직책
* Company (Companies 개체에 대한 관계)

필드에는 다양한 **유형**이 있습니다: 텍스트, 숫자, 날짜, 선택, 다중 선택, 관계 등. 어떤 개체에도 사용자 지정 필드를 추가할 수 있습니다.

### 기록

**기록**은 개체 내의 개별 항목으로, 사용자가 생성하고 관리하는 실제 데이터입니다.

예를 들어:

* "John Smith"는 People 개체의 **기록**입니다
* "Acme Corp"는 Companies 개체의 **기록**입니다

**비유:**

| 데이터 모델 개념 | 현실 세계 비유         |
| --------- | ---------------- |
| **개체**    | 책의 섹션(범주)        |
| **필드**    | 스프레드시트의 열(속성)    |
| **기록**    | 스프레드시트의 행(실제 항목) |

데이터 모델(개체 + 필드)을 한 번 설계한 후, 그 구조 내에서 많은 기록을 생성합니다.

## 왜 데이터 모델을 사용자 정의하나요?

모든 비즈니스는 다르게 작동합니다. 데이터 모델을 사용자 정의하면 경직된 시스템에 프로세스를 억지로 맞추는 대신, **귀하의** 프로세스에 맞춰 Twenty를 구성할 수 있습니다.

Twenty는 완전한 유연성을 제공합니다:

* 필요한 만큼 사용자 지정 개체를 생성하세요
* 무제한의 사용자 지정 필드를 추가하세요
* 사용자 정의 여부와 상관없이 가격은 변하지 않습니다

## 데이터 모델 설계 팁

### 1. 핵심 개체부터 시작하세요

업무에 사용하는 핵심 개념을 식별하세요. Twenty에서 이미 제공하는 항목:

* **People** — 귀하의 연락처
* **Companies** — 귀하의 조직
* **Opportunities** — 귀하의 거래

추가로 필요한 것이 무엇인지 생각해 보세요:

* Stripe에는 `Subscriptions` 개체가 필요합니다
* Airbnb에는 `Trips` 개체가 필요합니다
* 엑셀러레이터에는 `Batches` 개체가 필요합니다

### 2. 새 개체 대신 변형에는 필드를 사용하세요

어떤 것이 기존 개체의 특성에 불과하다면 **필드**로 만드세요.

**다음의 경우 필드를 사용하세요:**

* 카테고리와 레이블(예: Companies의 `Industry`)
* 상태 값(예: Opportunities의 `Stage`)
* 속성과 특성

### 3. 독립적으로 존재한다면 개체를 만드세요

그 개념에 자체 수명주기, 속성 또는 관계가 있다면 개체로 만들어야 합니다.

**다음의 경우 개체를 만드세요:**

* **Projects** — 기한, 소유자 및 작업이 있음
* **Subscriptions** — 회사, 제품 및 인보이스를 연결함
* **Events** — 참가자와 후속 조치를 포함함

이들은 자신의 데이터와 관계를 가지고 있기 때문에 단일 필드를 초과합니다.

### 4. 기록의 개수가 정해져 있지 않은 경우 개체를 만드세요

어떤 항목이 여러 번 연결될 수 있고 몇 개인지 알 수 없다면, 개체를 사용하세요.

**나쁜 접근법:**
`Product 1`, `Product 2`, `Product 3` 같은 필드 만들기...

**좋은 접근법:**
`Products` 개체를 만들고 레코드와 연결하세요. 이렇게 하면 모델을 변경하지 않고도 하나, 두 개, 또는 백 개의 제품을 지원할 수 있습니다.

### 5. 먼저 단순하게 유지하세요

필드부터 시작하세요. 한계를 느낄 때에만 새 객체로 전환하세요:

* 하나의 객체에 필드가 너무 많음
* 분리되어야 할 레코드가 반복됨
* 모델에 잘 맞지 않는 관계

## 사람, 회사 및 기회에 대한 특별 참고 사항

<Warning>
  **이메일 및 캘린더 동기화는 사람, 회사 및 기회에서만 작동합니다.**

  메일함/캘린더에서 동기화된 이메일과 미팅에 액세스할 수 있는 유일한 객체입니다. 가능한 한 많이 사용할 것을 권장합니다.
</Warning>

**모범 사례:**

* 사람 카테고리가 필요하다면 새 객체가 아닌 필드를 사용하세요
* 예: 별도의 객체를 만드는 대신 값이 "Prospect"와 "Partner"인 `Person Type` 필드를 사용하세요
* 필터링을 위해 서로 다른 **뷰**를 만드세요: 하나는 파트너를, 다른 하나는 잠재 고객을 표시합니다

**모든 레코드에 적용되지 않는 필드가 있어도 괜찮습니다.** 예: `Person Type = Partner`일 때만 적용되는 사람 객체의 `Referral Link` 필드. 관련 없는 뷰에서는 이 필드를 숨기세요.

## 데이터 모델에 액세스하기

1. 왼쪽 사이드바에서 **설정**으로 이동하세요
2. **데이터 모델**을 클릭하세요
3. 모든 객체(표준 및 사용자 지정)를 확인하세요
4. 임의의 객체를 클릭하여 해당 필드를 보고 편집하세요

<Note>
  **설정에서 데이터 모델이 보이지 않나요?**

  데이터 모델에 대한 액세스는 보통 관리자에게만 제한됩니다. 액세스가 필요하면 워크스페이스 관리자에게 문의하세요.
</Note>
