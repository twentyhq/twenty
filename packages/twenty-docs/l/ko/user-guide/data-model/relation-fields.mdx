## 관계 필드는 무엇입니까?

관계 필드는 한 객체의 기록을 다른 객체의 기록과 연결합니다. 예를 들어:

- **사람** → **회사** (각 사람은 한 회사에서 일합니다)
- **기회** → **사람** (각 거래에는 연락 담당자가 있습니다)
- **작업** → **기회** (각 작업은 특정 거래와 관련이 있습니다)

## 관계 필드 만들기

### 1. 필드 추가

**설정 → 데이터 모델 → [귀하의 객체]** 로 이동하여 **필드 추가**를 클릭하세요.

### 2. 관계 유형 선택

- **필드 유형**: "관계"를 선택하세요
- **대상 객체**: 연결할 객체를 선택하세요
- **관계**: 현재는 **1 대 다** 관계만 지원됩니다. 올바른 방향으로 관계를 설정하십시오.

### 3. 필드 이름 구성

관계 양쪽의 이름을 설정해야 합니다:

- **소스 필드 이름**: 현재 객체에 필드가 어떻게 나타나는지
- **대상 필드 이름**: 대상 객체에서 역방향 필드가 어떻게 나타나는지

<Warning>
필드 이름은 관계가 저장된 후에는 편집할 수 없으므로 API 구조에 영향을 미칩니다. 신중하게 선택하십시오.
</Warning>

## 팀 구성원과 관련된 방법

**작업 공간 구성원** (Twenty 팀 사용자)을 포함하여 모든 객체와의 관계를 생성할 수 있습니다. 이것은 소유권 필드를 생성하는 데 유용합니다:

- **계정 소유자**: 회사를 관리하는 팀 구성원과 회사를 연결합니다
- **거래 소유자**: 특정 영업 사원에게 기회를 할당합니다

`작업 공간 구성원`에 대한 관계를 생성하면, 드롭다운 선택에서 팀 구성원의 이름이 표시되어 소유권 및 책임을 쉽게 할당할 수 있습니다.

## 모범 사례

- 관계를 생성하기 전에 **계획을 세우십시오**
- **명확하고 설명적인 이름을 사용**하여 필드 이름을 지정하십시오
- 전체 구현 전에 샘플 데이터로 **관계를 테스트하십시오**
