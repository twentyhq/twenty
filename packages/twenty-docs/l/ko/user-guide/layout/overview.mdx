Twenty의 레이아웃은 세 가지 수준에서 사용자 지정할 수 있습니다: 앱을 탐색하는 방식, 레코드 목록을 살펴보는 방식, 개별 레코드를 열 때 보이는 내용.

## 네비게이션

왼쪽 사이드바는 완전히 사용자 지정할 수 있습니다. 다음을 수행할 수 있습니다:

* **항목 재정렬** — 끌어서 놓기하여 순서를 바꿉니다
* **폴더 만들기** — 관련 개체와 보기를 그룹화합니다
* **개체 숨기기** — 사용하지 않는 개체를 숨깁니다
* **사용자 지정 링크 추가** — 외부 도구에 대한 링크를 추가합니다
* **즐겨찾기 고정** — 보기, 레코드 또는 검색에 빠르게 액세스할 수 있도록 고정합니다

[네비게이션 참조 →](/l/ko/user-guide/layout/capabilities/navigation)

## 보기

보기는 레코드 목록의 표시 방식을 제어합니다. Twenty는 세 가지 유형의 보기를 지원합니다:

| 보기      | 적합한 용도                               |
| ------- | ------------------------------------ |
| **테이블** | 여러 레코드를 한 번에 다룰 때 — 스프레드시트 스타일의 행과 열 |
| **칸반**  | 파이프라인 추적 — 단계별로 구성된 카드를 끌어서 놓으며 관리   |
| **캘린더** | 시간 기반 계획 — 날짜 필드를 기준으로 레코드를 배치       |

각 보기는 자체 필터, 정렬, 필드 가시성, 그룹화 구성을 저장합니다. 보기는 워크스페이스와 공유하거나 비공개로 유지할 수 있습니다.

[테이블 보기 →](/l/ko/user-guide/views-pipelines/capabilities/table-views) · [칸반 보기 →](/l/ko/user-guide/views-pipelines/capabilities/kanban-views) · [캘린더 보기 →](/l/ko/user-guide/views-pipelines/capabilities/calendar-view)

## 레코드 페이지

레코드를 열면 상세 페이지는 구성 가능한 탭과 위젯으로 구성됩니다. 다음을 수행할 수 있습니다:

* **탭 추가, 제거 및 재정렬** — 모든 레코드 유형에서 수행할 수 있습니다
* **위젯 구성** — 필드, 관련 레코드, 이메일, 타임라인, 캘린더, 작업, 노트, 파일, 차트, iframes 등
* **위젯 끌어 이동 및 크기 조절** — 그리드 레이아웃에서 수행할 수 있습니다
* **위젯별 필드 가시성 제어**

명령 메뉴(`Cmd+K` → "레코드 페이지 레이아웃 편집")에서 레이아웃 사용자 지정 모드로 진입하세요.

[레코드 페이지 참조 →](/l/ko/user-guide/layout/capabilities/record-pages)
