## 워크플로우의 중요성

Twenty는 사용자가 최대한 유연성을 누릴 수 있도록 설계되었습니다. 정해진 기능에 비즈니스 프로세스를 맞추는 대신, 워크플로우를 통해 귀사만의 고유한 비즈니스 사례를 지원할 수 있는 CRM 자동화를 빌드할 수 있습니다.

워크플로우는 이러한 자동화를 구축하기 위한 Twenty의 앱 내 기능입니다. 필요할 때 필요한 것을 정확히 만들 수 있는 빌딩 블록을 제공합니다.

## 워크플로우로 무엇을 할 수 있나요?

두 가지 주요 목적으로 자동화를 구축하는 것을 권장합니다:

1. **팀의 일상적인 작업을 촉진하기 위한 내부 자동화**: 수작업 입력과 반복적인 작업을 줄여 팀의 속도를 높입니다.
2. **데이터를 받아오고 Twenty에서 내보내기**: API 호출 및 웹훅을 통해 데이터베이스 및 다른 도구와 Twenty를 연결합니다.

방법에 대한 세부 사항을 탐구하기 전에 가능한 예를 살펴보겠습니다.

## 1. 내부 자동화

데이터가 최신 상태여야 CRM이 유용합니다. 그러나 아무도 데이터 업데이트를 선호하지 않습니다. 워크플로우를 사용하여 저가치, 반복 가능한 작업을 자동화하고 팀이 중요한 작업에 집중할 수 있도록 합니다.

내부에서 자동화할 수 있는 예:

- **데이터 관리**: 개인 이메일 자동 플래그 지정, 리드 할당, 데이터 검증
- **판매 프로세스**: 단계별 업데이트, 이탈 관리, 기회 상실 알림
- **생산성 향상**: 주간 작업 알림, 회의 후속 조치, 교차 객체 필드 동기화

자세한 예시와 단계별 가이드는 [내부 자동화](/l/ko/user-guide/workflows/internal-automations) 가이드를 참조하세요.

## 2. 외부 통합

Twenty를 다른 도구 및 데이터 소스와 연결하여 통합된 비즈니스 시스템을 생성합니다. 웹 양식, 제품 데이터베이스, 통화 기록기, 데이터 강화 서비스에서 Twenty로 데이터를 가져올 수 있습니다. 또한 이메일 마케팅 도구, 청구 시스템 및 기타 비즈니스 애플리케이션으로 데이터를 내보낼 수 있습니다.

통합할 수 있는 예시들:

- **데이터 수집**: 웹 폼 제출, 제품 데이터 동기화, 회의 기록, 데이터 보강
- **데이터 배포**: 뉴스레터 관리, 이메일 시퀀스, 리드 점수, 청구서 생성

상세한 통합 패턴과 구현 지침은 [외부 도구 통합](/l/ko/user-guide/workflows/external-tool-integration) 가이드를 참조하십시오.

### 이 연결을 구축하고 싶지 않다면?

자동화를 위한 [전문 서비스](/l/ko/user-guide/workflows/professional-services)를 제공합니다.  
프로젝트 범위에 따라, 우리는 [온보딩 팩](https://twenty.com/onboarding-packages)을 제안하거나 인증된 구현 파트너와 연결시켜 드립니다. 데이터 모델 생성, 데이터 마이그레이션, 워크플로우 구축을 도울 수 있습니다.

### 필요한 연결이 가능한지 확신이 안 듭니다.

메시지를 보내주시면, 실현 가능성을 평가하는 데 도움을 드리겠습니다.

## 워크플로우 최적화 모범 사례

워크플로우를 구축하는 동안, 다음 팁을 염두에 두십시오:

- **단계 이름 편집**: 각 단계가 무엇을 하는지 명확히 설명하도록 워크플로우 단계의 이름을 바꾸십시오. 이 방법은 유지 관리에 도움이 되고 동료에게 인계하기 쉽게 만듭니다.
- **이전 단계 데이터 활용**: 워크플로우의 모든 이전 단계에서 반환된 레코드의 필드를 사용할 수 있습니다.
- **간단하게 시작**: 기본 워크플로우로 시작하고 시스템에 익숙해지면서 점차 복잡도를 추가하십시오.
- **구축 전 계획**: 워크플로우 논리를 사전에 계획하여 구축 도중 막히는 것을 피하십시오.
- **분기 현명하게 사용**: `레코드 검색` 단계 후에는 '기존 레코드 업데이트' 및 '새로운 레코드 생성' 시나리오를 처리할 분기를 만들고, 경로를 다시 통합하십시오.

## 우리가 작업 중인 내용

워크플로우 개선을 위해 작업 중인 주요 항목들:

- 루프 (Iterator는 현재 베타 버전으로 설정 → 릴리스 → 실험실에서 활성화하세요)
- If / Else 단계
- AI 에이전트가 대신 워크플로우를 구축합니다.
