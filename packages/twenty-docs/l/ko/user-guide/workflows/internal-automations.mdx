반복적인 작업을 자동화하여 수작업을 줄이고 데이터를 최신 상태로 유지할 수 있는 워크플로우 예제는 다음과 같습니다.

## 데이터 관리 자동화

### 개인 이메일 자동 플래그

**문제**: 팀은 더 나은 타겟팅과 규정을 준수하기 위해 개인 및 비즈니스 이메일 주소를 식별해야 합니다.

**해결 방법**: 이메일 주소가 추가되거나 업데이트될 때 "개인 이메일 여부" 필드를 자동으로 업데이트하는 워크플로우를 생성하십시오.

**트리거**: 레코드가 업데이트됨 (People 객체, Email 필드)

**작업**:

- 이메일 도메인이 일반적인 개인 제공업체 (gmail.com, yahoo.com 등)와 일치하는지 확인하는 코드 작업
- "개인 이메일 여부" 플래그를 설정하기 위해 레코드 업데이트

### 리드 할당 - 라운드 로빈

**문제**: 새로운 리드를 공평하게 영업 팀 전체에 배분하여 균형 잡힌 업무량을 확보해야 합니다.

**해결 방법**: 라운드 로빈 시스템을 사용하여 신입 리드를 영업 담당자에게 자동으로 할당하십시오.

**트리거**: 레코드가 생성됨 (People 객체)

**작업**:

- 마지막으로 할당된 담당자를 찾기 위해 레코드 검색
- 다음 회전 차례의 담당자를 결정하는 코드 작업
- 선택된 담당자에게 리드를 할당하기 위해 레코드 업데이트
- 할당된 담당자에게 알림 이메일 전송

### 리드 할당 - 지리 기반

**문제**: 리드는 지리적 영토 또는 회사 특성에 기반하여 할당되어야 합니다.

**해결 방법**: 위치, 회사 규모 또는 산업에 따라 적절한 영업 담당자에게 리드를 라우팅하십시오.

**트리거**: 레코드가 생성됨 (People 또는 Companies 객체)

**작업**:

- 위치/산업 규칙에 따라 지역을 결정하는 코드 작업
- 지역 소유자를 찾기 위해 레코드 검색
- 리드를 할당하기 위해 레코드 업데이트
- 후속 작업을 위한 레코드 생성

## 영업 프로세스 자동화

### 기회 단계 관리 - 성공적으로 종료

**문제**: 계약이 성사되면 여러 레코드와 팀 구성원 간에 수많은 수작업 업데이트가 필요합니다.

**해결 방법**: 기회가 "성공적으로 종료" 단계로 전환되면 모든 후속 작업을 자동으로 처리하십시오.

**트리거**: 레코드가 업데이트됨 (Opportunities 객체, 스테이지 필드 = "성공적으로 종료")

**작업**:

- 회사 유형을 "잠재 고객"에서 "고객"으로 변경하기 위해 레코드 업데이트
- 계정 관리자에게 배정된 온보딩 작업을 위한 레코드 생성
- 고객 성공 팀에게 이메일 알림 전송
- 외부 결제 시스템 업데이트를 위한 HTTP 요청

### 기회 단계 관리 - 실패한 갱신

**문제**: 재계약 기회가 손실되면 고객 상태를 적절한 계정 관리로 업데이트해야 합니다.

**해결 방법**: 재계약이 실패할 때 고객 상태를 자동으로 업데이트하십시오.

**트리거**: 레코드가 업데이트됨 (Opportunities 객체, 스테이지 = "실패한 종료" 및 유형 = "갱신")

**작업**:

- 회사 유형을 "고객"에서 "이탈한 고객"으로 변경하기 위해 레코드 업데이트
- 이탈 분석 작업을 위한 레코드 생성
- 고객 성공 관리자에게 이메일 경고 전송
- 이탈 날짜와 이유를 추가하기 위해 레코드 업데이트

### 오래된 기회 경고

**문제**: 업데이트가 없는 기회가 방치되어 계약이 소원해지고 예측이 신뢰할 수 없게 됩니다.

**해결 방법**: 최근에 업데이트되지 않은 기회에 대한 자동 알림을 보내십시오.

**트리거**: 일정에 따라 (매일)

**작업**:

- X 일 동안 업데이트되지 않은 기회를 찾기 위해 레코드 검색
- 기회 세부 사항으로 알림 메시지를 형식화하는 코드 작업
- 기회 소유자와 관리자에게 이메일 전송
- 응답이 없는 경우 후속 작업을 위한 레코드 생성

## 생산성 자동화

### 주간 작업 요약

**문제**: 팀원들은 다가오는 작업과 데드라인을 놓칩니다.

**해결 방법**: 자동화된 주간 이메일 알림과 작업 요약을 보내십시오.

**트리거**: 일정에 따라 (매주 월요일 오전 8시)

**작업**:

- 담당자별로 이번 주 마감인 작업을 찾기 위해 레코드 검색
- 사람별로 작업 목록을 형식화하는 코드 작업
- 각 팀원에게 작업 요약이 포함된 이메일 전송
- 관리자에게 팀 개요가 포함된 이메일 전송

### 회의 후속 자동화

**문제**: 회의에서 중요한 실행 항목들이 잊혀지거나 지연됩니다.

**해결 방법**: 회의가 예약되거나 완료될 때 자동으로 후속 작업을 생성하십시오.

**트리거**: 레코드가 생성됨 (활동 객체, 유형 = "회의")

**작업**:

- 회의 준비 작업을 위한 레코드 생성
- 회의 후속 작업을 위한 레코드 생성
- 참석자에게 이메일 알림 전송
- 작업을 회의에 연결하기 위해 레코드 업데이트

### 객체 간 필드 동기화

**문제**: 관련 레코드의 정보를 쉽게 접근할 수 있어야 합니다 (예: 기회 기록에 메인 연락처 이메일).

**해결 방법**: 중첩된 필드가 제공될 때까지 관련 객체 간에 필드를 자동으로 동기화하십시오.

**트리거**: 레코드가 업데이트됨 (Opportunities 객체, 담당자 필드)

**작업**:

- 연결된 사람의 세부 정보를 찾기 위해 레코드 검색
- 이메일 주소를 기회에 복사하기 위해 레코드 업데이트
- 전화번호를 기회에 복사하기 위해 레코드 업데이트
- 회사 정보를 복사하기 위해 레코드 업데이트

## 데이터 검증 및 정리

### 전화번호 표준화

**문제**: 전화번호가 다른 형식으로 입력되어 통화나 메시지에 사용하기 어렵습니다.

**해결 방법**: 전화번호가 입력될 때 표준 형식으로 자동으로 형식화하십시오.

**트리거**: 레코드가 업데이트됨 (사람 객체, 전화 필드)

**작업**:

- 전화번호를 구문 분석하여 형식화하는 코드 작업
- 표준화된 전화 형식으로 레코드 업데이트
- Update Record to add country code if missing

보다 복잡한 자동화 요구 사항의 경우 [전문 서비스](/l/ko/user-guide/workflows/professional-services) 또는 [외부 도구 통합](/l/ko/user-guide/workflows/external-tool-integration)를 고려하십시오.
