워크플로우 크레딧은 Twenty에서 자동화를 지원합니다. 작동 방식을 이해하면 비용을 최적화하고 자동화 예산을 효과적으로 관리할 수 있습니다.

## 플랜별 크레딧 할당

워크플로우 크레딧은 구독 플랜에 따라 각 작업 공간에 할당됩니다:

- **체험판**: 500만 크레딧
- **프로 플랜**: 매월 1,000만 크레딧
- **기관 플랜**: 매월 2,000만 크레딧

## 크레딧 소비 방식 이해

워크플로우가 실행될 때 크레딧이 소모되며, 생성할 때는 아닙니다. 각 워크플로우 동작은 그 복잡성에 따라 크레딧을 소모합니다:

### 동작 유형별 크레딧 소비

- **Basic internal operations**: Very low credit consumption
  - 레코드 검색
  - Create Record
  - 레코드 업데이트
  - 레코드 삭제
  - 양식 작업

- **복잡한 작업**: 높은 크레딧 소비
  - 코드 작업 (JavaScript 실행)
  - 외부 서비스로의 HTTP 요청

- **AI 기능**: 상당한 크레딧 소비 (출시 예정)
  - AI 프롬프트 및 처리는 사용량에 따라 크레딧을 소모합니다

### 실시간 차감

워크플로우가 실행될 때 크레딧이 실시간으로 차감됩니다. 이는 다음을 의미합니다:

- 초안 워크플로우는 크레딧을 소모하지 않습니다
- 활성화된 실행 중인 워크플로우만 크레딧 할당을 사용합니다
- 실패한 워크플로우도 완료된 단계에 대해 크레딧을 소모합니다

## 크레딧 관리

### Check Credit Usage

1. **설정 → 결제**로 이동
2. 현재 크레딧 소비량과 잔액 확인
3. 사용 패턴을 모니터링하여 워크플로우 최적화

### 추가 크레딧 구입

플랜 할당을 초과하는 추가 크레딧이 필요한 경우:

1. **설정 → 결제**로 이동
2. 추가 크레딧 구매 옵션을 클릭합니다. 다양한 크기의 패키지가 제공됩니다.
3. 크레딧이 현재 잔액에 추가됩니다

## 모범 사례

### 효율적인 워크플로우 설계

- **단순하게 시작**: 기본 동작으로 시작하고 점차 복잡성을 추가하세요
- **초안에서 테스트**: 오류로 인한 크레딧 낭비를 피하기 위해 워크플로우를 철저히 테스트하세요
- **HTTP 요청 최소화**: 효율적인 검색 기준을 사용하고 가능한 경우 작업을 결합하세요
- **배치 처리**: 대량 작업 및 이터레이터 작업을 효율적으로 사용하세요
- **오류 처리**: 불필요한 재시도를 방지하기 위해 적절한 오류 처리를 구현하세요
- **수동 트리거 최적화**: 수동 트리거의 경우, `대량` 가용성을 선택하여 여러 기록을 한 번의 워크플로우 실행에서 처리하세요 (자세한 내용은 [워크플로우 기능](/l/ko/user-guide/workflows/workflow-features) 참조)

### 동작 최적화

- 가능한 경우 기본 내부 작업 사용 (낮은 크레딧 소비)
- 효율성을 위한 코드 작업 최적화
- 긴급하지 않은 프로세스에 수동 트리거 고려
- 개별 작업 호출을 줄이기 위해 일괄 작업

### 크레딧 관리

- **정기적 모니터링**: 설정 → 결제에서 주간 크레딧 사용량을 확인하세요
- **사용량 추적**: 소비 패턴을 기록하여 예상치 못한 과잉 소비를 피하세요
- **계획에 맞춰**: 더 많은 크레딧이 지속적으로 필요하다면 플랜 업그레이드를 고려하세요
- **워크플로우 검토**: 높은 소비 워크플로우를 정기적으로 검토하고 사용하지 않는 것을 비활성화하세요
- **전략적 계획**: 크레딧 가용성을 기반으로 워크플로우 배포를 계획하세요

상세한 결제 정보와 크레딧 관리에 대해서는 Twenty 작업 공간의 **설정 → 결제**를 방문하세요.
