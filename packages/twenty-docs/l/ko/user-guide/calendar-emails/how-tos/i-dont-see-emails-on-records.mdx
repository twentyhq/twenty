## 일반적인 원인

### 1. 초기 동기화가 아직 진행 중

이메일 동기화에는 시간이 걸리며, 특히 메일함이 큰 경우 더 오래 걸립니다. Twenty는 분당 약 **400건의 메시지**를 가져옵니다(이것은 Gmail API의 속도 제한에 의해 제한됩니다).

* **캘린더 동기화**: 몇 분 안에 완료됨
* **이메일 동기화**: 메일함 크기에 따라 달라짐 (예: 이메일 10,000개 ≈ 25분, 50,000개 ≈ 2시간)

**해결 방법**: 초기 가져오기가 완료될 때까지 기다리세요. 메일함 크기를 기준으로 소요 시간을 추정할 수 있습니다.

### 2. Twenty에 연락처가 존재하지 않음

이메일은 기존 사람 레코드에만 표시됩니다. 연락처가 아직 생성되지 않았다면:

* 메일함 설정에서 **연락처 자동 생성**을 활성화하세요
* 또는 먼저 사람 레코드를 수동으로 생성하세요

**해결 방법**: **Settings → Accounts**로 이동하여 메일함을 선택하고 연락처 자동 생성을 활성화하세요.

### 3. 내부 이메일은 제외됨

동료 간 이메일(같은 이메일 도메인)은 개인 정보 보호를 위해 절대 동기화되지 않습니다.

**해결 방법**: 이는 의도된 동작입니다. 외부 이메일만 동기화됩니다.

### 4. 그룹 또는 배포 목록에서 온 이메일

그룹 이메일과 배포 목록은 동기화에서 제외됩니다.

**해결 방법**: 이는 의도된 동작입니다.

### 5. 동기화 대상으로 폴더가 선택되지 않음

Message Folder 기능을 사용하는 경우 일부 폴더가 제외되어 있을 수 있습니다.

**해결 방법**: **Settings → Accounts**로 이동하여 메일함을 선택하고 폴더 동기화 설정을 확인하세요.

### 6. 레코드의 이메일 주소가 잘못됨

사람 레코드에 이메일에서 사용된 주소와 다른 이메일 주소가 등록되어 있을 수 있습니다.

**해결 방법**: 올바른 이메일 주소를 사람 레코드에 추가하세요.

## 아직도 작동하지 않나요?

1. 메일함 연결을 끊었다가 다시 연결해 보세요
2. 문제가 지속되면 지원팀에 문의하세요
