**설정 → 구성원**에서 워크스페이스에 접근할 수 있는 사람을 관리하세요.

## 새 멤버 초대

### 이메일 초대 사용

1. **설정 → 구성원**으로 이동하세요.
2. **+ 초대**를 클릭하세요.
3. 해당 사람의 이메일 주소를 입력하세요.
4. 새 구성원의 역할을 선택하세요.
5. **초대 보내기**를 클릭하세요.

초대받은 사람은 워크스페이스에 참여할 수 있는 링크가 포함된 이메일을 받습니다.

### 초대 링크 사용

1. **설정 → 구성원**으로 이동하세요.
2. 작업 공간 초대 링크 복사
3. 새로운 팀 구성원에게 링크 공유
4. 가입 후 접근이 가능합니다

## 구성원 보기 및 관리

### 모든 구성원 보기

**설정 → 구성원**으로 이동하여 다음을 확인하세요:

* 모든 활성 구성원
* 보류 중인 초대

### 구성원 프로필 편집

구성원을 클릭하여 프로필 페이지를 엽니다. 관리자인 경우 다음을 수행할 수 있습니다:

* **이름** 변경
* **프로필 사진** 변경
* 해당 계정을 **가장하기** (문제 해결에 유용)
* 해당 계정 **삭제**

### 구성원 역할 변경

구성원 프로필 페이지에서:

1. **권한** 탭 열기
2. 현재 할당된 역할 보기
3. 드롭다운에서 다른 역할 선택
4. 변경 사항은 즉시 적용됩니다.

→ [역할과 권한에 대해 더 알아보기](/l/ko/user-guide/permissions-access/capabilities/permissions)

### 구성원 제거

1. 구성원을 클릭하여 프로필을 엽니다.
2. 워크스페이스에서 제거하려면 **삭제**를 클릭하세요.

<Warning>
  제거된 구성원은 즉시 접근 권한을 잃습니다. 해당 구성원의 데이터(레코드, 노트, 작업)는 워크스페이스에 남아 있습니다.
</Warning>

<Warning>
  **이메일 동기화도 제거됩니다.** 삭제된 사용자가 특정 이메일을 동기화한 유일한 사용자였다면 해당 이메일은 워크스페이스에서 영구적으로 제거됩니다.
</Warning>

## 보류 중인 초대

수락되지 않은 초대를 관리합니다:

* **재전송**: 초대 이메일을 다시 보냅니다
* **취소**: 초대가 수락되기 전에 철회합니다

## 승인된 접근 도메인

팀원이 이메일 도메인에 따라 자동으로 참여할 수 있도록 허용합니다:

1. **설정 → 도메인**으로 이동하세요.
2. 회사 도메인을 추가하세요(예: `yourcompany.com`).
3. 해당 이메일 도메인을 가진 사람은 초대 없이 참여할 수 있습니다.
