## 현재 통합 옵션

### 네이티브 통합

Twenty는 현재 다음과 함께 네이티브 통합을 제공합니다:

- **이메일 & 캘린더**: Gmail, Outlook 또는 SMTP/CalDAV 제공업체 연결
- **API 액세스**: 당사의 REST 및 GraphQL API를 사용하여 사용자 정의 통합을 구축하세요.

이메일 및 캘린더 설정은 [이메일 & 캘린더 설정](/l/ko/user-guide/settings/email-calendar-setup)을 참조하세요.

### 워크플로우(추천)

Twenty를 다른 도구에 연결하는 기본 방법은 앱 내 **워크플로우** 기능을 사용하는 것입니다:

- **HTTP 노드**: 외부 서비스에 API 호출 수행
- **코드 노드**: 복잡한 통합을 위한 사용자 정의 로직 작성
- **웹훅 트리거**: 외부 시스템에서 데이터 수신
- **필드 업데이트 트리거**: CRM 변경에 기반한 작업 자동화

저희 [워크플로우 섹션](/l/ko/user-guide/workflows/getting-started-workflows)에서 자세히 알아보세요.

### Zapier 통합(이전 버전)

코드 없는 자동화를 선호하는 사용자들을 위해 Zapier 통합을 유지합니다:

1. [Zapier에서의 Twenty](https://zapier.com/apps/twenty/integrations)를 방문하세요.
2. Create a new Zap with Twenty as trigger or action
3. Generate an API key in Settings → API & Webhooks
4. 당신의 Twenty 작업 공간을 Zapier에 연결하세요.

## 미래 비전: 커뮤니티에서 제작한 커넥터 (2026)

### 확장성 플랫폼

개발자가 코드로 앱을 만들 수 있는 확장성 플랫폼을 구축하고 있습니다. 다음을 가능하게 할 것입니다:

- **사용자 정의 커넥터**: 좋아하는 소프트웨어와의 통합 구축
- **커뮤니티 기여**: 다른 사람들이 만든 커넥터를 공유하고 탐색
- **유연한 아키텍처**: Twenty의 기능을 핵심 기능 이상으로 확장

### AI 지원 워크플로우 구축

2026년, AI 지원이 사용자들이 다음을 할 수 있도록 도울 것입니다:

- **워크플로우 자동 생성**: 여러분의 통합 요구 사항을 간단한 언어로 설명하세요.
- **스마트 제안**: 특정 도구를 연결하기 위한 추천을 받습니다.
- **템플릿 라이브러리**: 공통 사용 사례에 대한 사전 구축된 워크플로우에 접근

Twenty 통합의 미래는 커뮤니티 주도, AI 지원 및 무한히 확장 가능합니다.
