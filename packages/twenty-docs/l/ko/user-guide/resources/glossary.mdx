## API

API(응용 프로그래밍 인터페이스)는 Twenty를 다른 소프트웨어 시스템과 연결하고 맞춤형 통합을 구축할 수 있게 합니다.

## 명령 메뉴

명령 메뉴는 신속한 작업 수행, 기록 생성 및 작업 공간 내비게이션을 효율적으로 도와주는 빠른 접근 인터페이스입니다. (Mac에서는 `Cmd + K`, Windows에서는 `Ctrl + K`로 열림)

## 회사 및 사람들

CRM에는 두 가지 기본 유형의 기록이 있습니다:

- `회사`는 기업이나 조직을 나타냅니다.
- `사람들`은 현재 및 잠재 고객 또는 클라이언트를 나타냅니다.

## 맞춤 필드

맞춤 필드는 비즈니스 요구 및 프로세스에 특정한 정보를 캡처하도록 생성하는 데이터 필드입니다.

## 데이터 모델

데이터 모델은 CRM에서 정보가 어떻게 구성되는지를 정의하는 구조로, 어떤 객체가 존재하는지, 그 속성(필드), 그리고 서로 어떻게 관련되는지를 포함합니다.

## 즐겨찾기

즐겨찾기는 빠르게 접근할 수 있도록 사이드바에 나타나는 중요 데이터로, 표시해 둔 기록입니다.

## 필드

필드는 특정 엔터티에 대한 특정 데이터가 저장되는 영역을 나타냅니다.

## 통합

통합은 Twenty를 다른 소프트웨어나 시스템과 연결할 수 있게 해주는 내장 도구입니다.

## 칸반

`칸반`은 카드와 열을 사용하여 비즈니스 프로세스를 추적하는 시각적 방법입니다. 각 열은 프로세스의 단계를 나타내며(예: 신규, 진행 중, 승리, 패배), 기록이 진행됨에 따라 이 단계들로 이동합니다.

## 개체

객체는 CRM에서 특정 유형의 엔터티(예: 사람, 회사, 기회)를 나타내는 데이터 구조입니다. Objects can be standard (built-in) or custom (created by you).

## 기회

Twenty CRM에서 기회란 계정 또는 연락처와의 잠재적 거래 또는 판매를 의미합니다.

## 기록

기록은 특정 계정 또는 연락처와 같은 객체의 인스턴스를 나타냅니다.

## 관계 필드

관계 필드는 다른 객체들 간의 연결을 만들어 주며, 기록을 서로 연결할 수 있게 합니다. (예: 사람을 회사에 연결)

## 표준 필드

표준 필드는 기본적으로 제공되는 사전 빌드 데이터 필드로, 모든 작업 공간에서 일반적인 기능을 제공합니다.

## 작업

Twenty CRM의 작업은 연락처, 계정 또는 기회와 관련된 할당된 활동입니다.

## 보기들

필터, 레이아웃 및 정렬 옵션을 설정하여 각 보기를 위해 기록의 표시를 사용자 지정할 수 있습니다.

## Webhooks

Webhooks are automated messages sent from Twenty to other applications when specific events occur, enabling real-time data synchronization.

## Workflows

워크플로우는 특정 조건에 따라 작업을 자동으로 트리거하여 반복적인 작업 및 비즈니스 프로세스를 자동화하는 데 도움이 되는 프로세스입니다.

## 워크스페이스

`워크스페이스`는 일반적으로 Twenty를 사용하는 회사를 나타냅니다. Twenty에 팀원들이 추가하는 모든 기록과 데이터를 보유합니다.  
일반적으로 회사가 직원 이메일 주소에 사용하는 도메인 이름을 가지는 단일 도메인 이름이 있습니다.

## 워크스페이스 멤버들

워크스페이스 멤버는 작업 공간에 접근할 수 있는 팀의 Twenty 사용자들입니다. 그들은 기록의 소유자 또는 할당자로 지정될 수 있습니다.
