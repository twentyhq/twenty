## 크레딧이란 무엇인가요?

크레딧은 실제 비용이 드는, 여러분을 대신해 우리가 실행하는 모든 것 — **워크플로 자동화**, **AI 에이전트**, **AI 채팅** — 을 구동하는 단일 잔액입니다. "automation runs"와 "AI tokens"에 대한 개별 할당량을 따로 관리하는 대신, 모니터링하기 쉬운 하나의 단순한 잔액만 살펴보면 됩니다.

**크레딧 1개 = $1 사용량.** 크레딧은 말 그대로 1달러어치의 작업이므로, 추상적인 단위를 따로 환산할 필요 없이 잔액을 직관적으로 이해할 수 있습니다.

## 크레딧 하나로 어느 정도까지 사용할 수 있나요?

비용은 작업량에 비례해 증가합니다. 일상적인 자동화와 짧은 AI 메시지는 크레딧의 아주 일부분만 사용하지만, 크고 여러 단계를 거치는 AI 에이전트 작업은 더 많은 크레딧을 사용합니다. 대략적인 개요는 다음과 같습니다:

| 하는 일                                    | 대략적인 비용                         | 의미                     |
| --------------------------------------- | ------------------------------- | ---------------------- |
| 표준 워크플로 단계 실행(검색, 레코드 생성, 레코드 업데이트)     | ~$0.0001                        | 크레딧 **1개로 약 10,000단계** |
| 외부 API를 호출하거나 코드 노드를 실행                 | 크레딧의 작은 일부                      | 크레딧 1개당 수천 회           |
| 빠른 AI 채팅 메시지를 보내거나 단순한 AI 단계를 실행        | 1센트의 일부                         | 크레딧 1개당 수백 회           |
| 대형 AI 에이전트 작업 실행(예: "워크스페이스 객체 10개 구성") | 비용이 **약 $1 또는 그 이상**까지 올라갈 수 있음 | 크레딧 1개당 몇 회            |
| AI 웹 검색 실행                              | 검색 1회당 고정 비용                    | 크레딧 1개당 수백 회 검색        |

<Note>
대부분의 팀에게 표준 자동화는 사실상 무료에 가깝고, 눈에 띄게 잔액이 줄어드는 것은 AI 기능에 많이 의존하거나 무거운 코드 노드를 실행할 때뿐입니다. 이는 의도된 설계로, 포함된 크레딧은 사용량 계측기를 신경 쓰지 않고도 자유롭게 구축할 수 있도록 제공됩니다.
</Note>

## AI 사용량은 어떻게 계측되나요

액션이 AI 모델을 호출하면, 사용된 토큰 수를 정확히 측정하고 **모델 제공자가 공개한 요금**을 기준으로 가격을 산정한 뒤, 그 달러 금액을 그대로 크레딧으로 환산합니다. 즉, 실제 작업 비용에 맞춰 사용량이 청구되며, 내부 마진이 붙은 요율로 과금되지 않습니다. 짧은 채팅 메시지는 1센트의 일부만 비용이 듭니다. 큰 에이전트 작업(예: 워크스페이스 객체 10개 구성)은 점점 커지는 컨텍스트에서 여러 번 모델을 호출하므로, 비용이 1달러 이상까지 쌓일 수 있습니다. 실제로 사용한 양, 토큰 단위까지 정확히 과금됩니다.

비용은 모델에 따라 달라지며, 가벼운 모델은 최고급 추론 모델에 비해 메시지당 비용이 훨씬 저렴합니다. **설정 → 결제**에서 각 액션별 사용 내역과 사용된 모델을 확인할 수 있습니다.

## 포함 내용

모든 구독에는 크레딧이 포함되며, 크레딧 양은 요금제가 아니라 **결제 주기**에 따라 달라집니다:

| 청구 주기 | 포함 크레딧 | 동등 사용량           |
| ----- | ------ | ---------------- |
| 월간    | 월 5    | 매월 약 $5 상당의 사용량  |
| 연간    | 연 50   | 매년 약 $50 상당의 사용량 |

이 크레딧은 일상적인 자동화를 넉넉하게 처리할 수 있도록 설계되었습니다. 크레딧 5개면 수만 번의 표준 자동화 단계나 수백 건의 빠른 AI 메시지를 처리할 수 있지만, 일부 대형 다단계 에이전트 작업은 크레딧을 더 빨리 소진시킬 수 있습니다. 언제든지 크레딧을 추가 충전할 수 있습니다(아래 참조).

## 크레딧 이월

사용하지 않은 크레딧은 다음 결제 기간으로 자동 이월됩니다.

* **한도**: 이월은 한 기간의 전체 할당량으로 제한되므로, 기간당 요금제가 제공하는 크레딧보다 더 많이 이월할 수 없습니다.
* **표시**: 이월 크레딧이 있으면 **설정 → 결제**에서 별도의 **이월 크레딧** 항목으로 표시되며, 그 옆에 **총 사용 가능** 잔액이 함께 표시됩니다.

## 사용량 모니터링

잔액과 사용처를 계속 파악하세요:

1. **설정 → 결제**로 이동
2. **남은 크레딧**과 **현재 기간의 사용량**을 확인
3. 세부 내역을 확인해 어떤 워크플로와 AI 액션이 크레딧을 사용하고 있는지, 또한 AI 액션이 어떤 모델을 사용했는지 파악하세요

이를 정기적으로 검토하면 무거운 워크플로를 일찍 발견해 최적화할지, 아니면 크레딧을 추가 충전할지 쉽게 결정할 수 있습니다.

## 크레딧이 더 필요하신가요?

크레딧을 모두 사용했거나, 대규모 자동화를 진행하기 전에 여유분을 확보하고 싶다면 언제든지 추가 크레딧 팩을 구매할 수 있습니다:

1. **설정 → 결제**로 이동
2. **추가 크레딧 구매**를 선택
3. 필요한 크레딧 수량을 선택하세요

구매한 크레딧은 즉시 사용 가능 잔액에 추가됩니다.
