## 개요

워크플로우 크레딧은 Twenty에서 자동화를 지원합니다. 각 워크플로우 동작은 그 복잡성에 따라 크레딧을 소모합니다.

## 크레딧 할당

크레딧은 플랜이 아니라 청구 주기에 따라 결정됩니다:

| 청구 주기 | 크레딧    |
| ----- | ------ |
| 월간    | 월 500만 |
| 연간    | 연 5천만  |

<Note>
  월 500만 크레딧은 비용을 걱정하지 않고 자동화를 실행할 수 있도록 설계되었습니다. 표준 동작을 사용하는 대부분의 워크플로우에는 이 정도로 충분합니다. 고급 코드 노드나 AI 기반 기능을 실행할 때에만 추가 크레딧이 필요합니다.
</Note>

## 크레딧 소모

동작에 따라 소모되는 크레딧 양이 다릅니다:

| 동작 유형                         | 크레딧 사용량     |
| ----------------------------- | ----------- |
| **기본 작업** (검색, 업데이트, 레코드 생성)  | 최소          |
| **복잡한 작업** (코드 노드, 외부 API 호출) | 더 많은 크레딧 소모 |
| **AI 프롬프트** (곧 제공 예정)         | 사용량에 따라 변동  |

작업흐름이 실행될 때 크레딧은 실시간으로 차감됩니다.

## 사용량 모니터링

크레딧 소모량을 추적하세요:

1. **설정 → 결제**로 이동
2. 현재 사용량과 남은 크레딧을 확인하세요
3. 추세를 모니터링하여 필요 시 추가 크레딧을 계획하세요

## 추가 크레딧 구입

크레딧이 더 필요하신가요?

1. **설정 → 결제**로 이동
2. 추가 크레딧 팩을 구매하는 옵션을 클릭하세요
3. 필요한 크레딧 수량을 선택하세요
