## كيف يعمل تعيين الحقول

عند تحميل ملف، يقوم Twenty بتحليل أعمدتك ومحاولة مطابقتها مع الحقول الموجودة.

### التعيين التلقائي

يحاول Twenty مطابقة الأعمدة استنادًا إلى:

* أسماء رؤوس الأعمدة (مطابقات دقيقة أو متشابهة)
* اكتشاف نوع البيانات (تواريخ، أرقام، عناوين بريد إلكتروني)
* أنماط الحقول الشائعة

<Note>**نصيحة سريعة:** صدِّر بضعة صفوف من الكائن الذي تريد استيراده. سيحتوي الملف المصدَّر على أسماء الأعمدة الدقيقة التي يتوقعها Twenty، مما يجعل التعيين التلقائي سلسًا أثناء الاستيراد.</Note>

### خيارات التعيين اليدوي

لكل عمود، يمكنك:

* **تعيين إلى حقل**: اختر الحقل المطابق في Twenty من قائمة منسدلة
* **عدم التعيين**: تخطَّ العمود بالكامل (لن يتم استيراد البيانات)

<Note>**يجب أن تكون الحقول موجودة قبل الاستيراد.** الاستيراد يُنشئ السجلات، لا الحقول. أنشئ الحقول المخصصة ضمن **الإعدادات → نموذج البيانات** قبل الاستيراد.</Note>

## توافق أنواع الحقول

جميع أنواع الحقول المتاحة في نموذج البيانات مدعومة للاستيراد.

يمكنك أيضًا استيراد قيم `id` إما لتعيين معرّف محدد للسجلات الجديدة أو لتحديث السجلات القائمة.

## متطلبات تنسيق البيانات

<Note>**لبعض الحقول صياغة خاصة.** نوصي بتنزيل الملف النموذجي قبل إعداد عملية الاستيراد للاطلاع على الصياغة المتوقعة لكل نوع حقل.</Note>

### حقول العنوان

العنوان حقل متداخل يضم عدة أعمدة. يمكن ترك بعضِها فارغًا.

* **Address / Address 1**: سطر عنوان الشارع 1
* **Address / Address 2**: سطر عنوان الشارع 2
* **Address / City**: اسم المدينة
* **Address / State**: الولاية أو المقاطعة
* **Address / Country**: اسم الدولة
* **Address / Post Code**: الرمز البريدي/ZIP

### حقول المصفوفة

استخدم التنسيق التالي:

```
[\"value1\",\"value2\"]
```

### حقول القيم المنطقية

استخدم `TRUE` أو `FALSE` (أحرف كبيرة) - وليس `true` أو `false`

### حقول العملة

العملة حقل متداخل يحتوي على عمودين يجب **ملؤهما كلاهما**:

* **Amount / Amount**: القيمة الرقمية (مثال: `1234.56`)
* **Amount / Currency**: رمز العملة (مثال: `USD`، `EUR`)

### حقول التاريخ

التنسيقات المدعومة:

* `YYYY-MM-DD` (موصى به)
* `MM/DD/YYYY`
* `DD/MM/YYYY`
* تنسيق ISO 8601

### حقول النطاق

* يُوصى باستخدام التنسيق `https://domain.com` لتجنب إنشاء التكرارات، إذ إن هذا هو التنسيق المستخدم للشركات التي يتم إنشاؤها عبر مزامنة صندوق البريد والتقويم
* يمكن ملء حقلي `Domain Label` و`Domain URL`: وأفضل ممارسة هي وضع `domain.com` في التسمية و`https://domain.com` في عنوان URL
* يجب أن تكون النطاقات فريدة داخل كائن الشركات
* **يجب أن تكون النطاقات فريدة داخل الملف المُراد استيراده**

### حقول البريد الإلكتروني

* يجب أن تكون بتنسيق بريد إلكتروني صحيح
* يجب أن تكون عناوين البريد الإلكتروني فريدة داخل كائن الأشخاص
* **يجب أن تكون عناوين البريد الإلكتروني فريدة داخل الملف المُراد استيراده**
* للبريد الإلكتروني الإضافي: استخدم **Emails / Primary Email** للبريد الرئيسي، و**Emails / Additional Emails** بهذا التنسيق:

```
[\"jane@twenty.com\",\"jane.doe@twenty.com\"]
```

### حقول المعرّف

تحديد `id` أثناء الاستيراد اختياري. يقوم Twenty بإنشاء واحد تلقائيًا إذا لم يتم توفيره.

حالات استخدام لتعيين عمود `id`:

* **تعيين معرّف محدد**: اختر UUID للسجلات التي سيتم إنشاؤها
* **تحديث السجلات الحالية**: طابِق السجلات الحالية لتحديثها بدلاً من إنشاء تكرارات. في هذه الحالة، يُنصح بعدم تعيين الحقول الفريدة الأخرى: فاقتصار التعيين على حقل فريد واحد يضمن عملية استيراد أكثر سلاسة.

<Note>إذا قدّمت قيمة `id`، فيجب أن تكون بتنسيق UUID (مثال: `c776ee49-f608-4a77-8cc8-6fe96ae1e43f`).</Note>

### حقول JSON

استخدم تنسيق JSON صالحًا:

```
{"key":"value","key2":"value2"}
```

### حقول الروابط

مشابه لحقول النطاق:

* املأ عمودي التسمية والرابط معًا: **Links / Link URL** و**Links / Link Label**
* استخدم تنسيق الرابط الكامل: `https://example.com`
* للروابط الثانوية، استخدم عمود **Links / Secondary Links** بهذا التنسيق:

```
[{\"url\":\"https://twenty.com\",\"label\":\"Twenty\"}]
```

### حقول التحديد المتعدد

استخدم **أسماء واجهة برمجة التطبيقات (API)** (وليس تسميات العرض) بالتنسيق التالي:

```
[\"VALUE1\",\"VALUE2\"]
```

اطّلع [هنا](#finding-api-names-for-select-fields) لمعرفة مكان العثور على أسماء واجهة برمجة التطبيقات.

<Note>لن يتم إنشاء خيارات التحديد الجديدة تلقائيًا بواسطة الاستيراد. يجب إضافتها ضمن **الإعدادات → نموذج البيانات** قبل الاستيراد.</Note>

<Warning>
  **الاستيراد يستبدل، لا يضيف.**

  إذا كان السجل يحتوي بالفعل على `VALUE2` و`VALUE3` محدّدين، ثم استوردت `[\"VALUE1\"]`، فسيحتوي السجل على `VALUE1` فقط بعد الاستيراد. يتم استبدال التحديدات السابقة، وليس دمجها.
</Warning>

### حقول الأرقام

* أرقام فقط
* تُكتب الكسور العشرية بنقطة: `1234.56`
* من دون فواصل آلاف

### حقول الهاتف

الهاتف حقل متداخل يضم عدة أعمدة يجب **ملؤها**

* **Phones / Primary Phone Number**: رقم الهاتف (مثال: `4159095555`)
* **Phones / Primary Phone Country Code**: رمز الدولة (مثال: `US`)
* **Phones / Primary Phone Calling Code**: رمز الاتصال (مثال: `+1`)

### حقول التقييم

استخدم تنسيق اسم واجهة برمجة التطبيقات: `RATING_1`، `RATING_2`، `RATING_3`، `RATING_4`، `RATING_5`

### حقول العلاقات

يرجى الاطّلاع على مقالتنا المخصّصة: [استيراد العلاقات بين الكائنات](/l/ar/user-guide/data-migration/capabilities/import-relations)

### حقول الاختيار

استخدم **اسم واجهة برمجة التطبيقات** للخيار (وليس تسمية العرض):

```
VALUE1
```

اطلع [هنا](#finding-api-names-for-select-fields) على مكان العثور على أسماء واجهة برمجة التطبيقات.
<Note>لن يتم إنشاء خيارات الاختيار الجديدة تلقائيًا بواسطة الاستيراد. يجب إضافتها ضمن **الإعدادات → نموذج البيانات** قبل الاستيراد.</Note>

### الحقول النصية

* لا يلزم تنسيق خاص
* تُزال المسافات في البداية والنهاية

## العثور على أسماء واجهة برمجة التطبيقات

بالنسبة لحقول الاختيار، والاختيار المتعدد، والمصفوفة ذات الخيارات المحددة مسبقًا، يجب استخدام **أسماء واجهة برمجة التطبيقات**، وليس تسميات العرض.

### كيفية العثور على أسماء واجهة برمجة التطبيقات

1. اذهب إلى **الإعدادات → نموذج البيانات**
2. اختر الكائن والحقل
3. فعّل **الوضع المتقدم** (زر التبديل في أسفل يمين صفحة الإعدادات)
4. اعرض اسم واجهة برمجة التطبيقات لكل خيار
