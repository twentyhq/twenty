## كيف تعمل سير العمل

كل سير عمل يتكون من ثلاثة أجزاء:

1. **المشغل (Trigger)** — ما يبدأ سير العمل
2. **الخطوات (Steps)** — ما يحدث لاحقًا (إجراء واحد أو أكثر بالتسلسل)
3. **المتغيرات (Variables)** — البيانات التي تتدفق بين الخطوات

## المشغلات

| المشغّل                      | متى يتم تشغيله                                                                  |
| ---------------------------- | ------------------------------------------------------------------------------- |
| **حدث السجل (Record event)** | يتم إنشاء سجل أو تحديثه أو حذفه أو إجراء upsert له                              |
| **يدوي**                     | ينقر المستخدم على زر (على سجل واحد أو سجلات متعددة أو على مستوى النظام بالكامل) |
| **الجدولة (Schedule)**       | على فاصل زمني متكرر (بصيغة cron)                                                |
| **الويب هوك**                | نظام خارجي يرسل طلب HTTP POST                                                   |

## الإجراءات

يمكن لسير العمل ربط أي مجموعة من العناصر التالية:

* **عمليات السجل (Record operations)** — إنشاء السجلات أو تحديثها أو البحث عنها أو حذفها أو إجراء upsert لها
* **إرسال بريد إلكتروني (Send email)** — إرسال رسائل البريد الإلكتروني أو إعداد مسودات منها من الحسابات المتصلة
* **طلب HTTP (HTTP request)** — استدعاء أي واجهة برمجة تطبيقات خارجية
* **الكود (Code)** — تشغيل JavaScript مخصص من أجل منطق معقد
* **التفرعات (Branches)** — شروط If/else لتقسيم مسار سير العمل
* **Iterator** — التكرار عبر المصفوفات من البيانات
* **وكيل ذكاء اصطناعي (AI Agent)** — تمكين وكيل ذكاء اصطناعي من معالجة البيانات بشكل مستقل
* **التأخير (Delay)** — الانتظار قبل المتابعة
* **النموذج (Form)** — جمع مدخلات المستخدم في منتصف سير العمل

## ما الذي يمكنك بناؤه

* إرسال تنبيهات Slack عند وصول صفقة إلى مرحلة معينة
* إثراء جهات الاتصال الجديدة تلقائيًا ببيانات من واجهات برمجة تطبيقات خارجية
* اكتشاف الفرص الراكدة وإخطار المالك
* مزامنة البيانات بين Twenty ونظام الفوترة لديك
* إنشاء ملفات PDF أو فواتير من بيانات السجل
* الرد تلقائيًا على رسائل البريد الإلكتروني الواردة التي تطابق معايير معينة
