## Objektverwaltung

<AccordionGroup>
<Accordion title="Can I reorder objects in the left navigation bar?">
Noch nicht. Die Reihenfolge der Objekte in der Navigation ist derzeit festgelegt, aber diese Funktion ist für eine zukünftige Veröffentlichung geplant.
</Accordion>

<Accordion title="Can I hide objects from the left navigation bar?">
Alle aktiven Objekte erscheinen in der Navigation. Sie können Objekte, die Sie nicht benötigen, unter **Einstellungen → Datenmodell** deaktivieren.
</Accordion>

<Accordion title="Can I delete standard objects (People, Companies, etc.)?">
You can deactivate any standard objects but you cannot hard delete them.
</Accordion>
</AccordionGroup>

## Feldmöglichkeiten

<AccordionGroup>
<Accordion title="Can I create formula fields?">
Formelfelder kommen im **1. Quartal 2026**. In der Zwischenzeit können Sie Workflows verwenden, um Feldwerte automatisch zu berechnen und zu aktualisieren.
</Accordion>

<Accordion title="Can I have nested fields in my objects?">
Verschachtelte Felder kommen im **1. Quartal 2026**. Aktuell können Sie Workflows verwenden, um Felderwerte von verwandten Objekten einzuholen. Zum Beispiel: Um die Branche eines Unternehmens in einem Personen-Datensatz anzuzeigen, erstellen Sie ein benutzerdefiniertes Feld für Personen und verwenden Sie einen Workflow, um den Wert zu synchronisieren.
</Accordion>

<Accordion title="Why can't I update relation field names?">
Die Namen von Beziehungsfeldern beeinflussen die API-Struktur und können nach der Erstellung nicht geändert werden. Wenn Sie ein Beziehungsfeld umbenennen müssen, erstellen Sie ein neues und löschen Sie das alte.
</Accordion>

<Accordion title="Why do I need different singular and plural names?">
Our GraphQL API uses both forms for different operations:
- `createPerson` (singular) for single record actions
- `createPeople` (plural) for bulk operations

Dies schafft Einschränkungen, wenn Singular- und Pluralformen gleich sind, verbessert jedoch die Entwicklererfahrung. </Accordion>

<Accordion title="Why are some field names protected?">
Bestimmte Feldnamen wie `Typ` oder `Applikation` sind für Systemnutzung reserviert. Wählen Sie stattdessen alternative Namen wie `Kategorie` oder `Klassifikation`.
</Accordion>
</AccordionGroup>

## Erweiterte Funktionen

<AccordionGroup>
<Accordion title="Can I create many-to-many relationships?">
Viele-zu-Viele-Beziehungen kommen im **1. Quartal 2026**. Derzeit erstellen Sie ein Zwischenobjekt mit zwei 1-zu-viele-Beziehungen als Workaround.
</Accordion>

<Accordion title="What are Morph Many relationships?">
'Morph Many'-Beziehungen ermöglichen einem Objekt, sich auf mehrere unterschiedliche Objekttypen zu beziehen. Zum Beispiel könnte eine Gelegenheit sich entweder auf eine Person oder ein Unternehmen beziehen. Diese Funktion ist jetzt verfügbar.
</Accordion>

<Accordion title="Can I reorder fields in objects?">
Feld-Umsortierung wird mit benutzerdefinierten Layouts im **4. Quartal 2025** verfügbar sein. Derzeit werden Felder in alphabetischer Reihenfolge angezeigt.
</Accordion>
</AccordionGroup>

## Zugriff und Berechtigungen

<AccordionGroup>
<Accordion title="Where can I see and edit my data model?">
You can access your Data Model under **Settings → Data Model**.
</Accordion>

<Accordion title="Why can't I see the Data Model under Settings?">
Wenden Sie sich an Ihren Arbeitsbereichs-Administrator. Der Zugriff auf das Datenmodell ist in der Regel nur für Administratoren eingeschränkt.
</Accordion>

<Accordion title="How many custom objects or fields can I create?">
You can create as many custom objects and fields as you need - the price won't change.
</Accordion>
</AccordionGroup>
