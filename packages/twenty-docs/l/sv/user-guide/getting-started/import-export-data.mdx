## Importera Data

- Du kan importera data för vilket objekt som helst med en .csv, .xlsx, eller .xls fil.
- Varje fil du laddar upp måste innehålla **endast en typ av objekt** (till exempel, bara personposter).
- Du kan använda Import för att **skapa eller uppdatera poster**.

### Ladda ner en exempelfil för att matcha det förväntade formatet

1. Gå till en vy med objektet du ska importera.
2. Klicka på `⋮` ikonen uppe till höger och välj sedan `Importera poster`.
3. Klicka på `Ladda ner exempelfil`.

### Förbered din csv

Nedan finns några punkter att kontrollera innan du laddar upp din fil.

- Begränsa antalet poster till **10 000 per fil**.
- **Ta bort dubbletter** från din fil.
  - Uniciteten på `Personer` sätts som standard av `id` och `email`. Du kan också definiera egna fält från `Personer` som unika när du konfigurerar din datamodell.
  - Uniciteten på `Företag` sätts som standard av `id` och `domän`. Du kan också definiera egna fält från `Företag` som unika när du konfigurerar din datamodell.
  - För alla andra objekt, inklusive anpassade objekt, kan du definiera några fält som unika. Se till att inte inkludera dubbletter när du förbereder dina filer.
- Vi rekommenderar att använda syntaxen `https://domain.com` när du laddar upp dina domäner, eftersom detta är den som används av vår koppling till din brevlåda och kalender.
- Du kan **importera relationerna mellan objekt** genom att tillhandahålla ett av de unika fälten i den associerade posten.
  - Exempel: Du vill koppla en person till ett företag. Add a column in the file containing all the `People` records that contains the `id` of the company -- or its `domain`. Du kommer kunna mappa detta fält under uppladdningen.
    <Warning>

**Viktigt meddelande:**

- Relationer mellan objekt i Twenty är "Ett till Många". This means each record of object A can be attached to several records of object B. But each record of object B can belong to only one record of object A.  
  \*For example, one company can be attached to several people. Och en person kan bara tillhöra ett företag.

- För att ladda upp relationer via Import-funktionen behöver du ange `id` (eller något annat unikt fält) för det kopplade objektet i filen som innehåller posterna på "Många-sidan" av relationen.  
  För exempel, ange företagets `id` eller `domän` när du laddar upp personposter. Du tillhandahåller inte personens `id` (eller `email`) när du laddar upp filen med företag.
  </Warning>

### Ladda upp din fil

1. Gå till en vy med objektet du ska importera.
2. Klicka på `⋮` ikonen uppe till höger och välj sedan `Importera poster`.
3. Klicka på `Välj fil`.
4. Validera mappningen av fälten.

- You don't have to import all of them, you can choose the "Do not map" option.
- För relationer rekommenderas att endast mappa ett av de unika fälten.
- Du kanske även behöver mappa värdena för dina val- och fler-val fälttyper.

5. Klicka på `Nästa steg` och `Granska rader med fel`. Celler med ett problem är markerade. **Du kan antingen ta bort raden eller uppdatera cellen direkt därifrån**.
6. När du är klar, klicka på `Bekräfta`

### Importerings FAQ

<details><summary>Jag ser dubblettproblem när jag laddar upp min fil, vad ska jag göra?</summary>

Vänligen se avsnittet **Förbered din csv** ovan i denna artikel, det innehåller riktlinjer om vad som kommer att betraktas som en dubblett.

</details>

<details><summary>Kan jag importera relationer mellan objekt?</summary>

Ja, vänligen se avsnittet **Förbered din csv** ovan i denna artikel, det innehåller ett avsnitt om import av relationer.

</details>

<details><summary>Kan jag uppdatera befintliga poster med Import-funktionen?</summary>

Ja du kan uppdatera befintliga poster med Import-funktionen. Se till att ange id (eller något annat unikt fält) när du laddar upp dina poster igen.

</details>

<details><summary>Kan jag migrera `id` från mina andra verktyg?</summary>

Ja. You need to create a field that you define as unique in your data model that will contain the `id` from your other tool(s). Observera att namnet `id` är skyddat eftersom det används för Twenty id.  
Om du vill skapa relationer mellan objekt med hjälp av detta fält, se avsnittet **Förbered din csv** ovan i denna artikel. Det innehåller ett avsnitt om import av relationer.

</details>

## Exportera Data

Du kan ladda ner data från de flesta av dina objekt och upp till 20 000 poster per export.
För att exportera data från ett objekt:

1. Besök objektsöversikten.
2. Välj vyn för dataexport. Konfigurera de kolumner du vill ladda ner genom att dölja eller lägga till kolumner och hitta de poster du behöver genom att filtrera din vy.
3. Få tillgång till sidopanelen via `⋮` ikonen uppe till höger.
4. Klicka på `Exportera vy`.
5. Välj lagringsplats för CSV-data. Observera att export kan ta tid vid ett stort antal poster.
