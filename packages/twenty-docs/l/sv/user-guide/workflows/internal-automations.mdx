Nedan följer exempel på arbetsflöden som du kan rulla ut för att automatisera repetitiva uppgifter, minska manuellt arbete och hålla dina data uppdaterade.

## Datahanteringsautomationer

### Auto-flagga personliga e-postadresser

**Problem**: Ditt team behöver identifiera personliga vs affärsmässiga e-postadresser för bättre målinriktning och efterlevnad.

**Lösning**: Skapa ett arbetsflöde som automatiskt uppdaterar ett fält för "är personlig e-post" när en e-postadress läggs till eller uppdateras.

**Trigger**: Posten uppdateras (Person objekt, E-post fält)

**Åtgärder**:

- Skapa kod för att kontrollera om domännamnet matchar vanliga personliga leverantörer (gmail.com, yahoo.com, etc.)
- Uppdatera posten för att sätta flaggan "är personlig e-post"

### Lead-assignment - Round Robin

**Problem**: Nya leads måste distribueras rättvist över ditt säljteam för att säkerställa balanserad arbetsbelastning.

**Lösning**: Tilldela nya leads automatiskt till säljarna genom ett roterande system.

**Trigger**: Posten skapas (Person objekt)

**Åtgärder**:

- Sök i poster för att hitta den senast tilldelade säljaren
- Skapa kod för att bestämma nästa säljare i rotationen
- Uppdatera posten för att tilldela leaden till vald säljare
- Skicka e-post för att meddela den tilldelade säljaren

### Lead Assignment - Territory Based

**Problem**: Leads bör tilldelas baserat på geografiska territorier eller företagskaraktäristik.

**Lösning**: Styra leads till lämplig säljare baserat på plats, företagsstorlek eller bransch.

**Trigger**: Posten skapas (Person- eller företagsobjekt)

**Åtgärder**:

- Skapa kod för att bestämma territorium baserat på plats/branschregler
- Sök i poster för att hitta territorieägaren
- Uppdatera posten för att tilldela leaden
- Skapa post för uppföljningsuppgift

## Säljprocess-automationer

### Opportunity steghantering - Stängd Vunnen

**Problem**: När affärer avslutas behövs flera manuella uppdateringar över olika poster och gruppmedlemmar.

**Lösning**: Automatiskt hantera alla aktiviteter efter seger när en möjlighet går till "Stängd Vunnen".

**Trigger**: Record is Updated (Opportunities object, Stage field = "Closed Won")

**Åtgärder**:

- Uppdatera posten för att ändra företagstyp från "Prospekt" till "Kund"
- Create Record for onboarding tasks assigned to account manager
- Skicka e-postnotifiering till kundframgångsteamet
- HTTP Request to update external billing system

### Opportunity steghantering - Stängd Förlorad Förnyelse

**Problem**: När förnyelsemöjligheter går förlorade behöver kundstatus uppdateras för korrekt kontohantering.

**Lösning**: Automatiskt uppdatera kundstatus när förnyelseavtal går förlorade.

**Trigger**: Posten uppdateras (Opportunities objekt, Stadium = "Stängd Förlorad" OCH Typ = "Förnyelse")

**Åtgärder**:

- Uppdatera posten för att ändra företagstyp från "Kund" till "Tappad Kund"
- Skapa post för analysuppgift för churn
- Skicka e-postvarning till kundframgångschefen
- Uppdatera posten för att lägga till churn datum och orsak

### Varningsmeddelanden för stående möjligheter

**Problem**: Möjligheter står utan uppdateringar, vilket får affärer att bli kalla och prognoser att bli opålitliga.

**Lösning**: Skicka automatiska varningar när möjligheter inte har uppdaterats nyligen.

**Trigger**: Enligt schema (dagligen)

**Åtgärder**:

- Sök i poster för möjligheter som inte har uppdaterats på X dagar
- Skapa kod för att formatera varningsmeddelande med möjlighetsdetaljer
- Skicka e-post till möjlighetens ägare och chef
- Skapa post för uppföljningsuppgift om ingen respons mottas

## Produktivitetsautomationer

### Veckosummering av uppgifter

**Problem**: Gruppmedlemmar tappar spår av sina kommande uppgifter och deadlines.

**Lösning**: Skicka automatiska veckomail med påminnelser och uppgiftssammanställningar.

**Trigger**: Enligt schema (varje måndag kl. 8:00)

**Åtgärder**:

- Sök i poster för uppgifter som förfaller denna vecka sorterade efter ansvarig
- Skapa kod för att formatera uppgiftslista per person
- Skicka e-post till varje teammedlem med deras uppdragssummering
- Skicka e-post till chefer med teamöversikt

### Automatisering av mötesuppföljning

**Problem**: Viktiga åtgärdspunkter från möten glöms eller försenas.

**Lösning**: Skapa automatiskt uppföljningsuppgifter när möten schemaläggs eller är slutförda.

**Trigger**: Posten skapas (Aktiviteter objekt, Typ = "Möte")

**Åtgärder**:

- Skapa post för förberedelseuppgift inför möte
- Skapa post för uppföljningsuppgift efter möte
- Skicka e-postpåminnelse till deltagarna
- Uppdatera posten för att länka uppgifter till mötet

### Synkronisering av fält över objekt

**Problem**: Du behöver information från relaterade poster enkelt tillgänglig (t.ex. huvudkontaktens e-post på möjlighetsobjekt).

**Lösning**: Synkronisera automatiskt fält mellan relaterade objekt tills inbäddade fält finns tillgängliga.

**Trigger**: Posten uppdateras (Opportunities objekt, Kontaktperson fält)

**Åtgärder**:

- Sök i poster för att hitta detaljer om länkad person
- Uppdatera posten för att kopiera e-postadress till möjlighet
- Uppdatera posten för att kopiera telefonnummer till möjlighet
- Uppdatera posten för att kopiera företagsinformation

## Datavalidering och städning

### Standardisering av telefonnummer

**Problem**: Telefonnumren anges i olika format, vilket gör dem svåra att använda för att ringa eller skicka meddelanden.

**Lösning**: Formatera automatiskt telefonnummer till ett standardformat när de matas in.

**Trigger**: Posten uppdateras (Person objekt, Telefon fält)

**Åtgärder**:

- Code action to parse and format phone number
- Uppdatera posten med standardiserat telefonformat
- Uppdatera posten för att lägga till landkod om den saknas
