## Objekthantering

<AccordionGroup>
<Accordion title="Can I reorder objects in the left navigation bar?">
Inte än. Objektordningen i navigering är för närvarande fast, men den här funktionen är planerad för en framtida version.
</Accordion>

<Accordion title="Can I hide objects from the left navigation bar?">
Alla aktiva objekt visas i navigeringen. Du kan avaktivera objekt du inte behöver under **Inställningar → Datamodell**.
</Accordion>

<Accordion title="Can I delete standard objects (People, Companies, etc.)?">
You can deactivate any standard objects but you cannot hard delete them.
</Accordion>
</AccordionGroup>

## Fältmöjligheter

<AccordionGroup>
<Accordion title="Can I create formula fields?">
Formelfält kommer i **Q1 2026**. Under tiden kan du använda arbetsflöden för att automatiskt beräkna och uppdatera fältvärden.
</Accordion>

<Accordion title="Can I have nested fields in my objects?">
Nästlade fält kommer i **Q1 2026**. För närvarande kan du använda arbetsflöden för att hämta fältvärden från relaterade objekt. Till exempel, för att visa ett företags bransch på en Person-post, skapa ett anpassat fält på Personer och använd ett arbetsflöde för att synkronisera värdet.
</Accordion>

<Accordion title="Why can't I update relation field names?">
Namn på relationsfält påverkar API-strukturen och kan inte ändras efter skapelse. Om du behöver byta namn på ett relationsfält måste du skapa ett nytt och radera det gamla.
</Accordion>

<Accordion title="Why do I need different singular and plural names?">
Our GraphQL API uses both forms for different operations:
- `createPerson` (singular) for single record actions
- `createPeople` (plural) for bulk operations

Detta skapar begränsningar när singular- och pluralformerna är desamma, men det förbättrar utvecklarupplevelsen. </Accordion>

<Accordion title="Why are some field names protected?">
Vissa fältnamn som `Typ` eller `Applikation` är reserverade för systemanvändning. Välj alternativa namn som `Kategori` eller `Klassificering` istället.
</Accordion>
</AccordionGroup>

## Avancerade Funktioner

<AccordionGroup>
<Accordion title="Can I create many-to-many relationships?">
Många-till-många relationer kommer i **Q1 2026**. För närvarande, skapa ett mellanobjekt med två 1-till-många relationer som en tillfällig lösning.
</Accordion>

<Accordion title="What are Morph Many relationships?">
Morph Many-relationer tillåter ett objekt att relatera till flera olika objekttyper. Till exempel, en Möjlighet kan relatera till antingen en Person eller ett Företag. Den här funktionen är nu tillgänglig.
</Accordion>

<Accordion title="Can I reorder fields in objects?">
Fältomordning kommer att bli tillgänglig med anpassade layouter i **Q4 2025**. För närvarande visas fält i alfabetisk ordning.
</Accordion>
</AccordionGroup>

## Åtkomst och Behörigheter

<AccordionGroup>
<Accordion title="Where can I see and edit my data model?">
You can access your Data Model under **Settings → Data Model**.
</Accordion>

<Accordion title="Why can't I see the Data Model under Settings?">
Kontakta din arbetsplatsadministratör. Tillgång till datamodellen är vanligtvis begränsad till administratörer.
</Accordion>

<Accordion title="How many custom objects or fields can I create?">
You can create as many custom objects and fields as you need - the price won't change.
</Accordion>
</AccordionGroup>
