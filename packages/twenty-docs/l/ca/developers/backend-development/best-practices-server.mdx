This document outlines the best practices you should follow when working on the backend.

## Follow a modular approach

The backend follows a modular approach, which is a fundamental principle when working with NestJS. Assegureu-vos de descompondre el vostre codi en mòduls reutilitzables per mantenir una base de codi neta i organitzada.
Cada mòdul hauria d'encapsular una funcionalitat o característica particular i tenir un abast ben definit. Aquest enfocament modular permet una clara separació de responsabilitats i elimina les complicacions innecessàries.

## Exposar serveis per utilitzar en mòduls

Sempre crea serveis que tinguin una responsabilitat clara i única, cosa que millora la llegibilitat i mantenibilitat del codi. Nomena els serveis de manera descriptiva i coherent.

You should also expose services that you want to use in other modules. Exposing services to other modules is possible through NestJS's powerful dependency injection system, and promotes loose coupling between components.

## Eviteu utilitzar el tipus `any`

Quan declareu una variable com a `any`, el comprovador de tipus de TypeScript no realitza cap verificació de tipus, cosa que fa possible assignar qualsevol tipus de valors a la variable. TypeScript utilitza la inferència de tipus per determinar el tipus de variable basat en el valor. En declarar-lo com a `any`, TypeScript ja no pot inferir el tipus. Això fa que sigui difícil detectar errors relacionats amb tipus durant el desenvolupament, cosa que condueix a errors durant l'execució i fa que el codi sigui menys mantenible, menys fiable i més difícil d'entendre per a altres.

És per això que tot hauria de tenir un tipus. Així que si creeu un nou objecte amb nom i cognom, hauríeu de crear una interfície o tipus que contingui el nom i el cognom que defineixi la forma de l'objecte que esteu manipulant.
