## Comandes útils

Aquestes comandes s'han d'executar des de la carpeta packages/twenty-server.
From any other folder you can run `npx nx {command} twenty-server` (or `npx nx run twenty-server:{command}`).

### Configuració inicial

```
npx nx database:reset twenty-server # setup the database with dev seeds
```

### Inicialitzant el servidor

```
npx nx run twenty-server:start
```

### Lint

```
npx nx run twenty-server:lint # pass --fix to fix lint errors
```

### Provar

```
npx nx run twenty-server:test:unit          # run unit tests
npx nx run twenty-server:test:integration   # run integration tests
```

Nota: pots executar `npx nx run twenty-server:test:integration:with-db-reset` si necessites reiniciar la base de dades abans d'executar les proves d'integració.

### Reiniciant la base de dades

Si vols reiniciar i preparar la base de dades, pots executar la següent comanda:

```bash
npx nx run twenty-server:database:reset
```

### Migracions

#### Per objectes en esquemes de Core/Metadata (TypeORM)

```bash
npx nx run twenty-server:typeorm migration:generate src/database/typeorm/core/migrations/nameOfYourMigration -d src/database/typeorm/core/core.datasource.ts
```

#### Per objectes de l'espai de treball

No hi ha fitxers de migracions, les migracions es generen automàticament per a cada espai de treball, es guarden a la base de dades i s'apliquen amb aquesta comanda

```bash
npx nx run twenty-server:command workspace:sync-metadata -f
```

<Warning>

Això eliminarà la base de dades i tornarà a executar les migracions i les llavors.

Assegura't de fer una còpia de seguretat de qualsevol dada que vulguis conservar abans d'executar aquesta comanda.

</Warning>

## Tecnologia utilitzada

Twenty utilitza principalment NestJS per al backend.

Prisma va ser el primer ORM que vam utilitzar. Però per tal de permetre als usuaris crear camps i objectes personalitzats, un nivell més baix tenia més sentit ja que necessitem tenir un control precís. Ara el projecte utilitza TypeORM.

Aquesta és l'aparença actual de la tecnologia utilitzada.

**Core**

- [NestJS](https://nestjs.com/)
- [TypeORM](https://typeorm.io/)
- [GraphQL Yoga](https://the-guild.dev/graphql/yoga-server)

**Base de dades**

- [Postgres](https://www.postgresql.org/)

**Integracions de tercers**

- [Sentry](https://sentry.io/welcome/) per al seguiment d'errors

**Proves**

- [Jest](https://jestjs.io/)

**Eines**

- [Yarn](https://yarnpkg.com/)
- [ESLint](https://eslint.org/)

**Desenvolupament**

- [AWS EKS](https://aws.amazon.com/eks/)
