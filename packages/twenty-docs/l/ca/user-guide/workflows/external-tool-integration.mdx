A continuació es mostren exemples de fluxos de treball que podríeu desplegar per connectar Twenty amb la resta de la vostra pila.

## Casos d'Ús d'Ingestió de Dades

### Enviaments de Formularis Web

**Problem**: You need to capture leads from website forms, landing pages, or contact forms directly into Twenty.

**Solució**: Utilitzeu activadors de webhook per crear automàticament registres a partir dels enviaments de formularis.

**Configuració**:

- Crea un flux de treball amb un activador de webhook
- Configura el webhook per esperar dades del formulari (nom, correu electrònic, empresa, etc.)
- Estableix el mètode del webhook a POST
- Defineix l'estructura esperada del cos en l'activador

**Activador**: Webhook (petició POST des del vostre formulari)

**Accions**:

- Cerca Registres per verificar si la persona/empresa ja existeix
- Branca: Si existeix → Actualitza Registre, Si no → Crea Registre
- Crea un Registre per a una tasca de seguiment assignada a un representant de vendes
- Envia notificació per correu electrònic a l'equip de vendes

### Sincronització de Dades de Producte

**Problema**: El vostre equip de vendes necessita visibilitat sobre l'ús del producte, la facturació o les dades d'adopció de funcions emmagatzemades al vostre magatzem de dades.

**Solució**: Sincronitza regularment les dades del producte a Twenty per proporcionar als venedors el context sobre els comptes.

**Activador**: En un Horari (diari o setmanal)

**Accions**:

- Sol·licitud HTTP a l'API del vostre magatzem de dades
- Acció de codi per tractar i formatar les dades
- Utilitza la funció Iterator per als següents passos
- Cerca Registres per trobar registres d'empresa coincidents
- Actualitza el Registre per afegir les mètriques d'ús del producte
- Crea un Registre per a tasques quan l'ús cau per sota del llindar

### Notes de reunió dels enregistradors de trucades

**Problema**: Els punts destacats de les trucades de vendes es perden o no es documenten correctament al CRM.

**Solució**: Crea automàticament notes i tasques a partir dels sistemes d'enregistrament de trucades.

**Activador**: Webhook (de la plataforma d'enregistrament de trucades)

**Accions**:

- Acció de codi per extreure el resum de la reunió i les tasques d'acció
- Cerca Registres per trobar l'oportunitat o contacte relacionat
- Crea un Registre per a una nota amb el resum de la reunió
- Crea un Registre per a tasques de seguiment basades en les tasques d'acció
- Envia correu electrònic als assistents amb el resum i els propers passos

### Enriquiment de Dades

**Problema**: Els vostres registres de contacte i empresa manquen d'informació demogràfica i firmogràfica important.

**Solució**: Enriqueix automàticament els registres utilitzant proveïdors de dades externs.

**Activador**: Es crea un Registre (objecte de Persones o Empreses)

**Accions**:

- Sol·licitud HTTP a l'API d'enriquiment
- Acció de codi per processar la resposta d'enriquiment
- Utilitza la funció Iterator per als següents passos
- Actualitza el Registre amb informació addicional d'empresa/contacte
- Crea un Registre per a una tasca de vendes si s'identifica un prospecte de gran valor
- Envia alertes per correu electrònic si l'enriquiment revela senyals de compra claus

## Casos d'Ús de Distribució de Dades

### Gestió de Subscriptors de Butlletins

**Problema**: Voleu enviar correus electrònics de màrqueting a segments específics de les vostres dades CRM utilitzant eines especialitzades de correu electrònic.

**Solució**: Exporteu les llistes de subscriptors a la vostra plataforma de màrqueting per correu electrònic quan sigui necessari.

**Configuració**: Creeu una vista a Twenty amb tots els destinataris del butlletí

**Desencadenador**: Llançat Manualment (quan no es selecciona cap registre)

**Accions**:

- Cerca de Registres utilitzant els criteris de vista del butlletí
- Acció de codi per formatar adreces de correu electrònic per a la vostra plataforma de correu electrònic
- Sol·licitud HTTP per afegir subscriptors a la vostra eina de màrqueting per correu electrònic
- Crear Registre per al seguiment de campanyes
- Enviar un correu electrònic de confirmació a l'equip de màrqueting

### Desencadenadors de Seqüència de Correu Electrònic

**Problema**: Voleu desencadenar seqüències de correu electrònic sofisticades basades en esdeveniments CRM utilitzant eines d'automatització de correu electrònic dedicades.

**Solució**: Envieu nous leads o clients potencials a la vostra plataforma d'automatització de correu electrònic quan es produeixen esdeveniments específics.

**Desencadenador**: S'ha creat un registre (objecte de persones amb criteris específics)

**Accions**:

- Acció de codi per determinar la seqüència de correu electrònic adequada
- Sol·licitud HTTP per afegir el contacte a la plataforma d'automatització de correu electrònic
- Actualitza el Registre per fer un seguiment de la inscripció a la seqüència
- Crear Registre per a una tasca de seguiment per supervisar el compromís

### Integració de Puntuació de Leads

**Problema**: Necessiteu una puntuació sofisticada de leads que combini dades de CRM amb senyals externs.

**Solució**: Envieu dades de leads a eines de puntuació externes o implementeu la lògica de puntuació dins dels fluxos de treball.

**Opció 1 - Eina Externa**:

**Desencadenador**: S'ha actualitzat el registre (objecte de persones)

**Accions**:

- Sol·licitud HTTP per enviar dades de leads a la plataforma de puntuació
- Acció de codi per processar la resposta de puntuació
- Actualitza el Registre amb la puntuació del lead
- Crear Registre per a una tasca de vendes si la puntuació supera el llindar

**Opció 2 - Lògica Interna**:

**Desencadenador**: S'ha actualitzat el registre (objecte de persones)

**Accions**:

- Acció de codi amb un algorisme de puntuació (mida de l'empresa, indústria, comportament)
- Actualitza el Registre amb la puntuació calculada
- Envia un correu electrònic d'alerta al representant de vendes per a leads d'alta puntuació

### Generació de Factures

**Problema**: Quan es tanquen negocis, el vostre sistema de facturació necessita ser actualitzat amb la informació del client i del negoci.

**Solució**: Envieu automàticament les dades de l'operació al vostre sistema de facturació quan es guanyen les oportunitats.

**Desencadenador**: S'ha actualitzat el Registre (objecte d'oportunitats, Estat = "Tancat Guanyat")

**Accions**:

- Search Records to get complete customer information
- Acció de codi per formatar les dades per al sistema de facturació
- Sol·licitud HTTP per crear un client a la plataforma de facturació
- Sol·licitud HTTP per generar una factura
- Actualitza el Registre per emmagatzemar la referència de la factura
- Envia un correu electrònic a l'equip financer amb els detalls de la factura

## Patrons Avançats d'Integració

### Sincronització Bidireccional

**Problema**: Cal mantenir les dades sincronitzades entre Twenty i un altre sistema en ambdues direccions.

**Solució**: Combineu fluxos de treball programats amb desencadenadors de webhook per a una sincronització en temps real.

**De Twenty a Sistema Extern**:

**Desencadenador**: S'ha actualitzat el registre (qualsevol objecte rellevant)

**Accions**:

- Sol·licitud HTTP per actualitzar el sistema extern
- Actualitzeu el Registre per fer un seguiment de l'estat de sincronització i de la marca de temps

**De Sistema Extern a Twenty**:

**Desencadenador**: Webhook (del sistema extern)

**Accions**:

- Search Records to find matching record
- Actualitzeu el Registre amb noves dades del sistema extern
- Creeu un Registre per a una tasca de resolució de conflictes si cal

### Processament de Dades en Múltiples Passes

**Problema**: Les dades de fonts externes necessiten processament complex abans de ser utilitzades a Twenty.

**Solució**: Utilitzeu accions de codi per a la transformació i validació de dades.

**Desencadenador**: Webhook o En Programa

**Accions**:

- Acció de codi per validar el format de les dades entrants
- Acció de codi per transformar l'estructura de les dades
- Acció de codi per aplicar regles i càlculs empresarials
- Cerca de Registres per verificar duplicats
- Creeu o actualitzeu el registre amb les dades processades
- Envieu un correu electrònic d'alerta si es detecten problemes de qualitat de dades

## Consells per a la Implementació

- Emmagatzemeu les claus API de manera segura a Configuració → API i Webhooks
- Utilitzeu HTTPS per a totes les crides a API externes
- Preneu cura dels límits de velocitat de l'API - utilitzeu fluxos de treball programats quan sigui possible
- Considereu les actualitzacions en lot "En Programa" quan no cal processament en temps real
- Recordeu el límit de 100 fluxos de treball concurrents per espai de treball - utilitzeu la disponibilitat "Massiva" per a desencadenadors manuals quan processa diversos registres (vegeu [Característiques del Flux de Treball](/l/ca/user-guide/workflows/workflow-features) per a detalls)
- Prova amb dades de mostra abans d'activar fluxos de treball

Per a la resolució de problemes d'integració, consulteu la nostra [Guia de Resolució de Problemes del Flux de Treball](/l/ca/user-guide/workflows/workflow-troubleshooting). Per obtenir ajuda per a la implementació d'integracions complexes, penseu en els nostres [Serveis Professionals](/l/ca/user-guide/workflows/professional-services).
