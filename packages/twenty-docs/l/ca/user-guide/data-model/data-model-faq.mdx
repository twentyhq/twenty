## Gestió d'objectes

<AccordionGroup>
<Accordion title="Can I reorder objects in the left navigation bar?">
Encara no. La ordenació d'objectes a la navegació actualment està fixa, però aquesta funció està prevista per a una futura actualització.
</Accordion>

<Accordion title="Can I hide objects from the left navigation bar?">
Tots els objectes actius apareixen a la navegació. Pots desactivar els objectes que no necessitis a **Configuració → Model de dades**.
</Accordion>

<Accordion title="Can I delete standard objects (People, Companies, etc.)?">
You can deactivate any standard objects but you cannot hard delete them.
</Accordion>
</AccordionGroup>

## Capacitats de camp

<AccordionGroup>
<Accordion title="Can I create formula fields?">
Els camps de fórmula arribaran al **T1 2026**. Mentrestant, pots utilitzar fluxos de treball per calcular i actualitzar els valors dels camps automàticament.
</Accordion>

<Accordion title="Can I have nested fields in my objects?">
Els camps imbricats arribaran al **T1 2026**. Actualment, pots utilitzar fluxos de treball per portar valors de camps d'objectes relacionats. Per exemple, per mostrar la indústria d'una empresa en un registre de Persona, crea un camp personalitzat a Persones i utilitza un flux de treball per sincronitzar el valor.
</Accordion>

<Accordion title="Why can't I update relation field names?">
Els noms dels camps de relació afecten l'estructura de l'API i no es poden canviar després de la seva creació. Si necessites canviar el nom d'un camp de relació, hauràs de crear-ne un de nou i eliminar el vell.
</Accordion>

<Accordion title="Why do I need different singular and plural names?">
Our GraphQL API uses both forms for different operations:
- `createPerson` (singular) for single record actions
- `createPeople` (plural) for bulk operations

Això crea limitacions quan les formes singulars i plurals són les mateixes, però millora l'experiència del desenvolupador. </Accordion>

<Accordion title="Why are some field names protected?">
Certs noms de camps com `Type` o `Application` estan reservats per a l'ús del sistema. Tria noms alternatius com `Categoria` o `Classificació` en lloc d'aquests.
</Accordion>
</AccordionGroup>

## Funcions avançades

<AccordionGroup>
<Accordion title="Can I create many-to-many relationships?">
Les relacions de molts a molts arribaran al **T1 2026**. Actualment, crea un objecte intermedi amb dues relacions de 1 a molts com a solució alternativa.
</Accordion>

<Accordion title="What are Morph Many relationships?">
Les relacions Morph Many permeten que un objecte es relacioni amb diversos tipus d'objectes diferents. Per exemple, una Oportunitat podria relacionar-se amb una Persona o una Empresa. Aquesta funció ja està disponible.
</Accordion>

<Accordion title="Can I reorder fields in objects?">
La reorganització de camps estarà disponible amb dissenys personalitzats al **T4 2025**. Actualment, els camps apareixen en ordre alfabètic.
</Accordion>
</AccordionGroup>

## Accés i permisos

<AccordionGroup>
<Accordion title="Where can I see and edit my data model?">
You can access your Data Model under **Settings → Data Model**.
</Accordion>

<Accordion title="Why can't I see the Data Model under Settings?">
Contacta amb l'administrador del teu espai de treball. L'accés al model de dades generalment es limita només als administradors.
</Accordion>

<Accordion title="How many custom objects or fields can I create?">
You can create as many custom objects and fields as you need - the price won't change.
</Accordion>
</AccordionGroup>
