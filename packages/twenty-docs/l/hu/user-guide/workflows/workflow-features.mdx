## Munkafolyamat-indítók

Workflows always start with a single trigger that defines when the automation should run.

### Record is Created

Starts the workflow when a new record is created in a selected object (People, Companies, Opportunities, or any custom object).

**Konfiguráció**: Válassza ki az objektumtípust az új rekordok megfigyeléséhez.

### Record is Updated

Starts the workflow when changes are made to an existing record.

**Konfiguráció**:

- Válassza ki az objektumtípust
- Opcionálisan adja meg, hogy mely mezőket figyeljük a változások érdekében

### Record is Updated or Created

Starts the workflow when a record is either created or updated in a selected object.

**Miért fontos ez**: Ez az indító különösen hasznos, mert a különböző módszerekkel létrehozott rekordok eltérően viselkednek:

- **API/CSV importok**: A rekordok azonnal minden mezővel kitöltve jönnek létre
- **Kézi létrehozás**: A rekordok először jönnek létre, majd a mezők további frissítések során kerülnek hozzáadásra

**Konfiguráció**:

- Válassza ki az objektumtípust a monitorozáshoz
- Opcionálisan adja meg, hogy mely mezőket figyeljük a változások érdekében
- A munkafolyamat mind az első létrehozáskor, mind a későbbi frissítések során indulni fog

### Record is Deleted

Starts the workflow when a record is removed from an object.

**Konfiguráció**: Válassza ki az objektumtípust a törlések megfigyeléséhez.

### Indítás manuálisan

A munkafolyamat akkor indul, amikor egy felhasználói akció indítja. Ez az indító a Cmd+K menün vagy a felső navigációs sáv egyéni gombján keresztül érhető el.

**Elérhetőségi konfiguráció**:
Válassza ki, hogy a munkafolyamat hogyan kezelje a rekord kiválasztását:

- **Globális**: Ehhez a munkafolyamathoz nem szükséges rekord. A munkafolyamat bárhonnan (bármely objektumból) indítható, és nem használ rekordokat bemenetként.

- **Egyéni**: A kiválasztott rekordok átadásra kerülnek a munkafolyamatba. Ez egy adott objektumra van konfigurálva. Több rekord kiválasztása lehetséges a munkafolyamat indítása előtt. A munkafolyamat annyiszor fog futni, ahányszor rekord lett kiválasztva.

  <Warning>
  You cannot run more than 100 workflows in parallel at any given time.
  </Warning>

- **Tömeges**: A kiválasztott rekord(ok) át lesz(nak) adva a munkafolyamatodnak. Ez egy adott objektumhoz van beállítva. Több rekord is kijelölhető, mielőtt elindítja a munkafolyamatot. A munkafolyamat egyszer fut le, és az egész rekordlistát adja meg bemenetként. Ez azt jelenti, hogy a munkafolyamatnak tartalmaznia kell egy Iterátort. Ez legjobban azok számára ideális, akik a munkafolyamat futtatásainak számát szeretnék optimalizálni/korlátozni.

**További konfiguráció**:

- Válassza ki a célobjektumot (egyéni és tömeges elérhetőséghez)
- Válasszon egy parancsikont a munkafolyamat indításához
- Állítsa be a navigációs sáv elhelyezését (Kitűzött vagy Nem Kitűzött)

**Hozzáférési módszerek**:

- Cmd+K menü a kézi munkafolyamatok keresésére és indítására
- Egyéni gomb a felső navigációs sávban (ha be van állítva)

### Ütemezett

Beállított rendszeresség szerint indítja a munkafolyamatot.

**Konfiguráció**:

- Válassza ki az időegységet (perc, óra, nap)
- Adjon meg értéket, vagy használjon egyéni cron kifejezéseket az összetett ütemezéshez

### Webhook

Indítja a munkafolyamatot, amikor egy GET vagy POST kérés érkezik egy külső szolgáltatásból.

**Konfiguráció**:

- Fogadjon egyedi webhook URL-t
- A POST kérésekhez definiálja a várt teststruktúrát
- Ha szükséges, konfigurálja az azonosítást

## Munkafolyamat Műveletek

A műveletek határozzák meg, mi történik a trigger aktiválása után. Több műveletet láncolhat össze.

### Create a Record

Új rekord hozzáadása a kiválasztott objektumhoz.

**Konfiguráció**:

- Válassza ki a célobjektumot
- Fill out the required and optional fields
- Használjon adatokat az előző lépésekből a mezők kitöltéséhez

**Kimenet**: Az újonnan létrehozott rekord adatai felhasználhatók az azt követő lépésekben.

### Rekord Frissítése

Egy meglévő rekord módosítása a kiválasztott objektumban.

**Konfiguráció**:

- Válassza ki a célobjektumot
- Válassza ki a frissítendő konkrét rekordot
- Select fields to modify and enter new values

**Kimenet**: A módosított rekord adatai felhasználhatók az azt követő lépésekben.

### Rekord Törlése

Rekord eltávolítása a kiválasztott objektumból.

**Konfiguráció**:

- Válassza ki a célobjektumot
- Válassza ki a törlendő konkrét rekordot

**Kimenet**: A törölt rekord adatai továbbra is elérhetők az azt követő lépésekben.

### Rekordok Keresése

Rekordok keresése a kiválasztott objektumban szűrőfeltételek alapján.

**Konfiguráció**:

- Válassza ki a keresendő objektumot
- Állítson be szűrőkritériumokat az eredmények szűkítéséhez
- Állítsa be a rendezést és a korlátozásokat

**Kimenet**: Azokat a rekordokat adja vissza, amelyek megfelelnek a megadott szűrőfeltételeknek, amelyeket az azt követő lépésekben használhat.

**Legjobb gyakorlat**: Használjon ágazatokat a Keresés Rekordok után "megkerült" és "nem található" forgatókönyvek kezelésére.

### Iterátor

Loopol egy korábbi lépésből visszaadott rekordok tömbjén, lehetővé téve műveletek végrehajtását minden egyes rekordon.

**Konfiguráció**:

- Select the array of records from a previous step (e.g., results from Search Records)
- Határozza meg az egyes rekordok végrehajtandó műveleteit a hurkolás során
- Állítsa be a változó nevet, hogy hivatkozzon minden rekordon az iterációban

**Example**: Search Records returns 5 people, then use Iterator to send an email to each person or update each record individually.

**Megjegyzés**: Az iterátor jelenleg béta állapotban van. Aktiválja a Beállítások → Kiadások → Lab rész alatt.

### Szűrő

Kiszűri a rekordok tömbjét megadott feltételek alapján, csak azokat a rekordokat engedve át, amelyek megfelelnek a kritériumoknak.

**Konfiguráció**:

- Válassza ki a szűrendő rekordok tömbjét
- Határozza meg a szűrőfeltételeket és kritériumokat
- Állítsa be, mely rekordok jussanak át az azt követő lépésekhez

**Kimenet**: Csak azokat a rekordokat adja vissza, amelyek megfelelnek a megadott szűrőfeltételeknek.

### E-mail küldése

Küldjön emailt a munkafolyamatából.

**Előfeltételek**: Adjon hozzá egy email fiókot a Beállítások > Fiókok alatt

**Konfiguráció**:

- Adja meg a címzett email címét
- Állítsa be a tárgysort
- Fogalmazza meg az üzenet szövegét
- Hivatkozzon az előző lépésekből származó változókra a személyre szabáshoz

**Megjegyzés**: Az email mellékletek 2026 első negyedévében lesznek elérhetők.

### Kód

Futtasson egyedi JavaScriptet a munkafolyamatában.

**Konfiguráció**:

- Írjon JavaScript kódot a szerkesztőbe
- Hozzáférés az előző lépések változóihoz
- Visszatérési változók az azt követő lépésekben való használatra
- Tesztelje a kódot közvetlenül a lépésben

**Hozzáférés**: Kezelje az API kulcsokat a Beállítások → API & Webhooks rész alatt

### Űrlap

Űrlapot kér egy munkafolyamat futtatása során, hogy felhasználói bemenetet gyűjtsön.

**Konfiguráció**:

- Definiálja a beviteli mezőket típusokkal, címkékkel és helykitöltőkkel
- Állítsa be az érvényességi szabályokat
- Adja meg az űrlap címet és leírást

**Kimenet**: Az űrlap válaszait fel lehet használni a következő lépésekben.

<Warning>
Az űrlapokat jelenleg csak manuális indításokhoz tervezték. A többi indítót (Létrehozott Rekord, Frissített stb.) tartalmazó munkafolyamatokban az űrlapok csak a munkafolyamat futási felületen keresztül érhetők el, ami nem a várt felhasználói élmény. 2026-ban lesz kiadva egy értesítési központ, hogy az automatizált munkafolyamatokban megfelelően támogassuk az űrlapokat.
</Warning>

### HTTP kérés

Kérést küld egy külső API-nak a munkafolyamat részeként.

**Konfiguráció**:

- Adja meg az API végpont URL-t
- Válasszon HTTP módszert (GET, POST, PUT, PATCH, DELETE)
- Adjon hozzá kötelező fejléceket és értékeket
- Tartalmazza a kérés testét a POST/PUT/PATCH kéréseknél
- Biztosítson egy példa választ a szerkezet előnézetéhez

## Munkafolyamat Kezelés

### Új Munkafolyamat Létrehozása

1. Kattintson a '+ Munkafolyamat Hozzáadása' gombra a kezdéshez
2. Kattintson az "Névtelen" fülre a munkafolyamat elnevezéséhez
3. Válassza és konfigurálja a munkafolyamat indítóját
4. Adjon hozzá akciókat a munkafolyamatához
5. Tesztelje és iteráljon
6. Aktiválja a munkafolyamatát (jelenleg tervezet módban) miután befejezte a szerkesztést

**Megjegyzés**: Ha nem látja a munkafolyamatok szakaszt, ez jogosultsági kérdés miatt van. Vegye fel a kapcsolatot a munkaterület adminisztrátorával, hogy hozzáférést biztosítson a munkafolyamatokhoz.

### Munkafolyamat Állapotok

- **Tervezet**: Szerkesztés alatt, még nincs közzétéve
- **Aktív**: Éles verzió, amely reagál a triggerekre
- **Deaktivált**: Korábban aktív, de kézzel leállították
- **Archivált**: Korábbi verziók, amelyeket történeti okokból megőriznek

### Munkafolyamatok Aktiválása

Kattintson az **Aktiválás** gombra, hogy közzétegye a tervezetét új verzióként. Ez lehetővé teszi a munkafolyamat számára, hogy aktiváláskor fusson, de nem hajtja azt végre azonnal.

### Munkafolyamatok Tesztelése

Tesztelje a munkafolyamatokat aktiválás előtt a következőkkel:

- Kézi indítások (amikor nincs rekord kiválasztva)
- Egyéni akciók tesztelése (különösen Kód akciók)
- Tervezet mód tesztelése, amely nem aktiválja a munkafolyamatot

### Munkafolyamat Futások

Egy **Futás** a munkafolyamat végrehajtásának feljegyzése, amely tartalmazza:

- Állapot (sikeres, sikertelen, futó)
- Kimeneti adatok minden lépésből
- Szerző és időbélyegek
- Hibajelzések, ahol alkalmazható

**Futások Megtekintése**:

- Ellenőrizze a **Futások** panel a munkafolyamat szerkesztőjében
- Nyissa meg a **Munkafolyamat Futások** nézetet a monitorozáshoz az összes munkafolyamatnál

**Teljesítmény Tip**: Rejtsd el a munkafolyamat futásokat az "Minden munkafolyamat" oldalon és más munkafolyamat oldalakon a betöltési teljesítmény javítása érdekében, mivel sok futás lelassíthatja az oldal betöltését.

### Verziótörténet

- Tekintse meg az összes verziót a **Verziók** mező alatt
- Kattintson valamelyik verzióra a részletek megtekintéséhez
- **Használja tervezetként** opció alkalmazása korábbi verziók visszaállítására
- Kezelje a tervezet konfliktusokat az felülbírálás vagy visszatérés opciókkal

## Legjobb Gyakorlatok

### Munkafolyamat Szervezés

- **Leíró Nevek**: Használjon világos, specifikus munkafolyamatneveket
- **Lépés Elnevezés**: Nevezze át a lépéseket, hogy megnevezze a funkciójukat
- **Dokumentáció**: Írjon megjegyzéseket a Kód-akciókban
- **Kategorizálás**: Csoportosítsa a kapcsolódó munkafolyamatokat logikusan
- **Egyéni Mezők**: Adj hozzá mezőket a Munkafolyamat objektumhoz az adattruktúrájában (hasonlóan más objektumokhoz), hogy szervezze és kategorizálja a munkafolyamatokat egyéni tulajdonságokkal

### Teljesítmény Optimalizáció

- **Minimize API Calls**: Batch operations when possible
- **Efficient Searches**: Use specific filter criteria
- **Error Handling**: Plan for failure scenarios
- **Sebesség Korlátozás**: Tartsa be a külső API korlátait

### Adatáramlás Kezelés

- **Branch Logic**: Use branches after Search Records
- **Variable Usage**: Leverage data from previous steps
- **Data Validation**: Validate inputs in Code actions
- **Field Mapping**: Plan data transformations carefully

### Monitorozás és Karbantartás

- **Regular Monitoring**: Check workflow runs for errors
- **Teljesítmény Felülvizsgálat**: Elemezze a végrehajtási időket és sikerességi arányokat
- **Update Management**: Test changes in draft before activation
- **Csapat Koordináció**: Dokumentálja a munkafolyamatokat a csapattagok számára

For practical examples of these features in action, see our [Internal Automations](/l/hu/user-guide/workflows/internal-automations) and [External Tool Integration](/l/hu/user-guide/workflows/external-tool-integration) guides.
