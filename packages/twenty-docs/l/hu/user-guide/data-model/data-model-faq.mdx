## Objektumkezelés

<AccordionGroup>
<Accordion title="Can I reorder objects in the left navigation bar?">
Még nem. Az objektumok sorrendje jelenleg rögzített a navigációban, de a jövőbeni verziókban tervezik ezt a funkciót.
</Accordion>

<Accordion title="Can I hide objects from the left navigation bar?">
Minden aktív objektum megjelenik a navigációban. A **Beállítások → Adatmodell** alatt kikapcsolhatja a nem szükséges objektumokat.
</Accordion>

<Accordion title="Can I delete standard objects (People, Companies, etc.)?">
You can deactivate any standard objects but you cannot hard delete them.
</Accordion>
</AccordionGroup>

## Mezőképességek

<AccordionGroup>
<Accordion title="Can I create formula fields?">
A képletmezők **2026 Q1**-ben érkeznek. Addig is, használhat munkafolyamatokat a mezőértékek automatikus kiszámolására és frissítésére.
</Accordion>

<Accordion title="Can I have nested fields in my objects?">
Az egymásba ágyazott mezők **2026 Q1**-ben érkeznek. Jelenleg, munkafolyamatokat használhat, hogy mezőértékeket hozza át kapcsolódó objektumokból. Például, hogy megjelenítsen egy cég iparágát egy Személy rekordján, hozzon létre egy egyedi mezőt a Személyeknél, és használjon munkafolyamatot az érték szinkronizálásához.
</Accordion>

<Accordion title="Why can't I update relation field names?">
A kapcsolatmező nevek hatással vannak az API struktúrájára, és a létrehozás után nem változtathatók meg. Ha át kell nevezni egy kapcsolatmezőt, újat kell létrehozni és a régit törölni.
</Accordion>

<Accordion title="Why do I need different singular and plural names?">
Our GraphQL API uses both forms for different operations:
- `createPerson` (singular) for single record actions
- `createPeople` (plural) for bulk operations

Ez korlátozásokat okoz, amikor az egyes és többes szám ugyanaz, de javítja a fejlesztői élményt. </Accordion>

<Accordion title="Why are some field names protected?">
Bizonyos mezőnevek, mint a `Típus` vagy `Alkalmazás`, rendszerhasználatra vannak fenntartva. Válasszon alternatív neveket, mint például `Kategória` vagy `Osztályozás`.
</Accordion>
</AccordionGroup>

## Fejlett funkciók

<AccordionGroup>
<Accordion title="Can I create many-to-many relationships?">
A sok-a-sokhoz kapcsolatok **2026 Q1**-ben érkeznek. Jelenleg, hozza létre egy közbenső objektumot két 1-a-sokhoz kapcsolattal megoldásként.
</Accordion>

<Accordion title="What are Morph Many relationships?">
A Morph Many kapcsolatok lehetővé teszik, hogy egy objektum több különböző objektumtípushoz kapcsolódjon. Például egy Lehetőség kapcsolódhat akár egy Személyhez, akár egy Céghez. Ez a funkció már elérhető.
</Accordion>

<Accordion title="Can I reorder fields in objects?">
A mezők átrendezése elérhető lesz egyéni elrendezések mellett **2025 Q4**-ben. Jelenleg, a mezők ábécésorrendben jelennek meg.
</Accordion>
</AccordionGroup>

## Hozzáférés és engedélyek

<AccordionGroup>
<Accordion title="Where can I see and edit my data model?">
You can access your Data Model under **Settings → Data Model**.
</Accordion>

<Accordion title="Why can't I see the Data Model under Settings?">
Vegye fel a kapcsolatot a munkaterület adminisztrátorával. Az adatmodell elérése általában csak adminisztrátorokra van korlátozva.
</Accordion>

<Accordion title="How many custom objects or fields can I create?">
You can create as many custom objects and fields as you need - the price won't change.
</Accordion>
</AccordionGroup>
