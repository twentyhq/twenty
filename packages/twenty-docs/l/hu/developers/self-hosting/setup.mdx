# Konfigurációs menedzsment

<Warning>
**First time installing?** Follow the [Docker Compose installation guide](https://docs.twenty.com/l/hu/developers/self-hosting/docker-compose) to get Twenty running, then return here for configuration.
</Warning>

A Twenty **két konfigurációs módot** kínál a különböző telepítési igények kielégítésére:

**Admin panel elérése:** Csak admin jogokkal rendelkező felhasználók (`canAccessFullAdminPanel: true`) érhetik el a konfigurációs felületet.

## 1. Admin Panel Konfiguráció (Alapértelmezett)

```bash
IS_CONFIG_VARIABLES_IN_DB_ENABLED=true  # default
```

**A legtöbb konfiguráció a felhasználói felületen** történik a telepítés után:

1. Érd el a Twenty példányodat (általában `http://localhost:3000`)
2. Navigálj a **Beállítások / Admin Panel / Konfigurációs Változók** menüpontra
3. Konfiguráld az integrációkat, emailt, tárolást és még sok mást
4. A változások azonnal érvénybe lépnek (15 másodpercen belül többkonténeres telepítés esetén)

<Warning>
**Többkonténeres telepítések:** Ha adatbázis konfigurációt használsz (`IS_CONFIG_VARIABLES_IN_DB_ENABLED=true`), mind a szerver, mind a munkás konténerek ugyanabból az adatbázisból olvasnak. Az admin panel változásai mindkettőt automatikusan érintik, megszüntetve az infrastruktúra-változókon kívül a konténerek közötti környezeti változók duplikálásának szükségességét.
</Warning>

**Mi az, amit az admin panelen keresztül konfigurálhatsz:**

- **Hitelesítés** - Google/Microsoft OAuth, jelszó beállításai
- **Email** - SMTP beállítások, sablonok, hitelesítés
- **Tárolás** - S3 konfiguráció, helyi tárolási útvonalak
- **Integrációk** - Gmail, Google Naptár, Microsoft szolgáltatások
- **Munkafolyamat & Sebességkorlátok** - Végrehajtási korlátok, API fojtás
- **És még sok más...**

![Admin Panel Konfigurációs Változók](/images/user-guide/setup/admin-panel-config-variables.png)

<Warning>
Az admin panelen minden változó leírásokkal van dokumentálva a **Beállítások → Admin Panel → Konfigurációs Változók** menüpontban.
Bizonyos infrastruktúra beállítások, mint az adatbázis kapcsolatok (`PG_DATABASE_URL`), szerver URL-ek (`SERVER_URL`), és app titkok (`APP_SECRET`) csak `.env` fájlon keresztül konfigurálhatók.

[Complete technical reference →](https://github.com/twentyhq/twenty/blob/main/packages/twenty-server/src/engine/core-modules/twenty-config/config-variables.ts) </Warning>

## 2. Csak környezeti konfiguráció

```bash
IS_CONFIG_VARIABLES_IN_DB_ENABLED=false
```

**Minden konfiguráció `.env` fájlokon keresztül kezelve:**

1. Állítsd be az `IS_CONFIG_VARIABLES_IN_DB_ENABLED=false` értéket a `.env` fájlodban
2. Add hozzá az összes konfigurációs változót a `.env` fájlodhoz
3. Indítsd újra a konténereket, hogy a változások érvénybe lépjenek
4. Az admin panel megmutatja a jelenlegi értékeket, de nem változtathatja meg őket

## Gmail & Google Naptár Integráció

### Google Cloud Projekt létrehozása

1. Lépj be a [Google Cloud konzolra](https://console.cloud.google.com/)
2. Hozz létre egy új projektet vagy válassz egy meglévőt
3. Engedélyezd ezeket a API-kat:

- [Gmail API](https://console.cloud.google.com/apis/library/gmail.googleapis.com)
- [Google Calendar API](https://console.cloud.google.com/apis/library/calendar-json.googleapis.com)
- [People API](https://console.cloud.google.com/apis/library/people.googleapis.com)

### OAuth konfigurálása

1. Lépj a [Hitelesítő adatokhoz](https://console.cloud.google.com/apis/credentials)
2. OAuth 2.0 Ügyfélazonosító létrehozása
3. Add meg ezeket a visszairányítási URI-kat:
   - `https://{your-domain}/auth/google/redirect` (for SSO)
   - `https://{your-domain}/auth/google-apis/get-access-token` (for integrations)

### Konfigurálás a Twenty-ben

1. Navigálj a **Beállítások → Admin Panel → Konfigurációs Változók** menübe
2. Keresd meg a **Google Auth** szekciót
3. Állítsd be ezeket a változókat:
   - `MESSAGING_PROVIDER_GMAIL_ENABLED=true`
   - `CALENDAR_PROVIDER_GOOGLE_ENABLED=true`
   - `AUTH_GOOGLE_CLIENT_ID={client-id}`
   - `AUTH_GOOGLE_CLIENT_SECRET={client-secret}`
   - `AUTH_GOOGLE_CALLBACK_URL=https://{your-domain}/auth/google/redirect`
   - `AUTH_GOOGLE_APIS_CALLBACK_URL=https://{your-domain}/auth/google-apis/get-access-token`

<Warning>
**Environment-only mode:** If you set `IS_CONFIG_VARIABLES_IN_DB_ENABLED=false`, add these variables to your `.env` file instead.
</Warning>

**Kötelező jogosultságok** (automatikusan konfigurálva):
[Nézd meg a releváns forráskódot](https://github.com/twentyhq/twenty/blob/main/packages/twenty-server/src/engine/core-modules/auth/utils/get-google-apis-oauth-scopes.ts#L4-L10)

- `https://www.googleapis.com/auth/calendar.events`
- `https://www.googleapis.com/auth/gmail.readonly`
- `https://www.googleapis.com/auth/profile.emails.read`

### Ha az alkalmazásod teszt módban van

Ha az alkalmazásod teszt módban van, hozzá kell adnod teszt felhasználókat a projekthez.

Az [OAuth beleegyezési képernyő](https://console.cloud.google.com/apis/credentials/consent) alatt add hozzá a teszt felhasználókat a "Teszt felhasználók" szekcióhoz.

## Microsoft 365 Integráció

<Warning>
A felhasználóknak rendelkezniük kell [Microsoft 365 Licensszel](https://admin.microsoft.com/Adminportal/Home) a Naptár és Az Üzenetkezelési API használatához. Nem fogják tudni szinkronizálni a fiókjukat a Twenty-vel enélkül.
</Warning>

### Projekt létrehozása a Microsoft Azure-ban

Projektet kell létrehoznod a [Microsoft Azure-ban](https://portal.azure.com/#view/Microsoft_AAD_IAM/AppGalleryBladeV2) és beszerezned a hitelesítő adatokat.

### API-k engedélyezése

A Microsoft Azure Konzolon engedélyezd a következő API-kat a "Jogosultságok" szekcióban:

- Microsoft Graph: Mail.ReadWrite
- Microsoft Graph: Mail.Send
- Microsoft Graph: Calendars.Read
- Microsoft Graph: User.Read
- Microsoft Graph: openid
- Microsoft Graph: email
- Microsoft Graph: profile
- Microsoft Graph: offline_access

Megjegyzés: A "Mail.ReadWrite" és a "Mail.Send" csak akkor kötelezők, ha e-maileket szeretnél küldeni a munkafolyamat akciókkal. Használhatod a "Mail.Read"-t is, ha csak e-maileket szeretnél fogadni.

### Engedélyezett visszairányítási URI-k

Hozzá kell adnod a következő visszairányítási URI-kat a projekthez:

- `https://{your-domain}/auth/microsoft/redirect` if you want to use Microsoft SSO
- `https://{your-domain}/auth/microsoft-apis/get-access-token`

### Konfigurálás a Twenty-ben

1. Navigálj a **Beállítások → Admin Panel → Konfigurációs Változók** menübe
2. Keresd meg a **Microsoft Auth** szekciót
3. Állítsd be ezeket a változókat:
   - `MESSAGING_PROVIDER_MICROSOFT_ENABLED=true`
   - `CALENDAR_PROVIDER_MICROSOFT_ENABLED=true`
   - `AUTH_MICROSOFT_ENABLED=true`
   - `AUTH_MICROSOFT_CLIENT_ID={client-id}`
   - `AUTH_MICROSOFT_CLIENT_SECRET={client-secret}`
   - `AUTH_MICROSOFT_CALLBACK_URL=https://{your-domain}/auth/microsoft/redirect`
   - `AUTH_MICROSOFT_APIS_CALLBACK_URL=https://{your-domain}/auth/microsoft-apis/get-access-token`

<Warning>
**Environment-only mode:** If you set `IS_CONFIG_VARIABLES_IN_DB_ENABLED=false`, add these variables to your `.env` file instead.
</Warning>

### Jogosultságok konfigurálása

[Nézd meg a releváns forráskódot](https://github.com/twentyhq/twenty/blob/main/packages/twenty-server/src/engine/core-modules/auth/utils/get-microsoft-apis-oauth-scopes.ts#L2-L9)

- 'openid'
- 'e-mail'
- 'profil'
- 'offline_access'
- 'Mail.ReadWrite'
- 'Mail.Send'
- 'Calendars.Read'

### Ha az alkalmazásod teszt módban van

Ha az alkalmazásod teszt módban van, hozzá kell adnod teszt felhasználókat a projekthez.

Add hozzá a teszt felhasználókat a "Felhasználók és csoportok" szekcióban.

## Háttérfeladatok Naptár & Üzenetkezelés számára

A Gmail, Google Naptár vagy Microsoft 365 integrációk konfigurálása után el kell indítanod a háttérfeladatokat, amelyek szinkronizálják az adatokat.

Regisztráld a következő ismétlődő feladatokat a munkás konténeredben:

```bash
# from your worker container
yarn command:prod cron:messaging:messages-import
yarn command:prod cron:messaging:message-list-fetch
yarn command:prod cron:calendar:calendar-event-list-fetch
yarn command:prod cron:calendar:calendar-events-import
yarn command:prod cron:messaging:ongoing-stale
yarn command:prod cron:calendar:ongoing-stale
yarn command:prod cron:workflow:automated-cron-trigger
```

## Email konfiguráció

1. Navigálj a **Beállítások → Admin Panel → Konfigurációs Változók** menübe
2. Keresd meg az **Email** szekciót
3. Konfiguráld az SMTP beállításaidat:

<ArticleTabs label1="Gmail" label2="Office365" label3="Smtp4dev">

  <ArticleTab>

Szükséged lesz egy [Alkalmazásjelszóra](https://support.google.com/accounts/answer/185833).

- EMAIL_DRIVER=smtp
- EMAIL_SMTP_HOST=smtp.gmail.com
- EMAIL_SMTP_PORT=465
- EMAIL_SMTP_USER=gmail_email_address
- EMAIL_SMTP_PASSWORD='gmail_app_password'

  </ArticleTab>

  <ArticleTab>

**smtp4dev** egy hamis SMTP email szerver fejlesztéshez és teszteléshez.

- EMAIL_DRIVER=smtp
- EMAIL_SMTP_HOST=smtp.office365.com
- EMAIL_SMTP_PORT=587
- EMAIL_SMTP_USER=office365_email_address
- EMAIL_SMTP_PASSWORD='office365_password'

  </ArticleTab>

  <ArticleTab>

**smtp4dev** egy hamis SMTP email szerver fejlesztéshez és teszteléshez.

- Futtasd az smtp4dev képet: `docker run --rm -it -p 8090:80 -p 2525:25 rnwood/smtp4dev`
- Érdd el az smtp4dev felületét itt: [http://localhost:8090](http://localhost:8090)
- Állítsd be a következő változókat:
  - EMAIL_DRIVER=smtp
  - EMAIL_SMTP_HOST=localhost
  - EMAIL_SMTP_PORT=2525

  </ArticleTab>

</ArticleTabs>
