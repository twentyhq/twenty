## Áttekintés

Ez az útmutató lépésről lépésre ismerteti a Twenty alkalmazás telepítését és konfigurálását a Docker Compose használatával. The aim is to make the process straightforward and prevent common pitfalls that could break your setup.

**Fontos:** Csak a kifejezetten ebben az útmutatóban említett beállításokat módosítsa. Más konfigurációk módosítása problémákat okozhat.

See docs [Setup Environment Variables](https://docs.twenty.com/l/hu/developers/self-hosting/setup) for advanced configuration. All environment variables must be declared in the docker-compose.yml file at the server and / or worker level depending on the variable.

## Rendszerkövetelmények

- RAM: Ellenőrizze, hogy a környezete legalább 2GB RAM-mal rendelkezik-e. Ha a memória nem elegendő, a folyamatok összeomolhatnak.
- Docker & Docker Compose: Make sure both are installed and up-to-date.

## 1. opció: Egyvonalas szkript

Telepítse a Twenty legújabb stabil verzióját egyetlen paranccsal:

```bash
bash <(curl -sL https://raw.githubusercontent.com/twentyhq/twenty/main/packages/twenty-docker/scripts/install.sh)
```

Egy adott verzió vagy ág telepítése:

```bash
VERSION=vx.y.z BRANCH=branch-name bash <(curl -sL https://raw.githubusercontent.com/twentyhq/twenty/main/packages/twenty-docker/scripts/install.sh)
```

- Cserélje le x.y.z-t a kívánt verziószámra.
- Cserélje le branch-name-t arra az ágra, amelyet telepíteni szeretne.

## 2. opció: Kézi lépések

Kövesse ezeket a lépéseket a kézi telepítéshez.

### Step 1: Set Up the Environment File

1. **Create the .env File**

   Copy the example environment file to a new .env file in your working directory:

   ```bash
   curl -o .env https://raw.githubusercontent.com/twentyhq/twenty/refs/heads/main/packages/twenty-docker/.env.example
   ```

2. **Titkos tokenek generálása**

   Futtassa a következő parancsot egy egyedi véletlen karakterlánc generálásához:

   ```bash
   openssl rand -base64 32
   ```

   **Fontos:** Tartsa ezt az értéket titokban / ne ossza meg.

3. **Update the `.env`**

   Replace the placeholder value in your .env file with the generated token:

   ```ini
   APP_SECRET=first_random_string
   ```

4. **Set the Postgres Password**

   Frissítse a `PG_DATABASE_PASSWORD` értéket a .env fájlban erős jelszóval különleges karakterek nélkül.

   ```ini
   PG_DATABASE_PASSWORD=my_strong_password
   ```

### 2) lépés: Szerezze be a Docker Compose fájlt

Töltse le a `docker-compose.yml` fájlt a munkakönyvtárába:

```bash
curl -o docker-compose.yml https://raw.githubusercontent.com/twentyhq/twenty/refs/heads/main/packages/twenty-docker/docker-compose.yml
```

### 3. lépés: Alkalmazás elindítása

Start the Docker containers:

```bash
docker compose up -d
```

### 4. lépés: Alkalmazás elérése

If you host twentyCRM on your own computer, open your browser and navigate to [http://localhost:3000](http://localhost:3000).

Ha egy szerveren üzemelteti, ellenőrizze, hogy a szerver fut-e, és minden rendben van-e

```bash
curl http://localhost:3000
```

## Konfiguráció

### Tegye a Twenty külső hozzáférésűvé

Alapértelmezés szerint a Twenty a `localhost`-on fut a `3000` porton. Külső domain vagy IP cím eléréséhez konfigurálnia kell a `SERVER_URL`-t az `.env` fájlban.

#### `SERVER_URL` megértése

- **Protokoll:** Használja a `http` vagy `https`-t a beállításától függően.
  - Használja a `http`-t, ha nem állított be SSL-t.
  - Használja a `https`-t, ha SSL-t állított be.
- **Domain/IP:** Ez az a domain név vagy IP cím, ahol az alkalmazás elérhető.
- **Port:** Adja meg a portszámot, ha nem alapértelmezett portokat használ (`80` a `http`, `443` a `https` esetén).

### SSL követelmények

SSL (HTTPS) bizonyos böngészői funkciók zavartalan működéséhez szükséges. Miközben ezek a funkciók működhetnek helyi fejlesztés során (mivel a böngészők a localhost-ot másként kezelik), egy megfelelő SSL beállítás szükséges, ha a Twentyt rendszeres domainen szeretné üzemeltetni.

Például a vágólap API biztonságos kontextust igényelhet - egyes funkciók, mint például a másolás gombok az alkalmazásban, nem működhetnek HTTPS nélkül.

Erősen javasoljuk, hogy a Twentyt egy fordított proxy mögött SSL terminációval állítsa be a biztonság és funkcionalitás érdekében.

#### `SERVER_URL` konfigurálása

1. **Határozza meg a hozzáférési URL-t**
   - **Fordított proxy nélkül (közvetlen hozzáférés):**

     Ha közvetlenül fér hozzá az alkalmazáshoz fordított proxy nélkül:

     ```ini
     SERVER_URL=http://your-domain-or-ip:3000
     ```

   - **Fordított proxyval (standard portok):**

     Ha fordított proxyval, mint az Nginx vagy Traefik és SSL-lel konfigurál:

     ```ini
     SERVER_URL=https://your-domain-or-ip
     ```

   - **Fordított proxyval (egyedi portok):**

     Ha nem szabványos portokat használ:

     ```ini
     SERVER_URL=https://your-domain-or-ip:custom-port
     ```

2. **Frissítse az `.env` fájlt**

   Nyissa meg az `.env` fájlt, és frissítse a `SERVER_URL`-t:

   ```ini
   SERVER_URL=http(s)://your-domain-or-ip:your-port
   ```

   **Példák:**

   - Közvetlen hozzáférés SSL nélkül:
     ```ini
     SERVER_URL=http://123.45.67.89:3000
     ```
   - Hozzáférés domainen keresztül SSL-lel:
     ```ini
     SERVER_URL=https://mytwentyapp.com
     ```

3. **Az alkalmazás újraindítása**

   A változások érvénybe lépéséhez indítsa újra a Docker konténereket:

   ```bash
   docker compose down
   docker compose up -d
   ```

#### Megfontolások

- **Fordított Proxy Konfiguráció:**

  Győződjön meg róla, hogy a fordított proxy a kéréseket a megfelelő belső portra továbbítja (alapértelmezés szerint `3000`). Konfigurálja az SSL megszakítást és az esetleg szükséges fejléceket.

- **Tűzfal Beállításai:**

  Nyissa meg a szükséges portokat a tűzfalán, hogy külső hozzáférést engedélyezzen.

- **Konzisztencia:**

  A `SERVER_URL`-nek meg kell egyeznie azzal, ahogyan a felhasználók az alkalmazást elérik a böngészőjükben.

#### Tartósság

- **Data Volumes:**

  The Docker Compose configuration uses volumes to persist data for the database and server storage.

- **Stateless Environments:**

  Ha egy állapotmentes környezetbe telepít (pl. bizonyos felhőszolgáltatások), konfigurálja a külső tárolót az adatok megőrzésére.
