La **capa de operaciones** es todo lo que haces *a* tu aplicación en lugar de *con* ella: invocar comandos de CLI, ejecutar pruebas de integración contra un servidor Twenty real, configurar CI y enviar versiones, ya sea como un tarball implementado en un único servidor o como un paquete de npm listado en el marketplace.

```text
   develop ─▶ test ─▶ build ─▶ deploy / publish
   ───────    ────    ─────    ─────────────────
   yarn       yarn    yarn     yarn twenty app:publish --private  (tarball → one server)
   twenty     test    twenty
   dev                dev:build    yarn twenty app:publish         (npm → marketplace)
```
