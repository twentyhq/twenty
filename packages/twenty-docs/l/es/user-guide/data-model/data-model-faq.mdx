## Gestión de Objetos

<AccordionGroup>
<Accordion title="Can I reorder objects in the left navigation bar?">
Aún no. La ordenación de objetos en la navegación está actualmente fijada, pero esta función está prevista para una futura versión.
</Accordion>

<Accordion title="Can I hide objects from the left navigation bar?">
Todos los objetos activos aparecen en la navegación. Puedes desactivar los objetos que no necesites en **Configuración → Modelo de Datos**.
</Accordion>

<Accordion title="Can I delete standard objects (People, Companies, etc.)?">
You can deactivate any standard objects but you cannot hard delete them.
</Accordion>
</AccordionGroup>

## Capacidades de Campos

<AccordionGroup>
<Accordion title="Can I create formula fields?">
Los campos de fórmula llegarán en **Q1 2026**. Mientras tanto, puedes usar flujos de trabajo para calcular y actualizar automáticamente los valores de los campos.
</Accordion>

<Accordion title="Can I have nested fields in my objects?">
Los campos anidados llegarán en **Q1 2026**. Actualmente, puedes usar flujos de trabajo para traer valores de campo de objetos relacionados. Por ejemplo, para mostrar la industria de una empresa en un registro de Persona, crea un campo personalizado en Personas y usa un flujo de trabajo para sincronizar el valor.
</Accordion>

<Accordion title="Why can't I update relation field names?">
Los nombres de los campos de relación impactan la estructura de la API y no se pueden cambiar después de su creación. Si necesitas renombrar un campo de relación, tendrás que crear uno nuevo y eliminar el antiguo.
</Accordion>

<Accordion title="Why do I need different singular and plural names?">
Our GraphQL API uses both forms for different operations:
- `createPerson` (singular) for single record actions
- `createPeople` (plural) for bulk operations

Esto crea limitaciones cuando las formas singulares y plurales son iguales, pero mejora la experiencia del desarrollador. </Accordion>

<Accordion title="Why are some field names protected?">
Ciertos nombres de campo como `Tipo` o `Aplicación` están reservados para uso del sistema. Elige nombres alternativos como `Categoría` o `Clasificación`.
</Accordion>
</AccordionGroup>

## Características Avanzadas

<AccordionGroup>
<Accordion title="Can I create many-to-many relationships?">
Las relaciones de muchos a muchos llegarán en **Q1 2026**. Actualmente, crea un objeto intermedio con dos relaciones de 1 a muchos como solución alternativa.
</Accordion>

<Accordion title="What are Morph Many relationships?">
Las relaciones Morph Many permiten que un objeto se relacione con múltiples tipos de objetos diferentes. Por ejemplo, una Oportunidad podría relacionarse con una Persona o una Empresa. Esta función ya está disponible.
</Accordion>

<Accordion title="Can I reorder fields in objects?">
El reordenamiento de campos estará disponible con diseños personalizados en **Q4 2025**. Actualmente, los campos aparecen en orden alfabético.
</Accordion>
</AccordionGroup>

## Acceso y Permisos

<AccordionGroup>
<Accordion title="Where can I see and edit my data model?">
You can access your Data Model under **Settings → Data Model**.
</Accordion>

<Accordion title="Why can't I see the Data Model under Settings?">
Comunícate con tu administrador de espacio de trabajo. El acceso al modelo de datos suele estar restringido solo a administradores.
</Accordion>

<Accordion title="How many custom objects or fields can I create?">
You can create as many custom objects and fields as you need - the price won't change.
</Accordion>
</AccordionGroup>
