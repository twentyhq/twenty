Todo en Twenty se basa en **objetos** y **campos**, los componentes básicos de tu modelo de datos.

## Objetos

Los objetos son las tablas que contienen tus datos. Twenty incluye objetos estándar de forma predeterminada:

* **Companies** — Organizaciones con las que haces negocios
* **People** — Contactos individuales
* **Opportunities** — Acuerdos en tu flujo de ventas
* **Tasks** — Elementos de acción para tu equipo
* **Notes** — Texto de formato libre vinculado a registros

También puedes crear **objetos personalizados** para todo lo que tu negocio necesite: proyectos, tickets de soporte, productos, contratos o cualquier otra cosa.

## Campos

Los campos son las propiedades de cada objeto. Twenty admite una amplia variedad de tipos de campo:

| Categoría     | Tipos                                                                                                       |
| ------------- | ----------------------------------------------------------------------------------------------------------- |
| **Básico**    | Texto, Número, Booleano, Fecha, Moneda, Valoración, Selección                                               |
| **Compuesto** | Dirección (calle, ciudad, estado, código postal), Nombre completo, Enlaces, Teléfonos, Correos electrónicos |
| **Especial**  | Relación, Archivo adjunto, JSON, Actor (quién creó/modificó)                                                |

Cada objeto también obtiene campos de sistema automáticos: `id`, `createdAt`, `updatedAt`, `createdBy` y `position`.

## Relaciones

Los objetos se conectan entre sí mediante relaciones. Una Company tiene muchas People, una Opportunity pertenece a una Company, y así sucesivamente. Puedes crear relaciones personalizadas entre cualquier objeto, incluidas relaciones de muchos a muchos.

## Por qué esto es tan potente

A diferencia de los CRM tradicionales, donde estás limitado a campos predefinidos en objetos predefinidos, Twenty te permite modelar tus datos exactamente de la misma manera en que funciona tu negocio. Los objetos personalizados reciben el mismo tratamiento de primera clase que los incorporados, incluidos los endpoints de API, las vistas, los permisos y los disparadores de flujos de trabajo.
