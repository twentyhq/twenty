## Поддерживаемые объекты

Активность электронной почты в настоящее время доступна на **трёх стандартных объектах**:

| Объект          | Что вы увидите                                                                                 |
| --------------- | ---------------------------------------------------------------------------------------------- |
| **Люди**        | Все электронные письма, которыми обменивались с этим контактом                                 |
| **Компании**    | Все электронные письма с любым контактом из этой компании (на основе домена электронной почты) |
| **Возможности** | Электронные письма, связанные с компанией, привязанной к возможности                           |

## Почему только эти объекты?

Люди, Компании и Возможности — это ключевые объекты отношений, где контекст электронной почты даёт наибольшую ценность. Цепочки писем автоматически связываются на основании:

* **Адрес электронной почты** → сопоставляется с записями Людей
* **Домен электронной почты** → сопоставляется с записями Компаний
* **Связь с компанией** → сопоставляется с Возможностями

## Пользовательские объекты

**Отслеживание электронной почты недоступно на пользовательских объектах** в настоящее время.

Если вам нужен контекст электронной почты в пользовательском объекте, рассмотрите:

* Использование поля связи для привязки вашего пользовательского объекта к Людям или Компаниям
* Просмотр истории электронной почты в связанной записи человека/компании

## Планы на будущее

Рассматривается расширение видимости электронной почты на пользовательские объекты. Поделитесь своим сценарием использования в наших [обсуждениях на GitHub](https://github.com/twentyhq/twenty/discussions), чтобы помочь расставить приоритеты для этой функции.
