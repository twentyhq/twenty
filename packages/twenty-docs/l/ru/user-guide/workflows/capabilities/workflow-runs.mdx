## О запусках

**Запуск** — это запись выполнения рабочего процесса. Каждый раз, когда срабатывает триггер рабочего процесса — будь то событие записи, расписание, ручное действие или вебхук — создаётся новый запуск.

## Просмотр запусков

### В редакторе рабочего процесса

1. Откройте рабочий процесс, который хотите отслеживать
2. Нажмите на панель **Запуски** справа
3. Посмотрите список последних запусков со статусами

### В разделе «Запуски рабочего процесса»

1. Перейдите в **Запуски рабочего процесса** в боковой панели
2. Просматривайте запуски по всем рабочим процессам
3. Фильтруйте по статусу, рабочему процессу или дате

## Статусы запусков

| Статус          | Описание                                                                                 |
| --------------- | ---------------------------------------------------------------------------------------- |
| **Выполняется** | Рабочий процесс выполняется                                                              |
| **Завершено**   | Рабочий процесс успешно завершён                                                         |
| **Не удалось**  | Рабочий процесс столкнулся с ошибкой и остановился                                       |
| **Ожидание**    | Рабочий процесс приостановлен (например, ожидает действия «Задержка» или отправки формы) |

## Сведения о запуске

Нажмите на любой запуск, чтобы просмотреть:

* **Статус**: текущее состояние запуска
* **Начато**: время начала запуска
* **Длительность**: сколько времени занял запуск
* **Данные триггера**: входные данные, запустившие рабочий процесс
* **Результаты шагов**: данные, возвращённые каждым шагом
* **Сообщения об ошибках**: если запуск не удался, что пошло не так

## Пошаговое выполнение

Каждый запуск показывает прохождение по вашему рабочему процессу:

1. Смотрите, какие шаги завершились успешно
2. Определите, где произошли сбои
3. Просматривайте данные, передаваемые между шагами
4. Отлаживайте проблемы, изучая входные и выходные данные шагов

## Обработка ошибок

Если запуск не удался:

1. Откройте запуск со сбоем
2. Найдите шаг, вызвавший сбой
3. Посмотрите подробности в сообщении об ошибке
4. Распространённые проблемы:
   * Отсутствуют обязательные поля
   * Недопустимый формат данных
   * Ошибки внешнего API
   * Проблемы с правами доступа

## Повторный запуск рабочих процессов

Если запуск не удался, вы можете:

* Исправьте основную проблему и дождитесь следующего срабатывания триггера
* Для ручных рабочих процессов запустите снова с теми же или обновлёнными данными
* Проверьте логику рабочего процесса, чтобы предотвратить будущие сбои

## Рекомендации по производительности

### Управление историей запусков

* Запуски сохраняются для истории
* Очень старые запуски могут архивироваться автоматически
* Экспортируйте данные запусков, если нужно сохранить их для учёта

### Лучшие практики мониторинга

* Регулярно проверяйте запуски после активации новых рабочих процессов
* Анализируйте неудавшиеся запуски, чтобы выявить закономерности
