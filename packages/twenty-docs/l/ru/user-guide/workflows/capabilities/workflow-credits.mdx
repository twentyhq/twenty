Кредиты рабочих процессов обеспечивают работу ваших автоматизаций в Twenty. Понимание принципов их работы помогает оптимизировать затраты и эффективно управлять бюджетом автоматизации.

## Распределение кредитов

Кредиты рабочих процессов начисляются по вашему платёжному циклу, а не по уровню плана:

| Платёжный цикл           | Кредиты                      |
| ------------------------ | ---------------------------- |
| **Ежемесячная подписка** | 5 миллионов кредитов в месяц |
| **Годовая подписка**     | 50 миллионов кредитов в год  |

<Note>
  5 миллионов кредитов в месяц — щедрый лимит для стандартных автоматизаций. Большинство команд не превысят этот лимит при типичном использовании рабочих процессов. Дополнительные кредиты в основном нужны для продвинутых действий Code и рабочих процессов на базе ИИ.
</Note>

## Как работает потребление кредитов

Кредиты расходуются при выполнении рабочих процессов, а не при их создании. Каждое действие рабочего процесса потребляет кредиты в зависимости от своей сложности:

### Потребление кредитов по типу действия

* **Основные внутренние операции**: Очень низкое потребление кредитов
  * Поиск записей
  * Создать запись
  * Обновление записи
  * Удаление записи
  * Действия с формой

* **Сложные операции**: Более высокое потребление кредитов
  * Действия с кодом (выполнение JavaScript)
  * HTTP-запросы к внешним сервисам

* **Функции ИИ**: Более высокое потребление кредитов
  * Действия ИИ-агента потребляют кредиты в зависимости от используемой модели ИИ
  * Более сложные запросы и более длинные ответы расходуют больше кредитов

* **Действия Delay**: Минимальное потребление кредитов
  * Узел Delay потребляет **1 кредит** при выполнении
  * **Во время ожидания кредиты не потребляются**
  * Задержка на 5 минут стоит столько же, сколько задержка на 5 дней

### Списание в реальном времени

Кредиты списываются в режиме реального времени при выполнении рабочих процессов. Это означает, что:

* Черновые рабочие процессы не потребляют кредиты
* Только активные, выполняемые рабочие процессы используют ваш кредитный лимит
* Неудавшиеся рабочие процессы все равно потребляют кредиты за выполненные шаги

## Управление кредитами

### Проверить использование кредитов

1. Перейдите в **Настройки → Оплата**
2. Просмотрите текущее потребление кредитов и оставшийся баланс
3. Отслеживайте шаблоны использования, чтобы оптимизировать ваши рабочие процессы

### Приобретение дополнительных кредитов

Если вам нужно больше кредитов, чем в вашем лимите плана:

1. Перейдите в **Настройки → Оплата**
2. Нажмите на опцию, чтобы приобрести дополнительные кредиты. Доступны пакеты разного размера.
3. Кредиты добавляются к вашему текущему балансу

## Лучшие практики

* **Пакетная обработка**: Используйте массовые операции и действия с Итератором эффективно
* **Оптимизация ручного запуска**: Для ручных триггеров выберите вариант `Bulk`, чтобы обработать несколько записей за один запуск рабочего процесса
* Оптимизируйте действия с кодом для повышения эффективности
* Пакетные операции для снижения числа отдельных вызовов действий
