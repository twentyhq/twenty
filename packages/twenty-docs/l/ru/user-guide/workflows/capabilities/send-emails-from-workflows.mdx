Автоматически отправляйте письма при наступлении определённых событий в вашей CRM — приветствуйте новых контактов, делайте последующие касания по возможностям или уведомляйте членов команды.

## Требования

Прежде чем отправлять письма из рабочих процессов:

1. Подключите учетную запись электронной почты в разделе **Настройки → Учетные записи**
2. Убедитесь, что для учетной записи включены разрешения на отправку

## Базовый рабочий процесс электронной почты

### Пример: приветственное письмо для новых контактов

**Цель**: отправлять приветственное письмо, когда в CRM добавляется новый контакт.

**Настройка**:

1. **Создайте рабочий процесс**: перейдите в **Настройки → Рабочие процессы** и нажмите **+ Новый рабочий процесс**

2. **Добавьте триггер**: выберите **Запись создана** → **Люди**

3. **Добавьте действие Отправить письмо**:
   * Нажмите **+**, чтобы добавить действие
   * Выберите **Отправить письмо**
   * Настройте письмо:

| Поле           | Значение                                   |
| -------------- | ------------------------------------------ |
| **Кому**       | `{{trigger.object.email}}`                 |
| **Тема**       | `Добро пожаловать в {{Your Company Name}}` |
| **Содержимое** | `Hi {{trigger.object.firstName}}, ...`     |

4. **Протестируйте и активируйте**: протестируйте на тестовой записи, затем активируйте

## Использование переменных в письмах

Ссылайтесь на данные из предыдущих шагов, используя синтаксис `{{variable}}`:

```text
Hi {{trigger.object.firstName}},

Thank you for connecting with us!

Your company, {{trigger.object.company.name}}, is now in our system.

Best regards,
The Team
```

### Доступные переменные из триггеров

| Тип триггера                 | Общие переменные                       |
| ---------------------------- | -------------------------------------- |
| **Запись создана/обновлена** | `{{trigger.object.fieldName}}`         |
| **Вручную**                  | `{{trigger.selectedRecord.fieldName}}` |
| **Вебхук**                   | `{{trigger.body.fieldName}}`           |

## Продвинутое: условные письма

### Пример: разные письма в зависимости от источника лида

**Цель**: отправлять разные приветственные письма в зависимости от того, откуда пришёл лид.

**Настройка**:

1. **Триггер**: запись создана (Люди)

2. **Добавьте действие Фильтр**:
   * Условие: `{{trigger.object.source}}` равно `"Website"`
   * Если верно → перейти к приветственному письму для сайта

3. **Ветка для других источников**:
   * Создайте параллельные ветки для разных источников
   * В каждой ветке есть собственное действие Отправить письмо

## Отправка писем нескольким получателям

### Пример: уведомление команды при закрытии сделки

**Цель**: отправить письмо менеджеру по продажам и его руководителю, когда возможность помечена как выигранная.

**Настройка**:

1. **Триггер**: запись обновлена (Возможности, стадия = "Закрыта выиграна")

2. **Поиск записей**: найдите руководителя владельца возможности

3. **Отправить письмо №1**: владельцу возможности
   * Кому: `{{trigger.object.owner.email}}`
   * Тема: `Congratulations on closing {{trigger.object.name}}!`

4. **Отправить письмо №2**: руководителю
   * Кому: `{{searchRecords.manager.email}}`
   * Тема: `Deal Won: {{trigger.object.name}}`

## Запланированные последующие письма

### Пример: последующее письмо через 3 дня после встречи

**Цель**: отправить последующее письмо через 3 дня после регистрации встречи.

**Настройка**:

1. **Триггер**: запись создана (Активности, тип = "Встреча")

2. **Действие Задержка**: подождите 3 дня

3. **Отправить письмо**:
   * Кому: участнику встречи
   * Тема: Продолжение нашего разговора
   * Содержимое: укажите детали встречи из триггера

## Лучшие практики

### Содержимое письма

* Делайте темы писем краткими и по существу
* Персонализируйте, используя имя получателя
* Добавляйте чёткий призыв к действию
* Тестируйте письма перед активацией

### Доставляемость

* Не отправляйте слишком много писем за короткое время
* Используйте профессиональные подписи в письмах
* Избегайте слов, которые запускают спам-фильтры
* Обеспечьте наличие опции отписки для маркетинговых писем

### Устранение неполадок

* Убедитесь, что учетная запись электронной почты подключена и активна
* Проверьте, что адрес электронной почты получателя действителен
* Проверьте запуски рабочего процесса на наличие сообщений об ошибках
* Сначала протестируйте на собственном адресе электронной почты

<Note>
  **Скоро**: вложения к письмам будут доступны в первом квартале 2026 года.
</Note>
