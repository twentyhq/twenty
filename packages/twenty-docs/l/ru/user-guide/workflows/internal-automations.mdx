Below are workflow examples you could roll out to automate repetitive tasks to reduce manual work and keep your data up-to-date.

## Автоматизация управления данными

### Автоматическая отметка личных электронных адресов

**Проблема**: Вашей команде необходимо различать личные и рабочие электронные адреса для более точного таргетинга и соблюдения регламентов.

**Решение**: Создайте рабочий процесс, который автоматически обновляет поле "это личная электронная почта" каждый раз, когда электронный адрес добавляется или обновляется.

**Триггер**: Запись обновлена ​​(Объект Люди, Поле Email)

**Действия**:

- Code action to check if email domain matches common personal providers (gmail.com, yahoo.com, etc.)
- Обновить запись для установки флага "это личная электронная почта"

### Lead Assignment - Round Robin

**Проблема**: Новые лиды должны распределяться справедливо среди вашей команды продаж, чтобы обеспечить равномерную нагрузку.

**Solution**: Automatically assign new leads to sales reps using a round-robin system.

**Триггер**: Запись создана (Объект Люди)

**Действия**:

- Поиск записей, чтобы найти последнего назначенного менеджера по продажам
- Code action to determine next rep in rotation
- Обновить запись, чтобы назначить лид выбранному менеджеру по продажам
- Отправить электронное сообщение для уведомления назначенного менеджера

### Назначение лидов - Основанное на территории

**Проблема**: Лиды должны назначаться на основе географических территорий или характеристик компании.

**Solution**: Route leads to the appropriate sales rep based on location, company size, or industry.

**Триггер**: Запись создана (Объекты Люди или Компании)

**Действия**:

- Кодировать действие для определения территории на основе правил местоположения/отрасли
- Поиск записей, чтобы найти владельца территории
- Обновить запись, чтобы назначить лид
- Создать запись для задачи последующих действий

## Автоматизация процессов продаж

### Управление этапами возможности - Закрыто выиграно

**Проблема**: Когда сделки заключаются, требуется множество ручных обновлений через различные записи и команды.

**Решение**: Автоматически обрабатывать все действия после выигрыша, когда возможность переходит в состояние "Закрыто выиграно".

**Триггер**: Запись обновляется (Объект Возможности, Поле Этап = "Закрыто выиграно")

**Действия**:

- Обновить запись, чтобы изменить тип компании с "Перспективной" на "Клиента"
- Создать запись для задач по адаптации, назначенных менеджеру по аккаунтам
- Отправить уведомление по электронной почте в команду по успеху клиентов
- HTTP-запрос для обновления внешней системы биллинга

### Управление этапами возможности - Закрыто проиграно, Обновление

**Проблема**: Когда обновления не удаются, статус клиента нужно обновить для правильного управления аккаунтом.

**Решение**: Автоматически обновлять статус клиента, когда обновления потеряны.

**Триггер**: Запись обновлена (Объект Возможности, Этап = "Закрыто проиграно" И Тип = "Обновление")

**Действия**:

- Обновить запись, чтобы изменить тип компании с "Клиента" на "Потерявшего клиента"
- Создать запись для задачи анализа потерь
- Отправить предупреждение по электронной почте менеджеру по успеху клиентов
- Обновить запись, чтобы добавить дату и причину потери

### Оповещения о застойной возможности

**Problem**: Opportunities sit without updates, causing deals to go cold and forecasts to become unreliable.

**Решение**: Автоматически отправлять оповещения, когда возможности не обновлялись в последнее время.

**Триггер**: По расписанию (ежедневно)

**Действия**:

- Поиск записей для возможностей, давно не обновлявшихся
- Кодировать действие для форматирования сообщения с подробностями возможности
- Отправить электронное письмо владельцу возможности и менеджеру
- Создать запись для задачи последующих действий при отсутствии ответа

## Productivity Automations

### Еженедельный отчёт о задачах

**Проблема**: Члены команды теряют из виду свои предстоящие задачи и сроки.

**Решение**: Отправлять автоматические еженедельные электронные напоминания с кратким обзором задач.

**Триггер**: По расписанию (каждый понедельник в 8 утра)

**Действия**:

- Поиск записей для задач на текущую неделю по ответственному
- Кодировать действие для форматирования списка задач по лицу, отвечающему за их выполнение
- Отправить электронное письмо каждому члену команды с кратким обзором их задач
- Отправить электронное письмо менеджерам с обзором команды

### Автоматизация последующего наблюдения за собраниями

**Проблема**: Важные действия, вытекающие из встреч, забываются или откладываются.

**Решение**: Автоматически создавать задачи для последующих действий при планировании или завершении встреч.

**Триггер**: Запись создана (Объект Деятельности, Тип = "Встреча")

**Действия**:

- Создать запись для задачи подготовки к встрече
- Создать запись для задачи последующих действий после встречи
- Отправить напоминание по электронной почте участникам встречи
- Обновить запись, чтобы связать задачи со встречей

### Синхронизация полей между объектами

**Проблема**: Необходима лёгкая доступность информации из связанных записей (например, электронная почта основного контакта в записи возможности).

**Решение**: Автоматически синхронизировать поля между связанными объектами, пока вложенные поля недоступны.

**Триггер**: Запись обновлена (Объект Возможности, Поле Контактное лицо)

**Действия**:

- Поиск записей, чтобы найти данные связанного лица
- Обновить запись, чтобы копировать адрес электронной почты в возможность
- Обновить запись, чтобы копировать номер телефона в возможность
- Обновить запись, чтобы копировать информацию о компании

## Проверка данных и очистка

### Стандартизация телефонных номеров

**Проблема**: Номера телефонов вводятся в различных форматах, что затрудняет их использование для звонков или сообщений.

**Решение**: Автоматически форматировать номера телефонов в стандартный формат при их вводе.

**Триггер**: Запись обновлена (Объект Люди, Поле Телефон)

**Действия**:

- Code action to parse and format phone number
- Обновить запись со стандартизированным форматом телефона
- Обновить запись, чтобы добавить код страны, если он отсутствует
