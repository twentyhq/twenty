Ниже приведены примеры рабочих процессов, которые вы можете внедрить для подключения Twenty к остальной части вашей системы.

## Data Ingestion Use Cases

### Отправки веб-форм

**Проблема**: Вам необходимо захватывать лиды с веб-форм, целевых страниц или контактных форм непосредственно в Twenty.

**Решение**: Используйте триггеры вебхуков для автоматического создания записей из отправленных форм.

**Настройка**:

- Create a workflow with a Webhook trigger
- Configure the webhook to expect form data (name, email, company, etc.)
- Set the webhook method to POST
- Определите ожидаемую структуру тела в триггере

**Trigger**: Webhook (POST request from your form)

**Действия**:

- Поиск записей для проверки существования человека/компании
- Ветка: Если существует → Обновите запись, если нет → Создайте запись
- Create Record for follow-up task assigned to sales rep
- Отправить email уведомление команде продаж

### Синхронизация данных продукта

**Проблема**: Вашей команде продаж необходима видимость использования продукта, данных о счетах или усвоении функций, хранящихся в вашем хранилище данных.

**Решение**: Регулярно синхронизируйте данные о продукте в Twenty, чтобы предоставить продавцам контекст по их учетным записям.

**Триггер**: По расписанию (ежедневно или еженедельно)

**Действия**:

- HTTP запрос к вашему API хранилища данных
- Действие кода для обработки и форматирования данных
- Используйте функцию Итератора для следующих шагов
- Поиск записей для нахождения соответствующих записей компаний
- Обновление записи для добавления метрик использования продукта
- Создать запись для задач, когда использование падает ниже порога

### Meeting Notes from Call Recorders

**Проблема**: Важные инсайты из звонков продаж теряются или не документируются надлежащим образом в CRM.

**Решение**: Автоматически создавайте заметки и действия из систем записи звонков.

**Триггер**: Вебхук (с платформы записи звонков)

**Действия**:

- Действие кода для извлечения резюме встречи и действий
- Поиск записей для нахождения связанной возможности или контакта
- Создать запись для заметки с резюме встречи
- Создать запись для действия по завершению задач на основании пунктов действий
- Отправить email участникам с резюме и следующими шагами

### Обогащение данных

**Проблема**: Ваши контактные и корпоративные записи не содержат важной демографической и фирмографической информации.

**Решение**: Автоматически обогащайте записи с использованием внешних поставщиков данных.

**Триггер**: Создана запись (объект Люди или Компании)

**Действия**:

- HTTP запрос к API обогащения
- Действие кода для обработки ответа обогащения
- Используйте функцию Итератора для следующих шагов
- Обновление записи с дополнительной информацией о компании/контакте
- Создать запись для задачи по продаже, если определен перспективный клиент высокой ценности
- Отправить email предупреждение, если обогащение выявляет ключевые сигналы покупателя

## Сценарии использования распределения данных

### Управление подписчиками рассылки

**Проблема**: Вы хотите отправлять маркетинговые email конкретным сегментам данных в вашем CRM с использованием специальных инструментов для email.

**Решение**: Экспортируйте списки подписчиков в вашу платформу для email-маркетинга по мере необходимости.

**Настройка**: Создайте в Twenty представление для всех получателей рассылки.

**Trigger**: Launch Manually (when no record is selected)

**Действия**:

- Поиск записей по критериям представления рассылки.
- Код действия для форматирования email-адресов для вашей email-платформы.
- HTTP-запрос для добавления подписчиков в ваш инструмент email-маркетинга.
- Создание записи для отслеживания кампании.
- Отправка email-подтверждения маркетинговой команде.

### Триггеры почтовых последовательностей.

**Проблема**: Вы хотите запускать сложные почтовые последовательности на основе событий CRM, используя специализированные инструменты автоматизации.

**Решение**: Отправляйте новых потенциальных клиентов или клиентов на вашу платформу автоматизации email при возникновении определенных событий.

**Триггер**: Создана запись (объект 'Люди' с определенными критериями)

**Действия**:

- Код действия для определения подходящей почтовой последовательности.
- HTTP-запрос для добавления контакта на платформу автоматизации email.
- Обновление записи для отслеживания зачисления в последовательность.
- Создание записи для последующего задания по мониторингу вовлеченности.

### Интеграция с оценкой лидов.

**Проблема**: Вам нужна сложная оценка лидов, которая объединяет данные CRM с внешними сигналами.

**Решение**: Отправляйте данные о лидах во внешние инструменты оценки или внедряйте логику оценки внутри рабочих процессов.

**Вариант 1 — Внешний инструмент**:

**Триггер**: Запись обновлена (объект 'Люди')

**Действия**:

- HTTP-запрос для отправки данных о лидах на платформу оценки.
- Код действия для обработки ответа с оценкой.
- Обновление записи с оценкой лида.
- Создание записи для задачи продаж, если оценка превышает порог.

**Вариант 2 — Внутренняя логика**:

**Триггер**: Запись обновлена (объект 'Люди')

**Действия**:

- Код действия с алгоритмом оценки (размер компании, отрасль, поведение).
- Обновление записи с рассчитанной оценкой.
- Отправка email-уведомления представителю продаж для лидов с высокой оценкой.

### Генерация счетов.

**Проблема**: Когда сделки закрываются, ваша система выставления счетов должна быть обновлена информацией о клиенте и сделке.

**Решение**: Автоматически отправляйте данные о сделках в вашу систему выставления счетов, когда возможности выиграны.

**Триггер**: Запись обновлена (объект 'Возможности', стадия = "Закрыта выиграна")

**Действия**:

- Поиск записей для получения полной информации о клиенте.
- Код действия для форматирования данных для системы выставления счетов.
- HTTP-запрос для создания клиента в биллинговой платформе.
- HTTP-запрос для генерации счета.
- Обновление записи для хранения ссылки на счет.
- Отправка email финансовой команде с деталями счета.

## Продвинутые интеграционные шаблоны.

### Двунаправленная синхронизация.

**Проблема**: Вам нужно обеспечить синхронизацию данных между Twenty и другой системой в обоих направлениях.

**Решение**: Сочетайте запланированные рабочие процессы с вебхуками для синхронизации в реальном времени.

**От Twenty к внешней системе**:

**Триггер**: Запись обновлена (любой релевантный объект)

**Действия**:

- HTTP-запрос для обновления внешней системы.
- Обновление записи для отслеживания состояния синхронизации и временной метки.

**От внешней системы к Twenty**:

**Триггер**: Вебхук (из внешней системы)

**Действия**:

- Поиск записей для нахождения соответствующей записи.
- Обновление записи с новыми данными из внешней системы.
- Создание записи для задачи разрешения конфликтов, если это необходимо.

### Многошаговая обработка данных.

**Проблема**: Данные из внешних источников требуют сложной обработки перед их использованием в Twenty.

**Решение**: Используйте кодовые действия для трансформации и проверки данных.

**Триггер**: Вебхук или по расписанию.

**Действия**:

- Код действия для проверки формата входящих данных.
- Код действия для трансформации структуры данных.
- Код действия для применения бизнес-правил и вычислений.
- Поиск записей для проверки на дубликаты.
- Создание или обновление записи с обработанными данными.
- Отправка email-уведомления при обнаружении проблем качества данных.

## Советы по внедрению.

- Храните ключи API безопасно в Настройки → API и Вебхуки.
- Используйте HTTPS для всех внешних API вызовов.
- Соблюдайте ограничения скорости API — используйте запланированные рабочие процессы, когда это возможно.
- Рассмотрите возможность выполнения пакетных обновлений "по расписанию", когда реальная обработка не требуется.
- Помните о лимите 100 параллельных рабочих процессов для одного рабочего пространства — используйте "групповую" доступность для ручных триггеров при обработке нескольких записей (см. [Характеристики рабочих процессов](/l/ru/user-guide/workflows/workflow-features) для подробностей).
- Тестируйте с образцами данных до активации рабочих процессов.

Для устранения проблем интеграции, смотрите наш [Руководство по устранению ошибок в рабочих процессах](/l/ru/user-guide/workflows/workflow-troubleshooting). Для помощи в реализации сложных интеграций, рассмотрите наши [Профессиональные услуги](/l/ru/user-guide/workflows/professional-services).
