## Отладка с использованием запуска рабочего процесса

Используйте интерфейс **Запусков рабочих процессов** для устранения неполадок:

- Доступ через панель **Запуски** в редакторе рабочего процесса
- Нажмите на отдельные запуски, чтобы увидеть входные/выходные данные для каждого шага
- Проверьте статус выполнения, сообщения об ошибках и поток данных между шагами

## Распространенные проблемы и решения

### Рабочий процесс не запускается

**Проблема**: Рабочий процесс не выполняется, когда ожидается.

**Решения**:

- Убедитесь, что рабочий процесс **Активен** (не в режиме Черновика)
- Проверьте, что конфигурация триггера соответствует вашей структуре данных
- Для триггеров записи убедитесь, что выбран правильный объект и поля
- For Webhook triggers, verify the URL and expected data format
- Для запланированных триггеров проверьте конфигурацию времени

### Формы недоступны

**Проблема**: Формы сложно найти или получить к ним доступ в автоматизированных рабочих процессах.

<Warning>
Формы в настоящее время предназначены только для ручных триггеров. Для рабочих процессов с другими триггерами (запись создана, обновлена и т. д.) формы доступны только через интерфейс запуска рабочего процесса, и это не ожидаемый пользователем опыт. Центр уведомлений будет выпущен в 2026 году для полноценной поддержки форм в автоматизированных рабочих процессах.
</Warning>

**Решение**: Используйте ручные триггеры, когда требуются формы, или измените структуру рабочих процессов, чтобы избежать использования форм в автоматизированных потоках.

### Высокое потребление кредитов

**Проблема**: Рабочие процессы потребляют больше кредитов, чем ожидалось.

**Распространенные причины и решения**:

- **Неэффективные вызовы API**: группируйте вызовы API, когда это возможно, вместо индивидуальных запросов
- **Неправильная конфигурация ручного триггера**: используйте доступность `Bulk` вместо `Single`, чтобы обработать несколько записей за один запуск рабочего процесса
- **Отсутствуют фильтры**: добавляйте условия для остановки рабочих процессов, когда критерии не удовлетворены
- **Лишние шаги**: удалите избыточные действия и оптимизируйте логику рабочего процесса
- **Реальное время vs. запланированное**: используйте запланированные рабочие процессы для не срочных процессов

### Ограничения на одновременные рабочие процессы

**Проблема**: Достигнуто ограничение на 100 одновременных рабочих процессов на рабочее пространство.

<Warning>
You cannot run more than 100 workflows in parallel at any given time per workspace.
</Warning>

**Решения**:

- Используйте доступность `Bulk` для ручных триггеров, чтобы обработать несколько записей за один запуск
- Реализуйте задержки между выполнениями рабочих процессов, используя запланированные триггеры
- Оптимизируйте рабочие процессы, чтобы они работали быстрее и уменьшали время одновременного выполнения
- Рассмотрите пакетную обработку в нерабочие часы

### Ограничение на количество запросов API

**Проблема**: Внешние вызовы API не удаются из-за ограничений на количество запросов.

**Решения**:

- Используйте запланированные рабочие процессы вместо триггеров в реальном времени, когда это возможно
- Реализуйте задержки между вызовами API в кодовых действиях
- Объединяйте запросы API, когда внешняя служба поддерживает это
- Мониторьте запуски рабочих процессов на наличие ошибок лимита запросов и корректируйте время

### Проблемы с итератором

**Проблема**: Действия итератора не работают как ожидалось.

**Решения**:

- **Примечание**: Итератор в настоящее время находится в стадии бета-тестирования. Активируйте его в Настройки → Релизы → Лаб
- Убедитесь, что ввод представляет собой массив записей из предыдущего шага
- Проверьте, что действия в итераторе правильно настроены
- Используйте итератор с `Bulk` ручными триггерами для оптимальной производительности

### Отсутствующие разрешения

**Проблема**: Невозможно получить доступ к разделу рабочих процессов.

<Warning>
Если вы не видите раздел Рабочие процессы, это связано с проблемой разрешений. Свяжитесь с администратором вашего рабочего пространства, чтобы предоставить вам доступ к рабочим процессам.
</Warning>

## Советы по оптимизации

### Лучшие практики производительности

- **Начните с простого** и добавляйте сложность постепенно
- **Используйте доступность `Bulk`** для ручных триггеров, чтобы обрабатывать несколько записей эффективно
- **Добавьте фильтры заранее**, чтобы остановить рабочие процессы, когда критерии не выполняются
- **Группируйте вызовы API** и используйте запланированные рабочие процессы для несрочных процессов
- **Скрывайте запуски рабочих процессов** со страниц, показывающих все рабочие процессы, чтобы улучшить производительность загрузки
- **Регулярно отслеживайте использование кредитов** в Настройки → Биллинг

### Предотвращение ошибок

- **Тестируйте в режиме черновика** перед активацией рабочих процессов
- **Проверяйте ответы API** и реализовывайте запасные действия
- **Используйте описательные названия шагов** для более легкого обслуживания
- **Документируйте сложную логику** для членов команды
