<Note>
  **Прямое упоминание коллег в заметках пока не поддерживается.** Эта функция запланирована на вторую половину 2026 года. Тем временем этот рабочий процесс предоставляет практичный обходной путь.
</Note>

## Проблема

Вы хотите упомянуть коллегу в заметке, чтобы коллеге пришло уведомление о необходимости её проверить. На данный момент Twenty не поддерживает @упоминания в заметках или задачах.

## Обходной путь

Вместо упоминания в тексте заметки вы можете:

1. Создайте поле связи **Reviewer** в объекте «Заметки»
2. Назначайте проверяющего при создании или редактировании заметки
3. Рабочий процесс автоматически отправляет проверяющему письмо с ссылкой на заметку

<Warning>
  **Ограничение**: этот подход позволяет назначать по одному проверяющему на поле. Если нужно уведомить нескольких людей, создайте дополнительные поля связи (например, Reviewer 1, Reviewer 2). Это плохо масштабируется, если нужно упомянуть сразу многих участников.
</Warning>

## Пошаговая настройка

### Шаг 1: Создайте поле Reviewer

<img src="/images/user-guide/workflows/reviewer-field-for-note.png" style={{width:'100%'}} />

1. Перейдите в **Настройки → Модель данных → Заметки**
2. Нажмите **Добавить поле**
3. Настроить:

| Настройка            | Значение                                            |
| -------------------- | --------------------------------------------------- |
| **Название**         | Reviewer                                            |
| **Тип**              | Связь                                               |
| **Связанный объект** | Участник рабочего пространства                      |
| **Тип связи**        | Много заметок → Один участник рабочего пространства |

4. Сохраните поле

### Шаг 2: Создайте рабочий процесс

<VimeoEmbed videoId="1151897168" title="Видеодемонстрация" />

#### Создайте триггер рабочего процесса

1. Перейдите в основной вид и создайте новый рабочий процесс
2. Выберите **Record Updated** в качестве триггера
3. Выберите **Note** в качестве объекта и укажите поле Reviewer.
4. Это сработает каждый раз, когда у заметки обновляется проверяющий.

#### Добавьте фильтр, чтобы проверить, что проверяющий только что назначен

Добавьте действие **Фильтр**, чтобы рабочий процесс продолжался только когда назначен проверяющий:

| Условие      | Значение |
| ------------ | -------- |
| **Reviewer** | не пусто |

#### Найдите участника рабочего пространства

1. Добавьте действие **Search Records**
2. Выберите **Workspace Member** (в разделе расширенных объектов)
3. Добавьте фильтр:

| Условие | Значение                    |
| ------- | --------------------------- |
| **ID**  | равно `{{note.reviewerId}}` |

Это извлечёт полную запись участника рабочего пространства, включая адрес email и имя.

#### Отправьте уведомление по email

1. Добавьте действие **Send Email**
2. Настроить:

| Поле      | Значение                                            |
| --------- | --------------------------------------------------- |
| **Кому**  | Search Record → First Workspace Member → User Email |
| **Тема**  | Заметка требует вашей проверки                      |
| **Текст** | См. пример ниже                                     |

**Пример текста письма:**

```text
Hi {{searchRecord.firstWorkspaceMember.name.firstName}},

You've been assigned to review a note.

View the note here:
https://yourSubDomain.twenty.com/object/note/{{recordIsUpdated.id}}

Best,
Twenty
```

<Note>
  Замените `yourSubDomain` на фактический поддомен рабочего пространства Twenty.
</Note>

#### Протестируйте и активируйте

1. Нажмите **Test**, чтобы запустить рабочий процесс вручную
2. Создайте или отредактируйте заметку и назначьте проверяющего
3. Проверьте почтовый ящик проверяющего на письмо-уведомление
4. Активируйте рабочий процесс

## Использование

После настройки ваша команда сможет использовать этот рабочий процесс, выполняя следующее:

1. Создание новой заметки (или редактирование существующей)
2. Выбор коллеги в поле **Reviewer**
3. Коллега автоматически получает письмо с прямой ссылкой на заметку
