## Перед началом

### 1. Аудит текущих данных

- **Выберите несколько объектов и полей для переноса**: эта миграция — отличная возможность начать с чистого листа
- **Экспортируйте эти данные** из вашей текущей CRM
- **Удалите дубликаты** и устаревшие записи
- **Перечислите активные рабочие процессы** и автоматизации

### 2. Создайте свою новую модель данных

Следуйте нашему [руководству по модели данных](/l/ru/user-guide/data-model/customize-your-data-model), чтобы:

- **Спроектируйте необходимую модель данных**
- **Сопоставьте существующие поля** со стандартными объектами/полями Twenty
- **Определите пользовательские объекты/поля**, которые вам понадобятся
- **Создайте их** в разделе Настройки → Модель данных

## Процесс миграции

### 1. Импортируйте ваши данные

**Рекомендуемый порядок:**

1. **Компании** в первую очередь (в качестве основных записей)
2. **Люди** во вторую очередь (связаны с компаниями)
3. **Возможности** в третью очередь (связаны с людьми/компаниями)

Используйте импорт CSV через меню команды `Cmd + K` (Mac) или `Ctrl + K` (Windows). См. наше [руководство по импорту данных](/l/ru/user-guide/getting-started/import-export-data) для получения подробных инструкций.

### 2. Воссоздайте рабочие процессы

- **Начните с простого** - сначала воссоздайте критически важные автоматизации
- **Используйте конструктор рабочих процессов Twenty** для замены существующих автоматизаций

## Частые проблемы

### Проблемы с форматированием данных

- **Адреса электронной почты** - удалите дубликаты (требование объекта Люди)
- **Домен** - удалите дубликаты (требование объекта Компании)

<Warning>
Please note that domain URLs created by the synchronization with your mailbox and calendar have the following format ```https://domain.com```
</Warning>

- **Форматы дат** - обеспечьте единообразие форматирования (ГГГГ-ММ-ДД) или отредактируйте этот формат в Настройки → Интерфейс
- **Номера телефонов** - используйте международный формат (+1234567890)

### Сопоставление отношений

Для импорта отношений между записями с использованием функции импорта CSV, вы можете использовать следующие поля

- **Используйте идентификаторы Twenty** для сложных отношений
- **Используйте адреса электронной почты** для связывания записей Людей
- **Используйте доменные имена** для связывания записей Компаний
- **Используйте любое другое поле, которое вы установили как уникальное**, что можно сделать в разделе Модель данных.
  Прочтите наше [руководство по импорту-экспорту данных](/l/ru/user-guide/getting-started/import-export-data) для подробных инструкций по созданию отношений при импорте CSV.

## Профессиональная помощь

### Наши услуги

- **Упаковки на 4 часа обучения** для направленной миграции
- **Партнеры по внедрению** для более сложных проектов

Узнайте о наших [услугах по внедрению](/l/ru/user-guide/getting-started/implementation-services).

## Миграция с локального на облако

Если вы переходите с локальной версии Twenty на облако Twenty:

1. **Экспортируйте ваши данные** из вашей локальной версии
2. **Следуйте стандартному процессу миграции**, описанному выше
3. **Мы можем предоставить помощь в миграции**, свяжитесь с нашей командой
