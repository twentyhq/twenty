## Обзор

Экспортируйте данные своего рабочего пространства в CSV для резервного копирования, отчетности или миграции.

**Сценарии использования:**

* **Регулярные резервные копии** — храните копии ваших данных
* **Внешняя отчетность** — анализируйте данные в Excel, Google Sheets или инструментах BI
* **Миграция** — перенесите данные в другую систему
* **Массовые обновления** — экспортируйте, редактируйте и повторно импортируйте для обновления записей

## Что нужно знать

### Ограничения экспорта

* **Максимум 20 000 записей** на один экспорт
* Экспортируются только **видимые столбцы**
* Экспортируются только **отфильтрованные записи** (на основе вашего текущего представления)

<Note>Для больших экспортов (20 000+ записей) используйте фильтры для экспорта партиями или используйте [API](/l/ru/developers/extend/capabilities/apis).</Note>

### Разрешения

Для экспорта данных вам требуется разрешение **"Export CSV"**. Свяжитесь с администратором рабочего пространства, если у вас нет этой опции.

## Шаг 1: Перейдите к объекту

Перейдите к объекту, который хотите экспортировать:

* **People** — для контактов
* **Companies** — для организаций
* **Opportunities** — для сделок
* **Пользовательские объекты** — любой объект, который вы создали

## Шаг 2: Настройте ваше представление

**Важно:** Экспорт включает только то, что видно в вашем текущем представлении.

### Добавление/удаление столбцов

1. Нажмите **Параметры → Поля** (или **+** в конце столбцов)
2. Отметьте поля, которые хотите экспортировать
3. Снимите отметку с полей, которые вам не нужны

### Фильтрация записей (необязательно)

Если вам нужен только поднабор данных:

1. Нажмите **Фильтр**
2. Добавьте условия фильтра (например, "Created date > January 1, 2024")
3. Будут экспортированы только соответствующие записи

### Сортировка записей (необязательно)

1. Нажмите заголовок столбца, чтобы отсортировать
2. Экспорт сохранит ваш порядок сортировки

<Note>**Создайте отдельное представление для экспорта.** Сохраните представление, специально настроенное для экспорта, чтобы вам не приходилось перенастраивать его каждый раз.</Note>

## Шаг 3: Экспорт данных

1. Нажмите значок **⋮** в правом верхнем углу таблицы
2. Выберите **Экспорт представления**
3. Выберите, куда сохранить файл CSV
4. Дождитесь завершения загрузки

## Что экспортируется

| Включено                                 | Не включено                         |
| ---------------------------------------- | ----------------------------------- |
| Все видимые столбцы                      | Скрытые столбцы                     |
| Записи, соответствующие текущим фильтрам | Записи, исключенные фильтрами       |
| Значения пользовательских полей          | Поля, отсутствующие в представлении |
| ID записей                               | Файловые вложения                   |
| ID связей                                | Изображения                         |

### Поля связи

Идентификаторы связей экспортируются только на стороне **"many"** связи:

* **Экспорт People** включает столбец `companyId` (связь People → Company)
* **Экспорт Companies** НЕ включает `peopleIds` (Companies — это сторона "one")

Это означает, что вы можете использовать экспорт People для повторного импорта и сохранения связи с Company, но вам нужно будет повторно импортировать People после Companies, чтобы восстановить связи.

## Экспорт для конкретных целей

### Для резервного копирования

1. Создайте представление, в котором видны **все поля**
2. Удалите все фильтры, чтобы включить все записи
3. Экспортируйте каждый тип объекта отдельно
4. Храните экспортированные файлы в защищенном месте
5. Настройте регулярное напоминание (еженедельно/ежемесячно)

### Для внешней отчетности

1. Включайте только те поля, которые нужны для анализа
2. Примените фильтры, чтобы сосредоточиться на релевантных данных
3. Рассмотрите сортировку по полю, которое вы будете анализировать

### Для массовых обновлений

1. Экспортируйте записи, которые хотите обновить
2. Включайте уникальный идентификатор (`email`, `domain` или `id`)
3. Отредактируйте экспортированный файл
4. Импортируйте повторно, чтобы обновить записи
   См.: [Как обновить существующие записи](/l/ru/user-guide/data-migration/how-tos/update-existing-records-via-import)

### Для миграции

Если вы экспортируете для миграции в другую систему:

1. **Экспортируйте каждый объект отдельно** — People, Companies, Opportunities и т. д.
2. **Включайте поля с ID** — они помогают сохранять связи
3. **Задокументируйте сопоставления полей** — отметьте, как поля Twenty соответствуют вашей целевой системе

## Обработка больших наборов данных (20 000+ записей)

Лимит экспорта — 20 000 записей. Для больших наборов данных:

### Вариант 1: Экспорт партиями

1. Добавьте фильтр (например, диапазоны "Created date")
2. Экспортируйте первую партию
3. Измените фильтр
4. Экспортируйте следующую партию
5. Объедините файлы в вашей таблице

**Примеры фильтров для разбиения на партии:**

* По диапазону дат (январь, февраль, март...)
* По владельцу (участник команды A, участник команды B...)
* По статусу (Активен, Неактивен...)

### Вариант 2: Используйте API

В API нет ограничения по числу записей:

1. Получите ключ API в **Настройки → Разработчики**
2. Используйте GraphQL API для выборки записей
3. Обрабатывайте результаты в вашем приложении

См.: [Документация по API](/l/ru/developers/extend/capabilities/apis)

## Советы и лучшие практики

### Создавайте представления для экспорта

Сохраняйте представления, специально настроенные для экспорта:

1. Настройте столбцы и фильтры
2. Нажмите **Параметры представления** → **Сохранить как новое представление**
3. Назовите его "Export - [Purpose]"

### Защитите экспортируемые данные

Экспортированные файлы могут содержать конфиденциальные данные:

* Храните в защищенных местах
* Удаляйте старые экспортированные файлы, когда они больше не нужны
* Будьте осторожны при обмене экспортированными файлами

## Часто задаваемые вопросы

<AccordionGroup>
  <Accordion title="Почему в моем экспорте отсутствуют некоторые столбцы?">
    Экспортируются только видимые столбцы. Перед экспортом добавьте нужные столбцы через **Параметры → Поля**.
  </Accordion>

  <Accordion title="Почему мой экспорт содержит не все записи?">
    Проверьте ваши фильтры. Экспорт включает только записи, соответствующие фильтрам вашего текущего представления. Удалите фильтры, чтобы экспортировать все записи.
  </Accordion>

  <Accordion title="Могу ли я экспортировать более 20 000 записей?">
    Не одним экспортом. Используйте фильтры для экспорта партиями или используйте API для больших наборов данных.
  </Accordion>

  <Accordion title="В каком формате файл экспорта?">
    CSV (значения, разделенные запятыми). Открывается в Excel, Google Sheets или любом табличном приложении.
  </Accordion>

  <Accordion title="Включаются ли идентификаторы связей?">
    Да, но только на стороне "many" в связях. Например, экспорт People включает `companyId`, а экспорт Companies не включает ID людей.
  </Accordion>

  <Accordion title="Могу ли я автоматизировать экспорт?">
    Не напрямую через интерфейс. Используйте API, чтобы создать автоматизированные рабочие процессы экспорта.
  </Accordion>
</AccordionGroup>
