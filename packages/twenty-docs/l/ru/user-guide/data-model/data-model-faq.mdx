## Управление объектами

<AccordionGroup>
<Accordion title="Can I reorder objects in the left navigation bar?">
Пока нет. На данный момент последовательность объектов фиксирована, но эта функция планируется в будущем выпуске.
</Accordion>

<Accordion title="Can I hide objects from the left navigation bar?">
Все активные объекты отображаются в навигации. Вы можете деактивировать ненужные объекты через **Настройки → Модель данных**.
</Accordion>

<Accordion title="Can I delete standard objects (People, Companies, etc.)?">
You can deactivate any standard objects but you cannot hard delete them.
</Accordion>
</AccordionGroup>

## Возможности полей

<AccordionGroup>
<Accordion title="Can I create formula fields?">
Поля-формулы появятся в **Q1 2026**. До тех пор вы можете использовать рабочие процессы для автоматического вычисления и обновления значений полей.
</Accordion>

<Accordion title="Can I have nested fields in my objects?">
Вложенные поля появятся в **Q1 2026**. На данный момент вы можете использовать рабочие процессы для извлечения значений полей из связанных объектов. Например, чтобы отображать отрасль компании на записи Персоны, создайте пользовательское поле в Людях и используйте рабочий процесс для синхронизации значения.
</Accordion>

<Accordion title="Why can't I update relation field names?">
Имена полей отношения влияют на структуру API и не могут быть изменены после создания. Если вам нужно переименовать поле отношения, вам нужно создать новое и удалить старое.
</Accordion>

<Accordion title="Why do I need different singular and plural names?">
Our GraphQL API uses both forms for different operations:
- `createPerson` (singular) for single record actions
- `createPeople` (plural) for bulk operations

Это создает ограничения, когда формы единственного и множественного числа совпадают, но улучшает опыт разработчика. </Accordion>

<Accordion title="Why are some field names protected?">
Определенные названия полей, такие как `Тип` или `Приложение`, зарезервированы для системного использования. Выбирайте альтернативные названия, такие как `Категория` или `Классификация`.
</Accordion>
</AccordionGroup>

## Расширенные функции

<AccordionGroup>
<Accordion title="Can I create many-to-many relationships?">
Отношения многие-ко-многим появятся в **Q1 2026**. На данный момент создайте промежуточный объект с двумя отношениями один-ко-многим в качестве временного решения.
</Accordion>

<Accordion title="What are Morph Many relationships?">
Отношения Morph Many позволяют одному объекту связаться с несколькими разными типами объектов. Например, Возможность может быть связана с Персоной или Компанией. Эта функция теперь доступна.
</Accordion>

<Accordion title="Can I reorder fields in objects?">
Переупорядочивание полей будет доступно вместе с настраиваемыми макетами в **Q4 2025**. На данный момент поля отображаются в алфавитном порядке.
</Accordion>
</AccordionGroup>

## Доступ и разрешения

<AccordionGroup>
<Accordion title="Where can I see and edit my data model?">
You can access your Data Model under **Settings → Data Model**.
</Accordion>

<Accordion title="Why can't I see the Data Model under Settings?">
Обратитесь к администратору вашего рабочего пространства. Доступ к модели данных обычно ограничен только администраторами.
</Accordion>

<Accordion title="How many custom objects or fields can I create?">
You can create as many custom objects and fields as you need - the price won't change.
</Accordion>
</AccordionGroup>
