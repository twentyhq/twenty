## Что такое поля связи?

Поля связи соединяют записи из одного объекта с записями в другом объекте. Например:

- **Люди** → **Компании** (каждый человек работает на компанию)
- **Сделки** → **Люди** (в каждой сделке есть контактное лицо)
- **Задачи** → **Сделки** (каждая задача связана с конкретной сделкой)

## Создание полей связи

### 1. Добавьте поле

Перейдите в **Настройки → Модель данных → [Ваш объект]** и нажмите **Добавить поле**.

### 2. Choose Relation Type

- **Тип поля**: Выберите "Связь"
- **Целевой объект**: Выберите объект для подключения
- **Отношение**: В настоящее время поддерживаются только **1-к-многим** отношения. Убедитесь, что создаете связь в правильном направлении.

### 3. Настройка имен полей

Вам нужно будет задать имена для обеих сторон связи:

- **Имя исходного поля**: Как поле отображается в вашем текущем объекте
- **Имя целевого поля**: Как обратное поле отображается в целевом объекте

<Warning>
Имена полей не могут быть изменены после сохранения связи, так как это влияет на структуру API. Выбирайте осторожно.
</Warning>

## Связь с членами команды

Вы можете создавать связи с любым объектом, включая **членов рабочей области** (ваши пользователи команды Twenty). This is useful for creating ownership fields:

- **Владелец аккаунта**: Связать компанию с членом команды, который ею управляет
- **Владелец сделки**: Назначить сделку конкретному продавцу

Когда вы создаете связь с `членами рабочей области`, вы увидите имена членов команды в выпадающем списке, что упрощает назначение собственности и обязанностей.

## Лучшие практики

- **Планируйте свои отношения** перед их созданием
- **Use clear, descriptive names** for both field names
- **Тестируйте отношения** с образцами данных перед полной реализацией

## Будущие возможности

- **Морфология многих отношений** (ожидается в 4 квартале 2025 года)
- **Многие-к-многим отношения** (ожидается в 1 квартале 2026 года)
