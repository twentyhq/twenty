---
title: Настройки доменов
description: Настройте домен рабочего пространства, утверждённые домены доступа и публичные домены.
---

Domain settings live in three places, depending on what you're configuring.

## Домен рабочей области

Configure under **Settings → General → Workspace Domain**.

Измените имя своего субдомена или установите собственный домен для вашего рабочего пространства.

### Настроить домен

1. Нажмите **Настроить домен**
2. Измените свой субдомен (например, `yourcompany.twenty.com`)
3. Или настройте собственный домен (например, `crm.yourcompany.com`)

Для собственных доменов вам потребуется настроить параметры DNS у вашего провайдера домена.

## Утвержденные домены

Configure under **Settings → Members → Access**.

Любой пользователь с адресом электронной почты в этих доменах может автоматически зарегистрироваться в этом рабочем пространстве.

### Добавить утвержденный домен доступа

1. Нажмите **Добавить утверждённый домен доступа**
2. Введите домен вашей компании (например, `yourcompany.com`)
3. Сохранить

После настройки любой пользователь с адресом электронной почты в этом домене сможет присоединиться к вашему рабочему пространству без прямого приглашения.

<Note>
Это удобно: вся ваша команда сможет самостоятельно зарегистрироваться, при этом рабочее пространство останется доступным только для вашей организации.
</Note>

## Публичные домены

Configure under **Settings → Apps → Developer**.

Подготовьте полную и безопасную хостинг-среду на этих доменах. A public domain can be bound to a specific app — when bound, only that app's HTTP-routed logic functions are reachable on the domain. Leave the binding empty to expose all workspace HTTP routes.

### Добавить публичный домен

1. Нажмите **Добавить публичный домен**
2. Введите домен, который вы хотите использовать
3. Optionally bind it to an app
4. Настройте параметры DNS согласно инструкциям
5. Подтвердите домен

SSL-сертификаты автоматически выпускаются для публичных доменов.
