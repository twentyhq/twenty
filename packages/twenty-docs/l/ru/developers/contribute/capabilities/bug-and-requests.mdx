## Сообщение об ошибках

Чтобы сообщить об ошибке, пожалуйста, [создайте задачу на GitHub](https://github.com/twentyhq/twenty/issues/new).

Вы также можете попросить помощи на [Discord](https://discord.gg/cx5n4Jzs57).

## Запросы на Новые Функции

Если вы не уверены, что это ошибка, и вам кажется, что это ближе к запросу на новую функцию, то вам, вероятно, стоит [начать обсуждение вместо этого](https://github.com/twentyhq/twenty/discussions/new).

## Отправьте PR

Вклад в код Twenty начинается с PR.

### Перед началом

1. Проверьте [существующие обращения](https://github.com/twentyhq/twenty/issues) на предмет связанной работы
2. Для новых функций сначала откройте обращение для обсуждения
3. Ознакомьтесь с нашим [Кодексом поведения](https://github.com/twentyhq/twenty/blob/main/CODE_OF_CONDUCT.md)

### Сделайте форк и клонируйте

1. Сделайте форк репозитория на GitHub
2. Клонируйте свой форк:

```bash
git clone https://github.com/YOUR_USERNAME/twenty.git
cd twenty
```

3. Добавьте удалённый репозиторий upstream:

```bash
git remote add upstream https://github.com/twentyhq/twenty.git
```

### Создайте ветку

```bash
git checkout -b feature/your-feature-name
```

Используйте понятные названия веток:

* `feature/add-export-button`
* `fix/login-redirect-issue`
* `docs/update-api-guide`

### Внесите изменения

1. Пишите чистый, хорошо документированный код
2. Соблюдайте существующий стиль кода
3. Добавьте тесты для новой функциональности
4. Обновите документацию при необходимости

### Отправьте свой PR

1. Отправьте свою ветку:

```bash
git push origin feature/your-feature-name
```

2. Откройте PR на GitHub
3. Заполните шаблон PR
4. Привяжите связанные обращения

### Чек-лист PR

* [ ] Код соответствует руководству по стилю проекта
* [ ] Тесты проходят локально
* [ ] Документация обновлена
* [ ] Описание PR объясняет изменения
