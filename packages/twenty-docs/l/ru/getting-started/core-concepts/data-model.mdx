В Twenty всё построено вокруг **объектов** и **полей** — строительных блоков вашей модели данных.

## Объекты

Объекты — это таблицы, в которых хранятся ваши данные. В Twenty предусмотрены стандартные объекты:

* **Companies** — организации, с которыми вы работаете
* **People** — отдельные контакты
* **Opportunities** — сделки в вашем пайплайне
* **Tasks** — задачи для вашей команды
* **Notes** — произвольный текст, связанный с записями

Вы также можете создавать **пользовательские объекты** под любые нужды вашего бизнеса — проекты, заявки в поддержку, продукты, контракты и что угодно ещё.

## Поля

Поля — это свойства каждого объекта. Twenty поддерживает широкий набор типов полей:

| Категория       | Типы                                                                           |
| --------------- | ------------------------------------------------------------------------------ |
| **Базовые**     | Текст, Число, Булево значение, Дата, Валюта, Оценка, Список                    |
| **Составные**   | Адрес (улица, город, штат, индекс), Полное имя, Ссылки, Телефоны, Email-адреса |
| **Специальные** | Связь, Файловое вложение, JSON, Участник (кто создал/изменил)                  |

Каждый объект также получает автоматические системные поля: `id`, `createdAt`, `updatedAt`, `createdBy` и `position`.

## Связи

Объекты соединяются друг с другом с помощью связей. Компания имеет много людей, возможность принадлежит компании и так далее. Вы можете создавать пользовательские связи между любыми объектами, включая связи «многие ко многим».

## Почему это мощно

В отличие от традиционных CRM, где вы ограничены предопределёнными полями на предопределённых объектах, Twenty позволяет моделировать данные именно так, как устроен ваш бизнес. Пользовательские объекты получают такой же статус, как и встроенные, — включая API endpoints, представления, права доступа и триггеры рабочих процессов.
