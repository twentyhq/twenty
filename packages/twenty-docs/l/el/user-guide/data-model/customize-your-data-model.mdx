## Τι είναι ένα μοντέλο δεδομένων;

Ένα μοντέλο δεδομένων είναι η δομή που καθορίζει πώς οργανώνεται η πληροφορία στο CRM σας. Καθορίζει ποια αντικείμενα υπάρχουν (όπως εταιρείες, άτομα ή ευκαιρίες), ποιες ιδιότητες έχουν (αυτά είναι τα πεδία) και πώς συνδέονται μεταξύ τους. Μπορείτε να το σκεφτείτε σαν το χάρτη των δεδομένων πελατών σας.

## Γιατί να προσαρμόσετε το μοντέλο δεδομένων σας;

Κάθε επιχείρηση λειτουργεί διαφορετικά. Η πλήρης δυνατότητα προσαρμογής του μοντέλου δεδομένων σας σημαίνει ότι μπορείτε να διαμορφώσετε το Twenty γύρω από τις διαδικασίες σας αντί να πιέζεστε σε ένα άκαμπτο σύστημα.  
Το Twenty προσφέρει την ευελιξία που χρειάζεστε για να διαμορφώσετε το μοντέλο δεδομένων που θα υποστηρίζει καλύτερα την καθημερινότητά σας. Μπορείτε να δημιουργήσετε όσες προσαρμοσμένες οντότητες και πεδία χρειάζεστε, χωρίς αλλαγή στην τιμή.

## Συμβουλές για το σχεδιασμό του μοντέλου δεδομένων σας

Σπάνια υπάρχει μόνο ένας τρόπος να δημιουργήσεις ένα μοντέλο δεδομένων. Ακολουθούν μερικές συμβουλές για να δημιουργήσετε το δικό σας.

**1. Ξεκινήστε με τις βασικές οντότητες σας.**  
Αναγνωρίστε τις κυριότερες έννοιες με τις οποίες εργάζεστε (π.χ. Εταιρείες, Άτομα, Ευκαιρίες). Αυτά τα τρία αντικείμενα είναι ήδη διαθέσιμα καθώς χρησιμοποιούνται πολύ συχνά. Αλλά σκεφτείτε οποιοδήποτε άλλο μπορεί να χρειαστείτε.  
Παράδειγμα: Η Stripe θα χρειαζόταν ένα αντικείμενο `Συνδρομές`, το Airbnb ένα αντικείμενο `Ταξίδια`, ένας επιταχυντής νεοφυών επιχειρήσεων ένα αντικείμενο `Αποστολές`.

**2. Χρησιμοποιήστε πεδία για παραλλαγές, όχι νέο αντικείμενο.**  
Εάν κάτι είναι απλώς ένα χαρακτηριστικό ενός υπάρχοντος αντικειμένου (π.χ. `Κλάδος` για μια Εταιρεία ή `Κατάσταση` για μια Ευκαιρία), κάντε το πεδίο. Τα πεδία είναι καλύτερα για κατηγορίες, ετικέτες και χαρακτηριστικά.

**3. Δημιουργήστε νέο αντικείμενο όταν αυτό υπάρχει ανεξάρτητο.**  
Αν η έννοια έχει δική της πορεία ζωής, ιδιότητες ή σχέσεις, αξίζει συνήθως για αντικείμενο. Για παράδειγμα:

- **Έργα** που έχουν τις δικές τους προθεσμίες, ιδιοκτήτες και εργασίες
- **Συνδρομές** που συνδέουν εταιρείες, προϊόντα και τιμολόγια
- **Εκδηλώσεις** που περιλαμβάνουν πολλούς συμμετέχοντες και επακόλουθες ενέργειες

Αυτά υπερβαίνουν ένα απλό πεδίο, καθώς φέρουν τα δικά τους δεδομένα και σχέσεις.

**4. Δημιουργείτε αντικείμενο όταν ο αριθμός σχετιζόμενων αρχείων είναι αόριστος.**  
Αν κάτι μπορεί να συνδεθεί πολλές φορές και δεν γνωρίζετε πόσες, είναι καλύτερα ως δικό του αντικείμενο. Για παράδειγμα, αντί να δημιουργείτε πεδία όπως `Προϊόν 1`, `Προϊόν 2`, κ.λπ., καθορίστε ένα αντικείμενο `Προϊόν` και συσχετίστε το με το πρωτότυπο αρχείο. Με αυτό τον τρόπο, μπορείτε να υποστηρίξετε ένα, δύο ή εκατό προϊόντα χωρίς να αλλάξετε το μοντέλο σας.

**5. Απλοποιήστε πρώτα.**  
Ξεκινήστε με πεδία. Περάστε σε νέες οντότητες μόνο όταν αισθανθείτε τα όρια: πάρα πολλά πεδία, επαναλαμβανόμενες εγγραφές ή σχέσεις που δεν χωράνε ομαλά.

### Ειδικό σημείωμα για Άτομα, Εταιρείες και Ευκαιρίες

- **`Ατομα`, `Εταιρείες` και `Ευκαιρίες` είναι τα μόνα αντικείμενα από όπου μπορείτε να έχετε πρόσβαση στα emails και στις συναντήσεις που συγχρονίζονται από το γραμματοκιβώτιο / ημερολόγιο σας.** Συνιστούμε να τα χρησιμοποιείτε όσο το δυνατόν περισσότερο. Εάν χρειαστεί να δημιουργήσετε κατηγορίες `Ατόμων` ή `Εταιριών`, χρησιμοποιήστε πεδία αντί για νέα αντικείμενα.

  Παράδειγμα: είναι καλύτερο να χρησιμοποιείτε το αντικείμενο `Άτομα` τόσο για υποψήφιους όσο και για συνεργάτες, προσθέτοντας ένα πεδίο που ονομάζεται `Τύπος Ατόμου`. Αποφύγετε τη δημιουργία αντικειμένου `Συνεργάτης`, καθώς δεν θα μπορείτε να αποκτήσετε πρόσβαση σε νήματα email από αυτό. Αντ' αυτού, δημιουργήστε διαφορετικές προβολές υπό `Άτομα`: μία που να δείχνει συνεργάτες, μία άλλη που να δείχνει υποψηφίους.
- Δεδομένης της άνωθεν παρατηρήσεως, είναι αποδεκτό να έχετε πεδία που δεν ισχύουν για κάθε εγγραφή. Για παράδειγμα, υπό `Άτομα` μπορείτε να προσθέσετε ένα πεδίο `Σύνδεσμος Παραπομπής` που είναι σχετικό μόνο όταν `Τύπος Ατόμου = Συνεργάτης`. Αυτό είναι αποδεκτό: μπορείτε να αποκρύψετε αυτό το πεδίο από προβολές όπου δεν χρειάζεται.

### Ερωτήσεις για να καθοδηγήσουν την επιλογή σας

Ρωτήστε τον εαυτό σας:

- Είναι αυτό απλώς μια ιδιότητα κάποιου υπάρχοντος στοιχείου ή χρειάζεται τις δικές του ιδιότητες;
- Θα χρειαστεί ποτέ να παρακολουθήσω πολλαπλά από αυτά ανά εγγραφή, χωρίς να γνωρίζω πόσα εκ των προτέρων;
- Συνδέεται αυτή η έννοια με πολλά διαφορετικά αντικείμενα, όχι μόνο ένα;
- Θα έχει δική του πορεία ζωής (π.χ. στάδια, ημερομηνίες έναρξης/λήξης);

Αν η απάντηση είναι “ναι” σε μία ή περισσότερες, πιθανώς είναι η ώρα για νέο αντικείμενο.

## Χρειάζεστε βοήθεια;

Η ομάδα μας μπορεί να σας βοηθήσει στο σχεδιασμό και τη δημιουργία του μοντέλου δεδομένων που χρειάζεστε. Ανακαλύψτε το Πακέτο Εκκίνησης [εδώ](https://twenty.com/onboarding-packages).
