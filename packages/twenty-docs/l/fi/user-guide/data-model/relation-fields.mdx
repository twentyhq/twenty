## Mitä ovat relaatiokentät?

Relaatiokentät yhdistävät tietueet yhdestä objektista toiseen objektiin. Esimerkki:

- **Henkilöt** → **Yritykset** (jokainen henkilö työskentelee yritykselle)
- **Mahdollisuudet** → **Henkilöt** (jokaisella kaupalla on yhteyshenkilö)
- **Tehtävät** → **Mahdollisuudet** (jokainen tehtävä liittyy tiettyyn kauppaan)

## Relaatiokenttien luominen

### 1. Lisää kenttä

Mene kohtaan **Asetukset → Tietomalli → [Oma Objekti]** ja klikkaa **Lisää Kenttä**.

### 2. Valitse relaatiotyyppi

- **Kentän tyyppi**: Valitse "Relaatio"
- **Kohdeobjekti**: Valitse mihin objektiin yhdistetään
- **Suhde**: Tukee tällä hetkellä vain **1-moneen** suhteita. Varmista, että luot suhteen oikeaan suuntaan.

### 3. Configure Field Names

Sinun on määritettävä nimet molemmille suhteen puolille:

- **Lähdekentän nimi**: Miten kenttä näkyy nykyisessä objektissasi
- **Kohdekentän nimi**: Miten vastakenttä näkyy kohdeobjektissa

<Warning>
Kenttien nimiä ei voi muokata, kun suhde on tallennettu, koska se vaikuttaa API-rakenteeseen. Valitse huolellisesti.
</Warning>

## Suhteet tiimin jäseniin

Voit luoda suhteita mihin tahansa objektiin, mukaan lukien **Työtilan Jäsenet** (Twenty-tiimisi käyttäjät). Tämä on hyödyllistä omistajuuskenttien luomiseen:

- **Tilin omistaja**: Linkitä yritys sitä hallinnoivaan tiimin jäseneen
- **Kaupan omistaja**: Määrää mahdollisuus tietylle myyjälle

Kun luot suhteen `Työtilan Jäseniin`, näet tiimisi jäsenten nimet pudotusvalikoissa, mikä helpottaa omistajuuden ja vastuiden määrittämistä.

## Parhaat käytännöt

- **Suunnittele suhteesi** ennen niiden luomista
- **Use clear, descriptive names** for both field names
- **Kokeile suhteita** malliaineistolla ennen täydellistä käyttöönottoa
