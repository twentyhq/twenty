---
title: メンバー管理
description: チームメンバーを招待し、ワークスペースのアクセスを管理します。
---

**設定 → メンバー** で、ワークスペースにアクセスできるユーザーを管理します。

## 新規メンバーを招待

### メール招待を使用する

1. **設定 → メンバー** に移動
2. **+ 招待** をクリック
3. その人のメールアドレスを入力
4. 新しいメンバーのロールを選択します。
5. **招待を送信** をクリック

招待されたユーザーには、ワークスペースに参加するためのリンクが記載されたメールが届きます。

### 招待リンクを使用する

1. **設定 → メンバー** に移動
2. ワークスペースの招待リンクをコピー
3. 新しいチームメンバーにリンクを共有
4. サインアップするとアクセスできるようになります

## メンバーの表示と管理

### すべてのメンバーを表示

**設定 → メンバー** に移動すると、次を確認できます:

* すべてのアクティブなメンバー
* 保留中の招待

### メンバーのプロフィールを編集

メンバーをクリックしてプロフィールページを開きます。 管理者として、以下が可能です:

* **名前** を編集
* **プロフィール画像** を更新
* **代理ログイン** でそのアカウントとして操作 (トラブルシューティングに便利)
* そのアカウントを **削除**

### メンバーのロールを変更

メンバーのプロフィールページで:

1. **権限** タブを開く
2. 現在割り当てられているロールを表示
3. ドロップダウンメニューから別のロールを選択します。
4. 変更は即時に反映されます

→ [ロールと権限の詳細](/l/ja/user-guide/permissions-access/capabilities/permissions)

### メンバーを削除

1. メンバーをクリックしてプロフィールを開きます。
2. ワークスペースから削除するには **削除** をクリックします。

<Warning>
  削除されたメンバーは直ちにアクセス権を失います。 そのデータ（レコード、メモ、タスク）はワークスペースに残ります。
</Warning>

<Warning>
  **メール同期も削除されます。** もし削除されたユーザーだけが特定のメールを同期していた場合、そのメールはワークスペースから完全に削除されます。
</Warning>

## 保留中の招待

未承諾の招待を管理します:

* **再送**: 招待メールを再送します
* **キャンセル**: 承諾前の招待を取り消します

## 承認されたアクセスドメイン

メールドメインに基づき、チームメンバーが自動参加できるようにします:

1. **設定 → ドメイン** に移動する
2. 会社のドメインを追加します（例: `yourcompany.com`）
3. そのメールドメインを持つユーザーは、招待なしで参加できます。
