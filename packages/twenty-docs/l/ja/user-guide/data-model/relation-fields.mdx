## What are Relation Fields?

関連フィールドは、一つのオブジェクトのレコードと別のオブジェクトのレコードをリンクします。 例えば： 例えば：

- **人物** → **会社** (各人は会社のために働いています)
- **Opportunities** → **People** (each deal has a contact person)
- **タスク** → **機会** (各タスクは特定の取引に関連しています)

## 関連フィールドの作成

### 1. フィールドを追加

\*\*設定→データモデル→[あなたのオブジェクト]\*\*に進み、**フィールドを追加**をクリックします。

### 2. 関連タイプを選択

- **フィールドタイプ**: "関連"を選択
- **ターゲットオブジェクト**: 接続するオブジェクトを選択
- **関係**: 現在は**1対多**の関係をサポートしています。 関係の方向を正しく設定してください。 関係の方向を正しく設定してください。

### 3. フィールド名の設定

関係の両側に名前を設定する必要があります:

- **ソースフィールド名**: 現在のオブジェクト上でのフィールドの表示方法
- **ターゲットフィールド名**: ターゲットオブジェクト上での逆フィールドの表示方法

<Warning>

関係が保存されると、フィールド名はAPIストラクチャに影響を与えるため編集できません。 慎重に選択してください。
 慎重に選択してください。
</Warning>

## チームメンバーとの関連

You can create relations to any object, including **Workspace Members** (your Twenty team users). これは、所有権フィールドを作成するのに役立ちます:

- **アカウント所有者**: 企業とそれを管理するチームメンバーをリンク
- **取引所有者**: 特定の営業担当者に機会を割り当てる

`ワークスペースメンバー`への関係を作成すると、ドロップダウン選択にチームメンバーの名前が表示され、所有権と責任の割り当てが容易になります。
