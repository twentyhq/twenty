手作業を削減し、データを最新の状態に保つための自動化ワークフロー例が以下に示されています。

## データ管理オートメーション

### 個人メールを自動でフラグ付け

**問題**:ターゲット設定とコンプライアンスの向上のために、チームは個人メールアドレスとビジネスメールアドレスを特定する必要があります。

**解決策**:メールアドレスが追加または更新されるたびに「個人メール」のフィールドが自動で更新されるワークフローを作成します。

**トリガー**:レコードが更新される（人物オブジェクト、メールフィールド）

**アクション**:

- メールドメインが一般的な個人プロバイダ（gmail.com, yahoo.comなど）と一致するかどうかチェックするコードアクション
- 「個人メール」フラグを設定するためにレコードを更新

### リード割当 - ラウンドロビン

**問題**:ワークロードのバランスを確保するために、新しいリードを公平に販売チームに分配する必要があります。

**解決策**:ラウンドロビンシステムを使用して、新しいリードを販売担当者に自動的に割り当てます。

**トリガー**:レコードが作成される（人物オブジェクト）

**アクション**:

- 最後に割り当てられた担当者を見つけるためにレコードを検索
- ローテーションの次に来る担当者を決定するコードアクション
- 選ばれた担当者にリードを割り当てるためにレコードを更新
- 割り当てられた担当者に通知メールを送信

### リード割当 - テリトリーベース

**問題**:リードは地理的なテリトリーや企業の特性に基づいて割り当てられるべきです。

**解決策**:位置、企業規模、業界に基づいて適切な販売担当者にリードをルートさせます。

**トリガー**:レコードが作成される（人物または企業オブジェクト）

**アクション**:

- 場所/業界ルールに基づくテリトリーを決定するコードアクション
- テリトリーオーナーを見つけるためにレコードを検索
- リードを割り当てるためにレコードを更新
- フォローアップタスクのためにレコードを作成

## 販売プロセスオートメーション

### 機会ステージ管理 - 成約

**問題**:契約が成立すると、異なる記録やチームメンバーに対して多くの手動更新が必要になります。

**解決策**:機会が「成約」となるとすべてのポストウィン活動を自動的に処理します。

**トリガー**:レコードが更新される（機会オブジェクト、ステージフィールド = "成約"）

**アクション**:

- 会社タイプを「見込み客」から「顧客」に変更するためにレコードを更新
- アカウントマネージャーに割り当てられたオンボーディングタスクのためにレコードを作成
- 顧客成功チームへのメール通知を送信
- 外部請求システムを更新するためにHTTPリクエスト

### 機会ステージ管理 - 成約損失の更新

**問題**:更新機会を失うと、適切なアカウント管理のために顧客ステータスを更新する必要があります。

**解決策**:更新機会が失われた際に顧客ステータスを自動的に更新します。

**トリガー**:レコードが更新される（機会オブジェクト、ステージ = "成約損失" AND タイプ = "更新"）

**アクション**:

- 会社タイプを「顧客」から「離脱顧客」に変更するためにレコードを更新
- 離脱分析タスクのためにレコードを作成
- 顧客成功マネージャーへのメールアラートを送信
- 離脱日と理由を追加するためにレコードを更新

### 機会停滞警告

**問題**:更新されていない機会は、取引が冷え込み、予測が信用できないものになります。

**解決策**:最近更新されていない機会について自動的に警告を送信します。

**トリガー**:スケジュール (毎日)

**アクション**:

- X日更新されていない機会のためにレコードを検索
- 機会の詳細を含む警告メッセージをフォーマットするコードアクション
- 機会の所有者とマネージャーにメールを送信
- 応答がない場合にフォローアップタスクのためにレコードを作成

## 生産性オートメーション

### 週間タスク概要

**問題**:チームメンバーは、今後のタスクや締め切りを見失います。

**解決策**:タスクの要約を含めた自動化された毎週のメールリマインダーを送信します。

**トリガー**:スケジュール（毎週月曜日の午前8時）

**アクション**:

- 今週の担当者別のタスクのためにレコードを検索
- 人ごとにタスクリストをフォーマットするコードアクション
- 各チームメンバーにタスクの要約をメールで送信
- チーム概要を含むメールをマネージャーに送信

### 会議フォローアップの自動化

**問題**:会議での重要なアクションアイテムが忘れられたり、遅れたりします。

**解決策**:会議が予定または完了したときにフォローアップタスクを自動的に作成します。

**トリガー**:レコードが作成される（活動オブジェクト、タイプ = "会議"）

**アクション**:

- 事前準備タスクのためにレコードを作成
- 会議後のフォローアップタスクのためにレコードを作成
- 参加者にメールリマインダーを送信
- タスクを会議にリンクするためにレコードを更新

### クロスオブジェクトフィールド同期

**問題**:関連記録からの情報が容易にアクセス可能でなければなりません（例: 機会記録上の主要な連絡先のメール）。

**解決策**:ネストされたフィールドが利用可能になるまで、関連オブジェクト間でフィールドを自動的に同期します。

**トリガー**:レコードが更新される（機会オブジェクト、連絡先フィールド）

**アクション**:

- リンクされた人物の詳細を見つけるためにレコードを検索
- 機会にメールアドレスをコピーするためにレコードを更新
- 機会に電話番号をコピーするためにレコードを更新
- 会社情報をコピーするためにレコードを更新

## データ検証とクリーンアップ

### 電話番号の標準化

**問題**:電話番号が様々なフォーマットで入力され、通話やメッセージの使用が困難になります。

**解決策**:入力された際に電話番号を標準フォーマットに自動的に変更します。

**トリガー**:レコードが更新される（人物オブジェクト、電話フィールド）

**アクション**:

- 電話番号を解析しフォーマットするコードアクション
- 標準化された電話フォーマットにレコードを更新
- 国コードがない場合にそれを追加するためにレコードを更新

より複雑なオートメーション要件については、[プロフェッショナルサービス](/l/ja/user-guide/workflows/professional-services)を検討するか、[外部ツール統合](/l/ja/user-guide/workflows/external-tool-integration)を探索してください。
