ワークフロークレジットは、Twentyでの自動化をサポートします。 その仕組みを理解することで、コストを最適化し、自動化予算を効果的に管理できます。 その仕組みを理解することで、コストを最適化し、自動化予算を効果的に管理できます。

## プラン別クレジット配分

ワークフロークレジットは、サブスクリプションプランに基づいて各ワークスペースに割り当てられます。

- **トライアル**: 500万クレジット
- **Proプラン**: 月あたり1,000万クレジット
- **組織プラン**: 月あたり2,000万クレジット

## クレジット消費の仕組み

ワークフローは実行時にクレジットを消費し、それを作成する際には消費されません。 各ワークフローアクションはその複雑さに応じてクレジットを消費します。 各ワークフローアクションはその複雑さに応じてクレジットを消費します。

### アクションタイプ別クレジット消費

- **基本的な内部操作**: 非常に低いクレジット消費
  - レコードを検索
  - Create Record
  - レコードを更新
  - レコードを削除
  - フォームアクション

- **複雑な操作**: 高いクレジット消費
  - コードアクション (JavaScript実行)
  - HTTP Requests to external services

- **AI機能**: かなりのクレジット消費（近日公開）
  - AIプロンプトおよび処理は、使用量に基づいてクレジットを消耗します

### リアルタイム控除

ワークフローが実行されると、クレジットはリアルタイムで控除されます。 つまり： つまり：

- ドラフトのワークフローはクレジットを消費しません
- アクティブで実行中のワークフローだけがクレジット配分を使用します
- 失敗したワークフローでも、完了したステップに対してはクレジットが消費されます

## クレジットの管理

### クレジット使用の確認

1. **設定 → 請求** へ移動
2. 現在のクレジット消費と残高を確認
3. 消費パターンを監視してワークフローを最適化

### 追加クレジットの購入

プランのクレジット割り当てを超える場合は：

1. **設定 → 請求** へ移動
2. 追加クレジットを購入するオプションをクリックします。 さまざまなサイズのパッケージが用意されています。 さまざまなサイズのパッケージが用意されています。
3. クレジットは現在の残高に追加されます

## ベストプラクティス

### 効率的なワークフローデザイン

- **Start Simple**: Begin with basic actions and add complexity gradually
- **ドラフトでテスト**: エラーによるクレジットの無駄を避けるため、ワークフローをアクティベーション前に徹底的にテスト
- **HTTPリクエストを最小限に**: 効率的な検索条件を使用し、可能な場合は操作を結合
- **バッチ処理**: 効率的に一括操作やイテレータアクションを利用
- **エラーハンドリング**: 不要なリトライを防ぐために適切なエラーハンドリングを実装
- **手動トリガーの最適化**: 手動トリガーの場合、「一括」を選択して、単一のワークフロー実行で複数のレコードを処理（詳細は [ワークフロー機能](/l/ja/user-guide/workflows/workflow-features) を参照）

### アクションの最適化

- 可能な場合、基本的な内部操作を使用（クレジット消費が少ない）
- 効率性のためにコードアクションを最適化
- 緊急でないプロセスには手動トリガーを検討
- 個々のアクションコールを減らすためのバッチ操作

### クレジット管理

- **定期的な監視**: 設定→請求で毎週クレジット使用量を確認
- **使用状況の追跡**: 不意の超過を避けるために消費パターンを把握
- **適切に計画**: より多くのクレジットが常に必要ならプランのアップグレードを検討
- **ワークフローの見直し**: 消費の高いワークフローを定期的にレビューし、未使用のものを無効化
- **戦略的計画**: クレジットの利用可能性に基づいてワークフローを展開

詳細な請求情報とクレジット管理については、Twentyワークスペースの **設定 → 請求** を訪れてください。
