Dokument ten zawiera najlepsze praktyki, które należy stosować podczas pracy nad backendem.

## Stosuj podejście modułowe

Backend działa w podejściu modułowym, co jest podstawową zasadą pracy z NestJS. Upewnij się, że dzielisz swój kod na moduły do wielokrotnego użytku, aby utrzymać czystość i organizację bazy kodu.
Each module should encapsulate a particular feature or functionality and have a well-defined scope. Takie podejście modułowe umożliwia wyraźne oddzielenie zadań i usuwa niepotrzebne zawiłości.

## Udostępnij usługi do użycia w modułach

Zawsze twórz usługi o jasno określonej i pojedynczej odpowiedzialności, co zwiększa czytelność i utrzymywalność kodu. Nazywaj usługi w sposób opisowy i spójny.

Powinieneś także udostępniać usługi, które chcesz używać w innych modułach. Udostępnianie usług innym modułom jest możliwe dzięki potężnemu systemowi wstrzykiwania zależności NestJS i promuje luźne powiązanie między komponentami.

## Unikaj używania typu `any`

Gdy zadeklarujesz zmienną jako `any`, kontroler typów TypeScript nie wykonuje żadnego sprawdzania typów, co umożliwia przypisanie dowolnego rodzaju wartości do zmiennej. TypeScript używa inferencji typów do określenia typu zmiennej na podstawie wartości. Deklarując ją jako `any`, TypeScript nie może już wnioskować typu. This makes it hard to catch type-related errors during development, leading to runtime errors and makes the code less maintainable, less reliable, and harder to understand for others.

Dlatego wszystko powinno mieć określony typ. Jeśli więc tworzysz nowy obiekt z imieniem i nazwiskiem, powinieneś stworzyć interfejs lub typ, który zawiera imię i nazwisko, które definiuje kształt obiektu, którym zarządzasz.
