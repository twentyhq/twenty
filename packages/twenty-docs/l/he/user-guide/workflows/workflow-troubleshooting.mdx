## Debugging with Workflow Runs

השתמש בממשק **ריצות זרימת העבודה** כדי לבדוק בעיות:

- Access via the **Runs** panel in the workflow editor
- לחץ על ריצות בודדות כדי לראות נתוני קלט/פלט לכל שלב
- בדוק את מצב הביצוע, הודעות השגיאה, וזרימת הנתונים בין השלבים

## בעיות נפוצות ופתרונות

### זרימת עבודה לא מופעלת

**בעיה**: זרימת העבודה אינה מתבצעת כאשר היא צפויה.

**פתרונות**:

- ודא כי הזרימה **פעילה** (לא במצב טיוטה)
- בדוק כי הגדרת ההפעלה תואמת למבנה הנתונים שלך
- להפעלת רישום, ודא שהאובייקט והשדות המתאימים נבחרים
- For Webhook triggers, verify the URL and expected data format
- For Scheduled triggers, check the timing configuration

### טפסים אינם נגישים

**בעיה**: פעולות בטפסים קשות למציאה או גישה בזרימות עבודה אוטומטיות.

<Warning>
טפסים מתוכננים כרגע להטריגרים ידניים בלבד. בזרימות עבודה עם טריגרים אחרים (רשומה נוצרה, עודכנה), טפסים זמינים רק דרך ממשק ריצת הזרימה, וזה לא מה שחויית המשתמש מצפה לה. מרכז ההודעות ישוחרר ב-2026 כדי לתמוך בטפסים בזרימות עבודה אוטומטיות.
</Warning>

**Workaround**: Use manual triggers when forms are required, or restructure workflows to avoid forms in automated flows.

### צריכת קרדיט גבוהה

**בעיה**: זרימות עבודה צורכות יותר קרדיטים מהמצופה.

**גורמים נפוצים ופתרונות**:

- **קריאות API לא יעילות**: רכז קריאות API אפשריות במקום קריאות יחידות
- **הגדרות לא נכונות לטריגר ידני**: השתמש ב`Bulk` זמינות במקום `Single` לעיבוד מספר רשומות בריצת עבודה אחת
- **חסרות מסננים**: הוסף לוגיקה מותנית כדי לעצור זרימות כשקריטריונים לא מתקיימים
- **שלבים מיותרים**: הסר פעולות מיותרות ושפר את הלוגיקה של הזרימה
- **זמן אמת מול מתוזמן**: השתמש בזרימות מתוזמנות לתהליכים לא דחופים

### הגבלת זרימות עבודה במקביל

**בעיה**: מקסימום 100 זרימות עבודה מקבילית פר מרחב עבודה.

<Warning>
You cannot run more than 100 workflows in parallel at any given time per workspace.
</Warning>

**פתרונות**:

- השתמש ב`Bulk` זמינות לטריגרים ידניים לעיבוד מספר רשומות בריצה אחת
- הכנס עיכובים בין ביצועי הזרימות כאשר שולחים טריגרים מתוזמנים
- שפר את ביצועי הזרימה לצורך הפחתת זמן הביצוע במקביל
- שקול עיבוד קבוצתי בשעות שאינן שיא

### מגבלת קצב API

**בעיה**: קריאות API חיצוניות נכשלות עקב מגבלות קצב.

**פתרונות**:

- השתמש בזרימות מתוזמנות במקום טריגרים בזמן אמת כשאפשרי
- בצע עיכובים בין קריאות API בפעולות קוד
- רכז בקשות API כאשר השירות החיצוני תומך בכך
- עקוב אחר ריצות זרימות עבודה לטובת שגיאות מגבלת קצב והתאם את התזמון

### בעיות עם אריך

**בעיה**: פעולות אריך לא פועלות כפי שצפוי.

**פתרונות**:

- **הערה**: אריך כרגע בגרסת בטא. הפעל אותו בהגדרות → שחרור → מעבדה
- וודא שהקלט הוא מערך רשומות משלב קודם
- בדוק שהפעולות בתוך האריך מוגדרות כראוי
- השתמש באריך עם `Bulk` טריגרים ידניים לביצועים מיטביים

### הרשאות חסרות

**Problem**: Cannot access workflows section.

<Warning>
If you don't see the Workflows section, this is due to a permissions issue. פנה למנהל מרחב העבודה שלך כדי להעניק לך גישה לזרימות עבודה.
</Warning>

## עצות אופטימיזציה

### שיטות עבודה מיטביות לביצועים

- **התחל פשוט** והוסף מורכבות בהדרגה
- **השתמש ב`Bulk` זמינות** לטריגרים ידניים לעיבוד מספר רשומות ביעילות
- **הוסף מסננים מוקדם** לעצור זרימות כשקריטריונים אינם מתקיימים
- **רכז קריאות API** והשתמש בזרימות מתוזמנות לתהליכים לא דחופים
- **הסתר ריצות זרימה** מעמודים המציגים את כל הזרימות לשיפור ביצועי הטעינה
- **עקוב אחר הצריכה של קרדיטים** באופן קבוע בהגדרות → חיוב

### מניעת שגיאות

- **בדוק במצב טיוטה** לפני הפעלת זרימות עבודה
- **אמת תשובות של API** והטמע פעולות גיבוי
- **השתמש בשמות שלבים תיאוריים** לצורך תחזוקה קלה יותר
- **תעד לוגיקה מסובכת** לחברי הצוות
