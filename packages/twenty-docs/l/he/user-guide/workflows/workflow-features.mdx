## טריגרים לתהליכי עבודה

תהליכי עבודה תמיד מתחילים עם טריגר בודד המגדיר מתי יש להריץ את האוטומציה.

### רשומה נוצרת

מתחיל את תהליך העבודה כאשר רשומה חדשה נוצרת באובייקט נבחר (אנשים, חברות, הזדמנויות או כל אובייקט מותאם אישית).

**תצורה**: בחר את סוג האובייקט למעקב אחר רשומות חדשות.

### רשומה מעודכנת

מתחיל את תהליך העבודה כאשר נעשים שינויים ברשומה קיימת.

**תצורה**:

- בחר את סוג האובייקט
- באופן אופציונלי ציין באילו שדות לעקוב אחר שינויים

### רשומה מעודכנת או נוצרת

Starts the workflow when a record is either created or updated in a selected object.

**למה זה חשוב**: טריגר זה שימושי במיוחד כי רשומות שנוצרות במגוון שיטות מתנהגות באופן שונה:

- **ייבוא API/CSV**: רשומות נוצרות עם כל השדות מלאים מיידית
- **יצירה ידנית**: רשומות נוצרות תחילה, ולאחר מכן נוספים השדות בעדכונים הבאים

**תצורה**:

- בחר את סוג האובייקט למעקב
- באופן אופציונלי ציין באילו שדות לעקוב אחר שינויים
- הטריגר יפעל גם בעת יצירה ראשונית וגם בעדכונים שלאחר מכן

### רשומה נמחקת

מתחיל את תהליך העבודה כאשר רשומה מוסרת מאובייקט.

**תצורה**: בחר את סוג האובייקט למעקב אחר מחיקות.

### הפעלה ידנית

מתחיל את תהליך העבודה כאשר מופעל על ידי פעולה של משתמש. טריגר זה ניתן לגישה דרך תפריט Cmd+K או דרך כפתור מותאם אישית בסרגל הניווט העליון.

**תצורת זמינות**:
בחר כיצד על תהליך העבודה לטפל בבחירת רשומה:

- **גלובלי**: אין צורך ברשומה כדי להתחיל את תהליך העבודה הזה. תהליך העבודה מופעל מכל מקום (מכל אובייקט) ואינו משתמש ברשומה/יות כקלט.

- **בודד**: הרשומה/ות הנבחרת/ות תועבר/נה לתהליך העבודה שלך. זה מוגדר לאובייקט נתון. ניתן לבחור מספר רשומות לפני הפעלת תהליך העבודה. תהליך העבודה ירוץ ככל שמספר הרשומות שנבחרו.

  <Warning>
  You cannot run more than 100 workflows in parallel at any given time.
  </Warning>

- **בכמות גדולה**: הרשומות שנבחרו יועברו לזרימת העבודה שלך. זה מוגדר עבור אובייקט נתון. ניתן לבחור מספר רשומות לפני הפעלת זרימת העבודה. זרימת העבודה תופעל פעם אחת, ותספק את הרשימה המלאה של הרשומות כקלט. This means the workflow needs to contain an Iterator action. זה הכי מתאים לאנשים שרוצים לאופטימיז ולמזער את כמות ההרצות של זרימת העבודה.

**הגדרה נוספת**:

- בחר את האובייקט המטרה (לזמינות בכמות יחידה ובכמויות גדולות)
- בחר אייקון פקודה להפעלת הטריגר של זרימת העבודה
- צור מיקום בשורת הנאבאר (מותקן או לא מותקן)

**שיטות גישה**:

- תפריט Cmd+K למציאת והפעלה של זרימות עבודה ידניות
- לחצן מותאם אישית בשורת ההנאה העליונה (אם הוגדר)

### על לוח זמנים

מתחיל את זרימת העבודה באופן מחזורי שאתה מגדיר.

**הגדרה**:

- בחר יחידת זמן (דקות, שעות, ימים)
- הזן ערך או השתמש בביטויים מותאמים אישית של cron לתזמון מתקדם

### וובהוק

מתחיל את זרימת העבודה כאשר מתקבלת בקשת GET או POST משירות חיצוני.

**הגדרה**:

- קבל כתובת URL ייחודית ל-webhook
- לבקשות POST, הגדר את מבנה הגוף הצפוי
- הגדר אימות במידת הצורך

## פעולות זרימת עבודה

הפעולות מגדירות מה קורה לאחר שטריגר מופעל. ניתן לשלב מספר פעולות יחד.

### צור רשומה

מוסיף רשומה חדשה לאובייקט שנבחר.

**הגדרה**:

- בחר באובייקט המטרה
- מלא את השדות הנדרשים והאופציונליים
- השתמש בנתונים משלבים קודמים למילוי השדות

**תפוקה**: הנתונים של הרשומה החדשה זמינים לשימוש בשלבים הבאים.

### עדכן רשומה

משנה רשומה קיימת באובייקט שנבחר.

**הגדרה**:

- בחר באובייקט המטרה
- בחר את הרשומה הספציפית לעדכון
- בחר שדות לשינוי והזן ערכים חדשים

**תפוקה**: הנתונים של הרשומה המחודשת זמינים לשימוש בשלבים הבאים.

### מחק רשומה

מסיר רשומה מאובייקט שנבחר.

**הגדרה**:

- בחר באובייקט המטרה
- בחר את הרשומה הספציפית למחיקה

**תפוקה**: הנתונים של הרשומה המחוקה ניתנים לשימוש בשלבים הבאים.

### חיפוש רשומות

מאתר רשומות בתוך אובייקט שנבחר תוך שימוש בתנאי סינון.

**הגדרה**:

- בחר את האובייקט לחיפוש
- הגדר קריטריוני סינון להצר את התוצאות
- קבע תצורת סידור ומגבלות

**תפוקה**: מחזיר רשומות תואמות שניתן להשתמש בהן בשלבים הבאים.

**המלצה**: השתמש בענפים לאחר חיפוש רשומות לטיפול בתרחישים "נמצא" מול "לא נמצא".

### איטרטור

עובר ברשימה של רשומות שהוחזרו משלב קודם, ומאפשר לך לבצע פעולות על כל רשומה בנפרד.

**הגדרה**:

- בחר את רשימת הרשומות משלב קודם (לדוגמה, תוצאות חיפוש רשומות)
- הגדר את הפעולות לביצוע על כל רשומה בלולאה
- הגדר את שם המשתנה להתייחסות לכל רשומה באיטרציה

**Example**: Search Records returns 5 people, then use Iterator to send an email to each person or update each record individually.

**הערה**: אריך כרגע בגרסת בטא. הפעל אותו בהגדרות → שחרור → מעבדה.

### Filter

מסנן רשימה של רשומות על פי תנאים מוגדרים, ומאפשר רק לרשומות שעומדות בקריטריונים לעבור הלאה.

**הגדרה**:

- Select the array of records to filter
- הגדר תנאי סינון וקריטריונים
- הגדר אילו רשומות צריכות לעבור לשלבים הבאים

**תפוקה**: מחזיר רק את הרשומות שעומדות בתנאי הסינון שהוגדרו.

### שלח דוא"ל

שולח הודעת דוא"ל מזרימת העבודה שלך.

**דרישות מקדימות**: הוסף חשבון דוא"ל בהגדרות > חשבונות

**הגדרה**:

- הזן כתובת דוא"ל של הנמען
- הגדר שורת נושא
- ערוך את תוכן ההודעה
- השתמש במשתנים משלבים קודמים לצורך התאמה אישית

**הערה**: קבצים מצורפים להודעת דוא"ל יהיו זמינים ברבעון הראשון של 2026.

### קוד

מריץ JavaScript מותאם אישית בתוך זרימת העבודה שלך.

**הגדרה**:

- כתוב קוד JavaScript בעורך
- גש למשתנים משלבים קודמים
- החזר משתנים לשימוש בשלבים הבאים
- בדוק את הקוד ישירות בשלב

**Access**: Manage API keys in Settings → API & Webhooks

### טופס

מגיש טופס במהלך ביצוע זרימת העבודה כדי לאסוף קלט מהמשתמש.

**הגדרה**:

- הגדר שדות קלט עם סוגים, תוויות ומקומות מחזיקים
- הגדר כללי אימות
- הגדר כותרת ותיאור לטופס

**תפוקה**: התגובות לטופס זמינות לשימוש בשלבים הבאים.

<Warning>
טפסים מתוכננים כרגע להטריגרים ידניים בלבד. בזרימות עבודה עם טריגרים אחרים (רשומה נוצרה, עודכנה), טפסים זמינים רק דרך ממשק ריצת הזרימה, וזה לא מה שחויית המשתמש מצפה לה. מרכז ההודעות ישוחרר ב-2026 כדי לתמוך בטפסים בזרימות עבודה אוטומטיות.
</Warning>

### בקשת HTTP

שולח בקשה ל-API חיצוני כחלק מזרימת העבודה שלך.

**הגדרה**:

- הזן את כתובת ה-URL של ה-API
- בחר שיטת HTTP (GET, POST, PUT, PATCH, DELETE)
- הוסף כותרות וערכים נדרשים
- הוסף גוף בקשה לבקשות POST/PUT/PATCH
- ספק תגובה לדוגמה לתצוגת מבנה

## ניהול זרימת עבודה

### יצירת זרימות עבודה

1. לחץ על "+ הוסף זרימת עבודה" כדי להתחיל
2. לחץ על "ללא שם" כדי לקרוא לזרימת העבודה שלך
3. בחר והגדר את הטריגר לזרימת העבודה שלך
4. הוסף פעולות לזרימת העבודה שלך
5. בדוק והמשך לאשש
6. הפעל את זרימת העבודה שלך (כרגע במצב טיוטה) ברגע שתסיים לערוך אותו

**הערה**: אם אינך רואה את הקטע של זרימות עבודה, הדבר נובע מבעיית הרשאות. פנה למנהל מרחב העבודה שלך כדי להעניק לך גישה לזרימות עבודה.

### סטטוסים של זרימת עבודה

- **טיוטה**: נערך, לא פורסם עדיין
- **Active**: Live version responding to triggers
- **מושבת**: היה פעיל, אך נחסם ידנית
- **מאורכב**: גרסאות עבר נשמרות לצורכי היסטוריה

### הפעלת זרימות עבודה

לחץ על **הפעל** כדי לפרסם את הטיוטה שלך כגרסה חדשה. זה מאפשר לזרימת העבודה לפעול כאשר מפעילים אותו, אך לא מבצע אותו מיד.

### בודק של זרימות עבודה

בדוק זרימות עבודה לפני הפעלתן באמצעות:

- מפעילים ידניים (כאשר לא נבחרה רשומה)
- בדיקת פעולה אישית (במיוחד פעולות קוד)
- בדיקה במצב טיוטה שאינה מפעילה את זרימת העבודה

### הרצת זרימה של עבודה

**הרצה** היא רשומה של ביצוע זרימת עבודה הכוללת:

- סטטוס (הצלחה, נכשל, רץ)
- נתונים של תפוקה מכל שלב
- Author and timestamps
- הודעות שגיאות אם חלים

**צפייה בהרצות**:

- בדוק את לוח **הרצות** בעורך הזרימות
- פתח את תצוגת **הרצות זרימת העבודה** למעקב בכל זרימות העבודה

**טיפ לביצועים**: הסתר הרצות זרימת עבודה מהדף ״כל זרימות העבודה״ ומהדפים האחרים של זרימות העבודה כדי לשפר את מהירות הטעינה, כמויות גדולות של הרצות יכולות להאט את טעינת הדף.

### תולדות גרסאות

- ראה את כל הגרסאות תחת שדה **גרסאות**
- לחץ על כל גרסה כדי לצפות בפרטים
- השתמש ב**השתמש כטיוטה** כדי לשחזר גרסאות קודמות
- Handle draft conflicts with override or return options

## המלצות

### ארגון זרימת עבודה

- **שמות תיאוריים**: השתמש בשמות זרימת עבודה ברורים ומוגדרים
- **שמות שלבים**: שנה את שמות השלבים כדי לתאר את תפקידם
- **תיעוד**: הוסף הערות בפעולות קוד
- **קטלוג**: קבוצת זרימות עבודה קשורות בצורה לוגית
- **שדות מותאמים אישית**: הוסף שדות לאובייקט זרימת העבודה במודל הנתונים שלך (בדומה לאובייקטים אחרים) כדי לארגן ולהקטלג זרימות עבודה באמצעות תכונות מותאמות אישית

### שיפור ביצועים

- **צמצם קריאות API**: קבץ פעולות כשאפשרי
- **חיפושים יעילים**: השתמש בקריטריונים ספציפיים לסינון
- **טיפול בשגיאות**: תכנן תרחישים של כשל
- **הגבלת קצב**: שמור על מגבלות של API חיצוניים

### ניהול זרימה של נתונים

- **הגיון הסתעפות**: השתמש בהסתעפויות לאחר חיפוש רשומות
- **שימוש במשתנים**: השתמש בנתונים משלבים קודמים
- **אימות נתונים**: אמת קלטים בפעולות קוד
- **מיפוי שדות**: תכנן בקפידה את השנויים בנתונים

### ניטור ותחזוקה

- **Regular Monitoring**: Check workflow runs for errors
- **סקירה של הביצועים**: נתח זמני ביצוע ושיעורי הצלחה
- **ניהול עדכונים**: בדוק שינויים בטיוטה לפני הפעלתם
- **Team Coordination**: Document workflows for team members

לדוגמאות מעשיות של התכונות האלה בפעולה, ראה המדריכים שלנו [אוטומציות פנימיות](/l/he/user-guide/workflows/internal-automations) ו[שילוב כלים חיצוניים](/l/he/user-guide/workflows/external-tool-integration).
