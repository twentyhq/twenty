קרדיטים לתהליכי עבודה מספקים כוח לאוטומציות שלך ב-Twenty. הבנה כיצד הם עובדים עוזרת לך לייעל עלויות ולנהל את תקציב האוטומציה שלך ביעילות.

## הקצאת קרדיטים לפי תוכנית

קרדיטים לתהליכי עבודה מוקצים לפי מרחב עבודה בהתאם לתוכנית המנוי שלך.

- **ניסיון**: 5 מיליון קרדיטים
- **תוכנית מקצועית**: 10 מיליון קרדיטים בחודש
- **תוכנית ארגונית**: 20 מיליון קרדיטים בחודש

## How Credit Consumption Works

Credits are consumed when workflows execute, not when you create them. כל פעולה בתהליך צורכת קרדיטים בהתאם למורכבותה.

### צריכת קרדיטים על פי סוג פעולה

- **פעולות פנימיות בסיסיות**: צריכת קרדיט נמוכה מאוד
  - חיפוש רשומות
  - יצירת רשומה
  - עדכון רשומה
  - מחיקת רשומה
  - פעולות טופס

- **פעולות מורכבות**: צריכת קרדיט גבוהה יותר
  - פעולות קוד (JavaScript)
  - בקשות HTTP לשירותים חיצוניים

- **תכונות בינה מלאכותית**: צריכת קרדיט משמעותית (יגיע בקרוב)
  - AI prompts and processing will consume credits based on usage

### ניכוי בזמן אמת

Credits are deducted in real-time as workflows execute. זאת אומרת:

- Draft workflows don't consume credits
- רק תהליכים פעילים משתמשים בהקצאת הקרדיט שלך
- תהליכים שנכשלו עדיין צורכים קרדיטים עבור שלבים שהושלמו

## ניהול קרדיטים

### בדוק שימוש בקרדיטים

1. גש ל-**הגדרות → חיוב**
2. ראה את צריכת הקרדיטים הנוכחית שלך והמאזן שנותר
3. ניטור דפוסי שימוש על מנת לייעל את התהליכים שלך

### רכישת קרדיטים נוספים

אם אתה זקוק ליותר קרדיטים מעבר להקצאת התוכנית שלך:

1. גש ל-**הגדרות → חיוב**
2. לחץ על האפשרות לרכוש קרדיטים נוספים. חבילות בגדלים שונים זמינות.
3. קרדיטים מתווספים ליתרה הנוכחית שלך

## שיטות עבודה מומלצות

### עיצוב תהליכים יעיל

- **התחל פשוט**: התחל עם פעולות בסיסיות והוסף מורכבות בהדרגה
- **בדוק בטיוטה**: בדוק את התהליכים באופן יסודי לפני הפעלתן כדי למנוע בזבוז קרדיטים על טעויות
- **מזער בקשות HTTP**: השתמש בקריטריוני חיפוש יעילים ושילוב פעולות כשאפשרי
- **Batch Processing**: Use bulk operations and Iterator actions efficiently
- **Error Handling**: Implement proper error handling to prevent unnecessary retries
- **Manual Trigger Optimization**: For manual triggers, choose `Bulk` availability to process multiple records in a single workflow run (see [Workflow Features](/l/he/user-guide/workflows/workflow-features) for details)

### אופטימיזציה של פעולות

- Use basic internal operations when possible (lower credit consumption)
- ייעל פעולות קוד ליעילות
- שקול טריגרים ידניים לתהליכים לא דחופים
- עבד פעולות במנה כדי לצמצם קריאות פעולה בודדות

### ניהול קרדיטים

- **Regular Monitoring**: Check your credit usage weekly in Settings → Billing
- **מעקב שימוש**: עקוב אחרי דפוסי צריכה כדי למנוע הפתעות בלתי צפויות
- **תכנן בהתאם**: שקול שדרוג התוכנית אם אתה זקוק באופן קבוע ליותר קרדיטים
- **סקירת תהליכים**: סקור תהליכים עתירי קרדיטים והשבת תהליכים שאינם בשימוש
- **תכנון אסטרטגי**: תכנן פריסת תהליכים על בסיס זמינות קרדיטים

לפרטים מלאים על חיוב וניהול קרדיטים, בקר ב-**הגדרות → חיוב** במרחב העבודה שלך ב-Twenty.
