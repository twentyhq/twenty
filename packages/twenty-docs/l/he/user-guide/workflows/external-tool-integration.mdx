להלן דוגמאות לתהליכי עבודה שתוכלו לפרוס כדי לחבר את Twenty לשאר המחסנית שלכם.

## מקרים של שימוש בהכנסת נתונים

### הגשת טופס אינטרנט

**בעיה**: אתה צריך לתפוס לידים מדפי אתר, עמודי נחיתה או טפסים ישירות לתוך Twenty.

**פתרון**: השתמש בהפעלות Webhook כדי ליצור אוטומטית רשומות מהגשות טופס.

**הגדרה**:

- צור זרימת עבודה עם הפעלת Webhook
- הגדר את ה-webhook לצפות לנתוני טופס (שם, אימייל, חברה וכו')
- הגדר את שיטת ה-webhook ל-POST
- הגדר את מבנה הגוף המיוחל בהפעלה

**הפעלה**: Webhook (בקשת POST מהטופס שלך)

**פעולות**:

- חיפוש רשומות כדי לבדוק אם אדם/חברה כבר קיימים
- ענף: קיים → עדכון רשומה, לא קיים → יצירת רשומה
- צור רשומה למשימה מעקב המוקצית לנציג מכירות
- שלח הודעת דוא"ל לצוות המכירות

### סנכרון נתוני מוצר

**בעיה**: צוות המכירות שלך זקוק ליכולת לראות נתונים על שימוש במוצר, חיובים או אימוץ פיצ'רים המאוחסנים במחסן הנתונים שלך.

**פתרון**: סנכרן באופן שוטף נתוני מוצר ל-Twenty כדי לתת למוכרים הקשר על חשבונותיהם.

**הפעלה**: לפי לוח זמנים (יומי או שבועי)

**פעולות**:

- בקשת HTTP ל-API של מחסן הנתונים שלך
- בצע פעולה כדי לעבד ולפרמט את הנתונים
- השתמש בפונקציית ה-Iterator לשלבים הבאים
- חיפוש רשומות למציאת רשומות חברה תואמות
- עדכן רשומה כדי להוסיף מדדים לשימוש במוצר
- צור רשומה למשימות כששימוש נופל מתחת לסף

### הערות פגישה ממקליטי שיחות

**בעיה**: תובנות חשובות מהשיחות עם הלקוחות נאבדות או אינן מתועדות באופן נאות ב-CRM.

**פתרון**: יצור אוטומטי של הערות ופריטים פעולה ממערכות הקלטת שיחות.

**הפעלה**: Webhook (ממערכת הקלטת שיחות)

**פעולות**:

- בצע פעולה כדי לחלץ סיכום ומרכיבים לפגישה
- חיפוש רשומות למציאת ההזדמנות או איש הקשר הקשור
- צור רשומה עבור הערה עם סיכום פגישה
- צור רשומה למשימות מעקב מבוססות על פריטים פעולה
- שלח הודעת דוא"ל למשתתפים עם סיכום והשלבים הבאים

### העשרת נתונים

**בעיה**: לרשומות לקוח וחברה שלך חסר מידע דמוגרפי ופירמוגרפי חשוב.

**פתרון**: העשרה אוטומטית של רשומות באמצעות ספקי מידע חיצוניים.

**הפעלה**: רשומה נוצרת (אובייקט אנשים או חברות)

**פעולות**:

- בקשת HTTP ל-API להשלמה
- בצע פעולה כדי לעבד את תגובת ההשלמה
- השתמש בפונקציית ה-Iterator לשלבים הבאים
- עדכן רשומה עם מידע חברה/איש קשר נוסף
- צור רשומה למשימת מכירה אם זוהה לקוח פוטנציאלי בעל ערך גבוה
- שלח הודעת דוא"ל אם העשרה מגלה סימני קנייה מפתחיים

## מקרים לשימוש בהפצת נתונים

### ניהול מנויים לניוזלטר

**Problem**: You want to send marketing emails to specific segments of your CRM data using specialized email tools.

**פתרון**: ייצא רשימות מנויים לפלטפורמת שיווק האימייל שלך לפי הצורך.

**הגדרה**: צור תצוגה ב-Twenty עם כל מקבלי הניוזלטר

**הפעלה**: השקה ידנית (כאשר לא נבחרה רשומה)

**פעולות**:

- חיפוש רשומות באמצעות קריטריונים של תצוגת הניוזלטר
- Code action to format email addresses for your email platform
- בקשת HTTP להוספת מנויים לכלי שיווק האימייל שלך
- יצירת רשומה למעקב אחר קמפיינים
- Send Email confirmation to marketing team

### Email Sequence Triggers

**Problem**: You want to trigger sophisticated email sequences based on CRM events using dedicated email automation tools.

**פתרון**: שלח לקוחות חדשים או פוטנציאליים לפלטפורמת האוטומציה של המיילים שלך כאשר מתרחשים אירועים מסוימים.

**הפעלה**: רשומה נוצרת (אובייקט אנשים עם קריטריונים מסוימים)

**פעולות**:

- Code action to determine appropriate email sequence
- בקשת HTTP להוספת איש קשר לפלטפורמה לאוטומציה של מיילים
- עדכון רשומה למעקב אחר הרישום בסדרה
- צור רשומה למשימת מעקב לניטור מעורבות

### אינטגרציה של ניקוד ליד

**בעיית**: אתה זקוק לניקוד ליד מורכב שמשלב נתוני CRM עם איתותים חיצוניים.

**פתרון**: שלח נתוני ליד לכלים חיצוניים לניקוד או יישם לוגיקת ניקוד בתוך זרימות עבודה.

**אפשרות 1 - כלי חיצוני**:

**הפעלה**: רשומה מתחדשת (אובייקט אנשים)

**פעולות**:

- בקשת HTTP לשלוח נתוני ליד לפלטפורמת ניקוד
- פעולת קוד לעיבוד תגובת הניקוד
- עדכן רשומה עם ניקוד ליד
- צור רשומה למשימות מכירה אם הניקוד עולה על הרף

**אפשרות 2 - לוגיקה פנימית**:

**הפעלה**: רשומה מתחדשת (אובייקט אנשים)

**פעולות**:

- פעולת קוד עם אלגוריתם ניקוד (גודל חברה, תעשייה, התנהגות)
- Update Record with calculated score
- שלח התרעת אימייל לנציג מכירות עבור לידים עם ניקוד גבוה

### יצירת חשבוניות

**בעיית**: כאשר עסקאות נסגרות, מערכת החיוב שלך צריכה להתעדכן במידע על הלקוח והעסקה.

**פתרון**: שלח באופן אוטומטי נתוני עסקה למערכת החשבוניות שלך כאשר הזדמנויות נסגרות בהצלחה.

**Trigger**: Record is Updated (Opportunities object, Stage = "Closed Won")

**פעולות**:

- חפש רשומות לקבלת מידע לקוח מלא
- פעולת קוד לעיצוב נתונים למערכת חיוב
- בקשת HTTP ליצירת לקוח בפלטפורמת חיוב
- בקשת HTTP ליצירת חשבונית
- Update Record to store invoice reference
- שלח אימייל לצוות הכספים עם פרטי החשבונית

## תבניות אינטגרציה מתקדמות

### סנכרון דו-כיווני

**בעיית**: אתה צריך לשמור את הנתונים מסונכרנים בין Twenty לבין מערכת אחרת לשני הכיוונים.

**Solution**: Combine scheduled workflows with webhook triggers for real-time sync.

**מ-Twenty למערכת חיצונית**:

**הפעלה**: רשומה מתחדשת (כל אובייקט רלוונטי)

**פעולות**:

- בקשת HTTP לעדכון מערכת חיצונית
- עדכן רשומה למעקב אחר מצב סינכרון וזמן

**ממערכת חיצונית ל-Twenty**:

**Trigger**: Webhook (from external system)

**פעולות**:

- חיפוש רשומות כדי למצוא רשומה תואמת
- עדכן רשומה עם נתונים חדשים ממערכת חיצונית
- צור רשומה למשימת פתרון קונפליקטים אם יש צורך

### עיבוד נתונים מרובי שלבים

**Problem**: Data from external sources needs complex processing before it can be used in Twenty.

**פתרון**: השתמש בפעולות קוד עבור שינוי מבנה הנתונים ואימותם.

**הפעלה**: Webhook או בזמנים מתוזמנים

**פעולות**:

- פעולת קוד לאימות מבנה נתוני הכניסה
- פעולת קוד לשינוי מבנה הנתונים
- פעולת קוד ליישום כללי עסקים וחישובים
- חיפוש רשומות כדי לבדוק על כפילויות
- צור או עדכן רשומה עם נתונים מעובדים
- Send Email alert if data quality issues detected

## טיפים ליישום

- שמור מפתחות API בצורה בטוחה בהגדרות → API & Webhooks
- השתמש ב-HTTPS לכל הקריאות API חיצוניות
- שמור על מגבלות קצבי API - השתמש בזרימות עבודה מתוזמנות כשניתן
- שקול עדכונים בקבוצות "בתזמון מתכונן" כשעיבוד בזמן אמת אינו נדרש
- זכור את מגבלת 100 זרימות עבודה מקבילות לכל סביבה - השתמש ב"זמינות רחבה" עבור מופעלים ידניים כאשר מעבדים מספר רשומות
- בדוק עם נתוני דוגמה לפני הפעלת הזרימות
