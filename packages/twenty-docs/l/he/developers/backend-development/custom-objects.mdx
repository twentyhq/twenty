עצמים הם מבנים המאפשרים לך לאחסן נתונים (רשומות, תכונות וערכים) ספציפיים לארגון. Twenty מספקת גם עצמים סטנדרטיים וגם עצמים מותאמים אישית.

עצמים סטנדרטיים הם עצמים מובנים עם סדרת תכונות זמינות לכל המשתמשים. דוגמאות לעצמים סטנדרטיים ב-Twenty כוללות חברה ואדם. לעצמים סטנדרטיים יש שדות סטנדרטיים שגם הם זמינים לכל משתמשי Twenty, כמו Company.displayName.

עצמים מותאמים אישית הם עצמים שאתה יכול ליצור כדי לאחסן מידע ייחודי לארגון שלך. הם אינם מובנים; חברי סביבת העבודה שלך יכולים ליצור ולהתאים אישית עצמים מותאמים אישית כדי להכיל מידע שעצמים סטנדרטיים אינם מתאימים לו.

## איך זה עובד

עצמים מותאמים אישית מגיעים מטבלאות מטהדאטה שקובעות את הצורה, השם והסוג של העצמים. כל המידע הזה נמצא בבסיס נתונים של תרשים מטהדאטה, המורכב מטבלאות:

- **DataSource**: מפרט היכן הנתונים נמצאים.
- **Object**: מתאר את העצם ומחבר למקור מידע.
- **Field**: מתאר את שדות העצם ומחבר לעצם.

כדי להוסיף עצם מותאם אישית, יוזם סביבת עבודה ישאל את ממשק ה-API של המטהדאטה. זה מעדכן את המטהדאטה בהתאם ומחשב תרשים GraphQL המבוסס על המטהדאטה, מאחסן אותו במטמון GQL לשימוש עתידי.
