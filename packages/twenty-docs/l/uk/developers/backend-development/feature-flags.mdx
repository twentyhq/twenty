Прапори функцій використовуються для приховування експериментальних функцій. Для Twenty вони встановлюються на рівні робочої області, а не на рівні користувача.

## Додавання нового прапора функцій

У файлі `FeatureFlagKey.ts` додайте прапор функцій:

```ts
type FeatureFlagKey =
  | 'IS_FEATURENAME_ENABLED'
  | ...;
```

Також додайте його до перерахування в `feature-flag.entity.ts`:

```ts
enum FeatureFlagKeys {
    IsFeatureNameEnabled = 'IS_FEATURENAME_ENABLED',
    ...
}
```

Щоб застосувати прапор функцій для **бекенд** функції, використовуйте:

```ts
@Gate({
  featureFlag: 'IS_FEATURENAME_ENABLED',
})
```

Щоб застосувати прапор функцій для **фронтенд** функції, використовуйте:

```ts
const isFeatureNameEnabled = useIsFeatureEnabled('IS_FEATURENAME_ENABLED');
```

## Налаштування прапорів функцій для розгортання

Змініть відповідний запис у таблиці `core.featureFlag`:

| ідентифікатор | ключ                     | workspaceId | значення  |
| ------------- | ------------------------ | ----------- | --------- |
| Випадково     | `IS_FEATURENAME_ENABLED` | WorkspaceID | `істинно` |
