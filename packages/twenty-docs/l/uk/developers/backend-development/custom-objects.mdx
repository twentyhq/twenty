Об'єкти — це структури, що дозволяють зберігати дані (записи, атрибути та значення), специфічні для організації. Twenty надає як стандартні, так і користувацькі об'єкти.

Стандартні об'єкти є вбудованими об'єктами з набором атрибутів, доступних для всіх користувачів. Прикладами стандартних об'єктів у Twenty є Компанія та Особа. Стандартні об'єкти мають стандартні поля, які також доступні для всіх користувачів Twenty, наприклад, Company.displayName.

Користувацькі об'єкти — це об'єкти, які ви можете створити для зберігання інформації, унікальної для вашої організації. Вони не є вбудованими; члени вашого робочого простору можуть створювати та налаштовувати користувацькі об'єкти для зберігання інформації, для якої стандартні об'єкти не підходять.

## Як це працює

Користувацькі об'єкти походять з метаданих таблиць, які визначають форму, ім'я та тип об'єктів. Вся ця інформація міститься в базі даних схеми метаданих, що складається з таблиць:

- **Джерело Даних**: Деталі, де знаходяться дані.
- **Об'єкт**: Описує об'єкт і зв'язується з Джерелом Даних.
- **Поле**: Окреслює поля об'єкта і з'єднує їх із об'єктом.

Щоб додати користувацький об'єкт, учасник робочого простору буде запитувати API /metadata. Це відповідно оновлює метадані та обчислює схему GraphQL на основі метаданих, зберігаючи її в кеші GQL для подальшого використання.
