## Про

GitHub репозиторій Twenty містить величезний набір ресурсів, таких як вихідний код, документація, обговорення та відстеження проблем. Тут ви зможете отримати доступ до повного коду проекту Twenty.

[Відвідайте Twenty на GitHub](https://github.com/twentyhq/twenty)

## Співпраця

Співпраця в проекті Twenty на GitHub - це корисний спосіб допомогти покращити програмне забезпечення, яке ви використовуєте. Чи ви займаєтесь виправленням помилок, пропонуєте функціонали або покращуєте документацію — ваш внесок вітається.

### Reporting Issues

Зіткнулися з проблемою? [Create an issue](https://github.com/twentyhq/twenty/issues/new) on GitHub, providing as much detail as possible.

<img src="/images/user-guide/github/new-issue.png" style={{width:'100%'}}/>

### Пропозиції функцій

Які покращення ви хотіли б бачити в Twenty? Незалежно від технічних навичок, ви можете приєднатися до [обговорення тут](https://github.com/twentyhq/twenty/discussions).

<img src="/images/user-guide/github/github-conversations.png" style={{width:'100%'}}/>

### Coding a feature

Розпочніть свій шлях, знаходячи завдання для початківців:

1. Перейдіть на вкладку **[Issues](https://github.com/twentyhq/twenty/issues)** у репозиторії Twenty.
2. Відфільтруйте за міткою **[Good First Issue](https://github.com/twentyhq/twenty/labels/good%20first%20issue)**, щоб знайти завдання для новачків.
3. Виберіть проблему, форкуйте репозиторій та починайте співпрацювати.

<img src="/images/user-guide/github/good-first-issues.png" style={{width:'100%'}}/>

Переконайтеся, що ви призначені на проблему, щоб уникнути дублювання роботи з іншими учасниками.

### Кодекс поведінки

Пам’ятайте дотримуватися [Кодексу поведінки](https://github.com/twentyhq/twenty/blob/main/.github/CODE_OF_CONDUCT.md) Twenty протягом всього процесу співпраці.
