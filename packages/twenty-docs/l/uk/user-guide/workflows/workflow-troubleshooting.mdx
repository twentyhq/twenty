## Відлагодження за допомогою виконань робочого процесу

Use the **Workflow Runs** interface to debug issues:

- Доступ через панель **Виконання** в редакторі робочого процесу
- Натисніть на окремі виконання, щоб побачити дані введення/виведення для кожного кроку
- Перевірте статус виконання, повідомлення про помилки та потік даних між кроками

## Загальні проблеми та рішення

### Робочий процес не запускається

**Проблема**: Робочий процес не виконується, коли очікується.

**Рішення**:

- Переконайтеся, що робочий процес **Активний** (не в режимі чернетки)
- Перевірте відповідність конфігурації тригера вашій структурі даних
- Для тригерів запису, переконайтеся, що вибрано правильний об'єкт і поля
- Для тригерів вебхука, перевірте URL і очікуваний формат даних
- Для запланованих тригерів перевірте конфігурацію часу

### Forms Not Accessible

**Проблема**: Дії форми важко знайти або використовувати в автоматизованих робочих процесах.

<Warning>
Форми в даний момент розроблені лише для ручних тригерів. Для робочих процесів з іншими тригерами (запис створено, оновлено тощо), форми доступні лише через інтерфейс запуску робочого процесу, що не є очікуваним досвідом користувача. Центр повідомлень буде випущено у 2026 році для належної підтримки форм в автоматизованих робочих процесах.
</Warning>

**Вирішення**: Використовуйте ручні тригери, коли потрібні форми, або реорганізуйте робочі процеси, щоб уникнути форм в автоматизованих потоках.

### Високе споживання кредитів

**Проблема**: Робочі процеси споживають більше кредитів, ніж очікувалося.

**Загальні причини та рішення**:

- **Неоптимальні виклики API**: Здійснюйте пакетні виклики API, коли це можливо, замість окремих запитів
- **Неправильна конфігурація ручного тригера**: Використовуйте доступність `Bulk` замість `Single` для обробки декількох записів під час одного виконання робочого процесу
- **Відсутність фільтрів**: Додайте умовну логіку, щоб зупиняти робочі процеси, коли критерії не відповідають
- **Зайві кроки**: Видаліть зайві дії та оптимізуйте логіку робочого процесу
- **Реальний час проти запланованих**: Використовуйте заплановані робочі процеси для не термінових процесів

### Обмеження одночасних робочих процесів

**Проблема**: Досягнуто ліміту 100 одночасних робочих процесів на робочу область.

<Warning>
You cannot run more than 100 workflows in parallel at any given time per workspace.
</Warning>

**Рішення**:

- Використовуйте доступність `Bulk` для ручних тригерів для обробки декількох записів під час одного запуску
- Реалізуйте затримки між виконаннями робочого процесу, використовуючи заплановані тригери
- Оптимізуйте робочі процеси, щоб вони виконувалися швидше та знижували час одночасного виконання
- Розгляньте пакетну обробку поза піковими годинами

### Ліміт API

**Проблема**: Зовнішні виклики API зазнають невдач через обмеження на ліміти.

**Рішення**:

- Використовуйте заплановані робочі процеси замість режиму реального часу, коли це можливо
- Реалізуйте затримки між викликами API в кодових діях
- Використовуйте пакетні запити API, коли зовнішній сервіс їх підтримує
- Моніторьте виконання робочих процесів на помилки ліміту і регулюйте час

### Проблеми ітератора

**Проблема**: Дії ітератора працюють не так, як очікувалося.

**Рішення**:

- **Примітка**: Ітератор поки що в бета-версії. Активуйте його в Налаштуваннях → Релізи → Лабораторія
- Переконайтеся, що вхідні дані є масивом записів з попереднього кроку
- Перевірте, чи налаштовані дії в ітераторі правильно
- Використовуйте ітератор з `Bulk` ручними тригерами для оптимальної продуктивності

### Відсутність прав доступу

**Проблема**: Не можливо отримати доступ до секції робочих процесів.

<Warning>
If you don't see the Workflows section, this is due to a permissions issue. Зверніться до адміністратора вашої робочої області для надання доступу до робочих процесів.
</Warning>

## Поради з оптимізації

### Кращі практики продуктивності

- **Почніть з простого** і поступово додайте складність
- **Use `Bulk` availability** for manual triggers to process multiple records efficiently
- **Додайте фільтри на початку**, щоб зупинити робочі процеси, коли критерії не виконані
- **Пакуйте виклики API** і використовуйте заплановані робочі процеси для не термінових процесів
- **Приховуйте виконання робочого процесу** з сторінок, що відображають усі робочі процеси, для покращення продуктивності завантаження
- **Контролюйте використання кредитів** регулярно в Налаштуваннях → Білінг

### Запобігання помилок

- **Тестуйте в режимі чернетки** перед активацією робочих процесів
- **Перевірте відповіді API** і реалізуйте резервні дії
- **Використовуйте описові назви кроків** для зручнішого обслуговування
- **Документуйте складну логіку** для членів команди
