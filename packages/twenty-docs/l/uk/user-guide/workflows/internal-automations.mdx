Нижче наведено приклади робочих процесів, які ви можете розгорнути для автоматизації повторюваних завдань, щоб зменшити ручну роботу та підтримувати ваші дані в актуальному стані.

## Автоматизації Управління Даними

### Авто-Позначення Особистих Електронних Адрес

**Проблема**: Ваша команда повинна ідентифікувати особисті та робочі електронні адреси для кращого визначення цільової аудиторії та відповідності.

**Solution**: Create a workflow that automatically updates an "is personal email" field whenever an email address is added or updated.

**Trigger**: Record is Updated (People object, Email field)

**Дії**:

- Code action to check if email domain matches common personal providers (gmail.com, yahoo.com, etc.)
- Update Record to set the "is personal email" flag

### Lead Assignment - Round Robin

**Проблема**: Нові ліди повинні бути справедливо розподілені по вашій команді продажів, щоб забезпечити збалансоване робоче навантаження.

**Solution**: Automatically assign new leads to sales reps using a round-robin system.

**Тригер**: Запис Створено (Об'єкт Люди)

**Дії**:

- Пошук Записів для знаходження останнього призначеного представника
- Дія коду для визначення наступного представника в ротації
- Оновити Запис для призначення ліда виділеному представнику
- Надіслати Електронний Лист призначеному представнику

### Lead Assignment - Territory Based

**Проблема**: Ліди повинні бути призначені на основі географічних територій або характеристик компанії.

**Рішення**: Перенаправляти ліди до відповідного представника з продажу на основі розташування, розміру компанії або галузі.

**Тригер**: Запис створено (Об'єкт «Люди» або «Компанії»)

**Дії**:

- Code action to determine territory based on location/industry rules
- Search Records to find the territory owner
- Оновити Запис для призначення ліда
- Створити Запис для завдання щодо подальших дій

## Автоматизації Процесу Продажів

### Opportunity Stage Management - Closed Won

**Проблема**: Коли угоди закриваються, потрібні декілька ручних оновлень по різних записах та учасниках команди.

**Рішення**: Автоматично обробляти всі дії після перемоги, коли можливість переходить у "Закрито Переможений".

**Trigger**: Record is Updated (Opportunities object, Stage field = "Closed Won")

**Дії**:

- Оновити Запис для зміни типу Компанії з "Перспектива" на "Клієнт"
- Створити Запис для завдань ініціації, призначених менеджеру з облікових записів
- Надіслати Електронний Лист команді по роботі з клієнтами
- HTTP Request to update external billing system

### Opportunity Stage Management - Closed Lost Renewal

**Проблема**: Коли можливості для продовження втрачаються, потрібно оновити статус клієнта для належного управління обліковим записом.

**Рішення**: Автоматично оновлювати статус клієнта, коли угоди з продовженням втрачаються.

**Тригер**: Запис Оновлено (Об'єкт Можливості, Етап = "Закрито Втрачено" ТА Тип = "Продовження")

**Дії**:

- Оновити Запис для зміни типу Компанії з "Клієнт" на "Клієнт, що відтік".
- Створити Запис для завдання з аналізу відтоку
- Надіслати Електронний Лист попередження менеджеру по роботі з клієнтами
- Оновити Запис для додання дати відтоку та причини.

### Попередження про Застоялі Можливості

**Проблема**: Можливості залишаються без оновлень, що призводить до їх охолодження та застарілих прогнозів.

**Рішення**: Надсилати автоматичні попередження, коли можливості давно не оновлювались.

**Тригер**: За Розкладом (щодня)

**Дії**:

- Пошук Записів для можливостей, не оновлених за Х днів
- Дія коду для форматування повідомлення попередження з деталями можливості
- Надіслати Електронний Лист власнику можливості та керівнику
- Створити Запис для завдання щодо подальших дій у разі відсутності відповіді

## Автоматизації Продуктивності

### Щотижневий Підсумок Завдань

**Problem**: Team members lose track of their upcoming tasks and deadlines.

**Рішення**: Надсилати автоматизовані щотижневі нагадування електронною поштою з короткими викладами завдань.

**Тригер**: За Розкладом (щопонеділка о 8 ранку)

**Дії**:

- Пошук Записів для завдань, які мають бути виконані цього тижня за призначеним відповідальним
- Code action to format task list by person
- Надіслати Електронний Лист кожному члену команди з їх підсумком завдань
- Надіслати Електронний Лист керівникам з оглядом команди

### Автоматизація Слідкування за Зустрічами

**Проблема**: Важливі питання після зустрічей забуваються або відкладаються.

**Рішення**: Автоматично створювати завдання для слідкування, коли зустрічі заплановані або завершені.

**Тригер**: Запис Створено (Об'єкт Активності, Тип = "Зустріч")

**Дії**:

- Створити Запис для завдання по підготовці до зустрічі
- Створити Запис для завдання по слідкуванню після зустрічі
- Надіслати Електронний Лист-нагадування учасникам
- Оновити Запис для прив'язки завдань до зустрічі

### Синхронізація Полів Між Об'єктами

**Проблема**: Вам потрібна інформація з пов'язаних записів легко доступною (наприклад, електронна адреса головного контакту на картці можливості).

**Рішення**: Автоматично синхронізувати поля між пов'язаними об'єктами, поки структуровані поля не стануть доступними.

**Тригер**: Запис Оновлено (Об'єкт Можливості, Поле Контактна Особа)

**Дії**:

- Пошук Записів для знаходження деталей пов'язаної особи
- Update Record to copy email address to opportunity
- Update Record to copy phone number to opportunity
- Оновити Запис для копіювання інформації про компанію

## Перевірка Даних та Очищення

### Уніфікація Формату Номера Телефону

**Проблема**: Номери телефонів вводяться в різних форматах, що ускладнює їх використання для дзвінків або обміну повідомленнями.

**Рішення**: Автоматично форматувати номери телефонів до стандартного формату під час введення.

**Тригер**: Запис Оновлено (Об'єкт Люди, Поле Телефон)

**Дії**:

- Дія коду для розбору та форматування номера телефону
- Оновити Запис зі стандартизованим форматом телефону
- Оновити Запис для додавання коду країни, якщо він відсутній
