## Перед початком

### 1. Audit Your Current Data

- **Обирайте лише декілька об'єктів та полів для міграції**: ця міграція - шанс для нового початку
- **Експортуйте ці дані** зі своєї поточної CRM
- **Видалити дублікати** та застарілі записи
- **Перелічіть активні робочі процеси** та автоматизації

### 2. Створіть нову модель даних

Слідуйте нашому [посібнику з моделювання даних](/l/uk/user-guide/data-model/customize-your-data-model) для:

- **Спроектуйте модель даних**, яка вам потрібна
- **Співвіднесіть наявні поля** зі стандартними об'єктами/полями Twenty
- **Визначте користувацькі об'єкти/поля**, які вам знадобляться
- **Створіть їх** у Розділі для налаштувань → Модель даних

## Процес міграції

### 1. Import Your Data

**Рекомендований порядок:**

1. **Спершу компанії** (як базові записи)
2. **Потім люди** (що пов'язані з компаніями)
3. **Третє - можливості** (пов'язані з людьми/компаніями)

Використовуйте імпорт CSV через Меню команд `Cmd + K` (Mac) або `Ctrl + K` (Windows). Дивіться наш [посібник з імпорту даних](/l/uk/user-guide/getting-started/import-export-data) для детальних інструкцій.

### 2. Відновіть робочі процеси

- **Почніть з простого** - відновіть найважливіші автоматизації
- **Використовуйте будівельник робочих процесів Twenty** для заміни існуючих автоматизацій

## Поширені труднощі

### Проблеми з форматуванням даних

- **Електронні адреси** - видалити дублікати (вимога об'єкта Люди)
- **Домен** - видалити дублікати (вимога об'єкта Компанії)

<Warning>
Please note that domain URLs created by the synchronization with your mailbox and calendar have the following format ```https://domain.com```
</Warning>

- **Date formats** - ensure consistent formatting (YYYY-MM-DD) or edit this format under Settings → Experience
- **Номери телефонів** - використовуйте міжнародний формат (+1234567890)

### Relationship Mapping

To import relations between records using the csv import function, you can use the following fields

- **Використовуйте Twenty ідентифікатори** для складних відносин
- **Використовуйте електронні адреси** для зв'язування записів Людей
- **Використовуйте доменне ім'я** для зв'язування записів Компанії
- **Використовуйте будь-яке інше поле, яке ви встановили як унікальне**, зробити це можна в розділі Модель даних.
  Читайте наш [посібник з імпорту-експорту даних](/l/uk/user-guide/getting-started/import-export-data) для детальних інструкцій по створенню відносин під час імпорту CSV.

## Міграція з власного розміщення до хмари

Якщо ви переходите з Twenty власного розміщення на Twenty Cloud:

1. **Експортуйте ваші дані** з вашого власного примірника
2. **Виконуйте стандартний процес міграції** вище
3. **Ми можемо надати допомогу у міграції**, зверніться до нашої команди
