## Управління об’єктами

<AccordionGroup>
<Accordion title="Can I reorder objects in the left navigation bar?">
Ще ні. Порядок об'єктів у навігації наразі фіксований, але ця функція запланована на майбутнє оновлення.
</Accordion>

<Accordion title="Can I hide objects from the left navigation bar?">
Усі активні об’єкти відображаються в навігації. Ви можете деактивувати об’єкти, які вам не потрібні, у **Налаштування → Модель даних**.
</Accordion>

<Accordion title="Can I delete standard objects (People, Companies, etc.)?">
You can deactivate any standard objects but you cannot hard delete them.
</Accordion>
</AccordionGroup>

## Можливості поля

<AccordionGroup>
<Accordion title="Can I create formula fields?">
Поля формул будуть доступні в **Q1 2026**. Тим часом ви можете використовувати робочі процеси для автоматичного обчислення та оновлення значень полів.
</Accordion>

<Accordion title="Can I have nested fields in my objects?">
Вкладені поля будуть доступні в **Q1 2026**. Наразі ви можете використовувати робочі процеси для отримання значень полів з пов’язаних об’єктів. Наприклад, щоб відобразити індустрію компанії у записі Особа, створіть власне поле в Людях та використовуйте робочий процес для синхронізації значень.
</Accordion>

<Accordion title="Why can't I update relation field names?">
Імена зв’язкових полів впливають на структуру API і не можуть бути змінені після створення. Якщо вам потрібно перейменувати зв’язкове поле, вам потрібно буде створити нове і видалити старе.
</Accordion>

<Accordion title="Why do I need different singular and plural names?">
Our GraphQL API uses both forms for different operations:
- `createPerson` (singular) for single record actions
- `createPeople` (plural) for bulk operations

Це створює обмеження, коли однина і множина однакові, але покращує досвід розробника. </Accordion>

<Accordion title="Why are some field names protected?">
Деякі імена полів, такі як `Тип` або `Застосування`, зарезервовані для системного використання. Виберіть альтернативні назви, такі як `Категорія` або `Класифікація`.
</Accordion>
</AccordionGroup>

## Просунуті функції

<AccordionGroup>
<Accordion title="Can I create many-to-many relationships?">
Відношення багато до багато будуть доступні у **Q1 2026**. Наразі створіть проміжний об'єкт із двома 1-до-багато відношеннями як обхідний шлях.
</Accordion>

<Accordion title="What are Morph Many relationships?">
Морфологічні відношення багато до багато дозволяють одному об'єкту відноситись до різних типів об'єктів. Наприклад, можливість може стосуватися або особи, або компанії. Ця функція зараз доступна.
</Accordion>

<Accordion title="Can I reorder fields in objects?">
Зміна порядку полів буде доступна з користувацькими макетами у **Q4 2025**. Наразі поля відображаються в алфавітному порядку.
</Accordion>
</AccordionGroup>

## Доступ і права

<AccordionGroup>
<Accordion title="Where can I see and edit my data model?">
You can access your Data Model under **Settings → Data Model**.
</Accordion>

<Accordion title="Why can't I see the Data Model under Settings?">
Зверніться до адміністратора вашого робочого простору. Доступ до моделі даних зазвичай обмежений лише для адміністраторів.
</Accordion>

<Accordion title="How many custom objects or fields can I create?">
You can create as many custom objects and fields as you need - the price won't change.
</Accordion>
</AccordionGroup>
