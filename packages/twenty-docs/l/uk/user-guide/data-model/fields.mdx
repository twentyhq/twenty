## Про поля

Поля схожі на стовпці в електронних таблицях. Вони зберігають різні типи даних, такі як текст, числа або дати. Поля можуть бути стандартними (вбудованими) або користувацькими (створеними вами).

### Стандартні поля

Standard fields come built-in with Twenty to handle common business needs.

Наприклад, «Ім’я» та «Прізвище» є стандартними полями у об’єкті «Люди». Вони зберігають текстові дані для окремих імен.

Ви не можете видалити стандартні поля, але можете їх деактивувати, якщо вони не потрібні.

Ви також можете налаштувати опції стандартних полів типу `SELECT`, наприклад, опції для `Стадії` в Можливостях.

<img src="/images/user-guide/fields/standard-fields.png" style={{width:'100%'}}/>

### Користувацькі поля

Користувацькі поля можна додати до будь-якого об’єкта. Ви можете зберігати текст, числа, дати, вибір зі списку та інше. Використовуйте користувацькі поля для відстеження інформації, специфічної для вашого бізнесу.

For instance, a custom field for SpaceX could be `Rocket Active Status`, indicating if a rocket is operational.

<img src="/images/user-guide/fields/custom-fields.png"style={{width:'100%'}}/>

## Створити користувацьке поле

To add a custom field to any object, follow these steps:
1.	Перейдіть до «Налаштувань» у лівій боковій панелі.
2.	Перейдіть до «Модель даних», потім виберіть об'єкт, який хочете налаштувати.
3.	Продовжте, натиснувши «Додати поле».
4.	Виберіть назву та тип поля, які відповідають вашим вимогам. Розгляньте можливість додавання опису поля для кращого розуміння.

Новостворене поле тепер доступне в полях додатка. Щоб відобразити його у специфічному поданні, натисніть на меню опцій, а потім виберіть «Поля».

**Швидкий спосіб:** Натисніть кнопку **+** у верхньому правому куті будь-якої таблиці об'єктів, а потім виберіть «Налаштувати поля». Це безпосередньо направить вас до налаштувань моделі даних.

## Деактивувати поле

Ви можете деактивувати поле, щоб сховати його з додатка, не втрачаючи даних. Думайте про це як про приховування поля, а не видалення.

Ось як це можна зробити:

1. Знайдіть поле, яке хочете деактивувати у налаштуваннях об'єкта.

2. Натисніть на три крапки «⋮» поруч із полем, щоб відкрити меню.

3. Обрати «Деактивувати» із випадаючого списку.

<img src="/images/user-guide/fields/deactivate-field.png" style={{width:'100%'}}/>

Що відбувається, коли ви деактивуєте поле?

1. **В додатку:** Поле зникає, і ви не можете додавати нові значення в нього.

2. **Existing relationships:** If it's a relation field, existing connections stay but you can't create new ones.

3. **Доступ до API:** Ви все ще можете отримати доступ до поля та його даних через API.

Ви можете реактівувати стандартні та користувацькі поля або мати можливість їх постійного видалення.

## Зробити поля унікальними

Зробіть поле унікальним, щоб забезпечити, що різні записи не можуть мати однакове значення. Наприклад, електронні адреси унікальні для кожної особи.

Якщо ви отримуєте помилку при встановленні унікальності, перевірте на наявність дублюючих значень у ваших даних (включаючи видалені записи).

## Кращі практики конфігурації полів

### Кращі практики конфігурації полів

- **Назви полів відносин не можуть бути оновлені** після створення (впливає на структуру API)
- **Імена, що вказують на однину та множину, повинні відрізнятися**: Наша GraphQL API потребує різних назв для мутацій
- **Захищені назви полів**: деякі назви зарезервовані для системного використання (наприклад, `Тип`)

### Поля для валюти та телефону

- **Валюта за замовчуванням**: може бути налаштована через модель даних
- **Коди телефонних країн за замовчуванням**: можуть бути налаштовані для телефонних полів через модель даних

### Поля вибору

- **Один варіант може бути обраний за замовчуванням** для кожного поля вибору

### Поля тексту записів

- **Each object has one main display field**: This field appears in the leftmost column and represents the record when linked to other objects. Це повинно бути текстове поле. Наприклад, у «Людей» використовується «Ім'я» як основне поле, тому, коли ви пов'язуєте людину з компанією, у вигляді компанії побачите її ім'я.

### Поля відносин

- **З'єднати об'єкти разом**: Поля відносин з'єднують записи з різних об'єктів. Для детальної інформації про створення та управління відношеннями, дивіться нашу статтю [Поля відносин](/l/uk/user-guide/data-model/relation-fields).
