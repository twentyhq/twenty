## Що таке поля відносин?

Поля відносин зв'язують записи з одного об'єкта із записами в іншому об'єкті. Наприклад:

- **Люди** → **Компанії** (кожна людина працює на компанію)
- **Можливості** → **Люди** (кожна угода має контактну особу)
- **Завдання** → **Можливості** (кожне завдання відноситься до конкретної угоди)

## Створення полів відносин

### 1. Додати поле

Перейдіть до **Налаштування → Модель даних → [Ваш об'єкт]** і натисніть **Додати поле**.

### 2. Choose Relation Type

- **Тип поля**: виберіть "Відносини"
- **Target Object**: Choose which object to connect to
- **Relationship**: Currently supports **1-to-many** relationships only. Make sure to create the relationship in the right direction.

### 3. Конфігурація назв полів

Вам потрібно встановити назви для обох сторін відносин:

- **Назва вихідного поля**: як поле виглядає у вашому поточному об'єкті
- **Назва цільового поля**: як поле зворотного виглядає у цільовому об'єкті

<Warning>
Назви полів не можуть редагуватися після збереження відносин, оскільки це впливає на структуру API. Вибирайте обережно.
</Warning>

## Relating to Team Members

You can create relations to any object, including **Workspace Members** (your Twenty team users). This is useful for creating ownership fields:

- **Власник рахунку**: Зв'язати компанію з учасником команди, який нею керує
- **Власник угоди**: Призначити можливість конкретному продавцю

When you create a relation to`Workspace Members`, you'll see your team members' names in dropdown selections, making it easy to assign ownership and responsibilities.

## Кращі практики

- **Плануйте свої відносини** перед їх створенням
- **Використовуйте зрозумілі та описові назви** для обох назв полів
- **Тестуйте відносини** з тестовими даними перед повною реалізацією
