Система дозволів Twenty дозволяє контролювати доступ до трьох основних областей:

- **Об'єкти та поля**: Контролюйте, хто може переглядати, редагувати чи видаляти записи та окремі поля
- **Налаштування**: Керуйте доступом до конфігурації робочого простору та адміністративних функцій
- **Дії**: Контролюйте загальні дії в робочому просторі, такі як імпорт даних або надсилання електронної пошти

## Створити роль

Щоб створити нову роль:

1. Перейдіть у **Налаштування → Ролі**
2. У розділі **Усі ролі**, натисніть на **+ Створити роль**
3. Введіть назву ролі
4. У вкладці **Дозволи** за замовчуванням, налаштуйте дозволи
5. Натисніть **Зберегти**, щоб завершити

## Видалити роль

Щоб видалити роль:

1. Перейдіть у **Налаштування → Ролі**
2. Клацніть на роль, яку ви хочете видалити
3. Відкрийте вкладку **Налаштування**, потім натисніть **Видалити роль**
4. Натисніть **Підтвердити** у модальному вікні

Примітка: Якщо роль буде видалено, будь-якого учасника робочого простору, що має цю роль, буде автоматично перепризначено на роль за замовчуванням. Усі ролі, крім **Адміністратора**, можуть бути видалені. Має бути принаймні один учасник, призначений для ролі **Адміністратора**.

## Призначити ролі учасникам

### Переглянути поточні призначення

- Перейдіть у **Налаштування → Ролі**
- Перегляньте всі ролі та скільки учасників призначено до кожної
- Дивіться, які учасники мають які ролі

### Призначити роль учаснику

1. Перейдіть у **Налаштування → Ролі**
2. Клацніть на роль, яку хочете призначити
3. Відкрийте вкладку **Призначення**
4. Натисніть **+ Призначити учаснику**
5. Обрати учасника робочого простору зі списку
6. Підтвердіть призначення

### Установити роль за замовчуванням

1. Перейдіть у **Налаштування → Ролі**
2. У розділі **Опції** знайдіть **Роль за замовчуванням**
3. Виберіть роль, яку нові учасники повинні автоматично отримати
4. Нові учасники робочого простору будуть призначені цій ролі, коли вони приєднаються

**Примітка**: Ви можете призначати ролі тільки існуючим учасникам робочого простору. Щоб запросити нових учасників, скористайтеся [керуванням учасниками](/l/uk/user-guide/settings/member-management).

## Налаштування дозволів

Дозволи визначають, що кожна роль може отримувати доступ або змінювати у вашому робочому просторі, включаючи записи об'єктів, налаштування та дії.

### Дозволи на об'єкти та поля

Контролюйте доступ до записів та окремих полів:

#### Дозволи на рівні об'єкта

- Усі об'єкти: застосовувати дозволи такі, як **Перегляд запису**, **Редагування записів**, **Видалення записів** чи **Знищення записів** до всіх об'єктів
- У дозволах на рівні об'єкта налаштуйте виключення для окремих об'єктів. Ці виключення перевершують налаштування з **усіх об'єктів**

#### Дозволи на рівні полів

- Налаштуйте дозволи для окремих полів у кожному об'єкті
- Контролюйте, хто може **Переглядати поле**, **Редагувати поле**, або не має **доступу** до конкретних полів
- Дозволи на поля працюють так само, як дозволи на об'єкти з успадкуванням та виключеннями

#### Керування виключеннями дозволів

Щоб перевершити батьківські дозволи та застосувати суворіші правила:

- Натисніть **X**, щоб видалити успадковане правило
- Виберіть конкретні дозволи для вибраного об'єкта або поля
- Натисніть помаранчеву іконку **Скасувати** (кругова стрілка), щоб повернути зміни

Коли буде завершено, натисніть **Завершити**, потім **Зберегти**, після перенаправлення на сторінку ролі.

### Дозволи на налаштування робочого простору

Контроль доступу до налаштувань робочого простору двома способами:

- Перемкніть **Усі налаштування доступу** для надання повного доступу
- Or enable specific permissions (e.g., API key generation, workspace preferences, role assignment, data model configuration, security settings, and workflow management)

### Дозволи на дії в робочому просторі

Контроль доступу до загальних дій у робочому просторі:

- Перемкніть **Усі дії програми** для надання повних дозволів
- Або увімкніть окремі дії такі, як **Відправити електронний лист**, **Імпорт CSV** та **Експорт CSV**
