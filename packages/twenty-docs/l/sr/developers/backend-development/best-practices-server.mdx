Овај документ излаже најбоље праксе које треба да пратите приликом рада на бекенду.

## Пратите модуларни приступ

Бекенд следи модуларни приступ, који је основни принцип при раду са NestJS. Make sure you break down your code into reusable modules to maintain a clean and organized codebase.
Сваки модул треба да инкапсулира одређену функцију или карактеристику и има јасно дефинисан опсег. This modular approach enables clear separation of concerns and removes unnecessary complexities.

## Expose services to use in modules

Увек креирајте сервисе који имају јасну и једну одговорност, што побољшава читљивост и одрживост кода. Именујте сервисе описно и доследно.

Требало би да изложите и сервисе које желите да користите у другим модулима. Излагање сервиса другим модулима могуће је кроз моћан систем убризгавања зависности NestJS, што промовише лабаву повезаност између компоненти.

## Avoid using `any` type

When you declare a variable as `any`, TypeScript's type checker doesn't perform any type checking, making it possible to assign any type of values to the variable. TypeScript користи инференцију типа да одреди тип променљиве на основу вредности. By declaring it as `any`, TypeScript can no longer infer the type. Ово отежава откривање грешака повезаних са типовима током развоја, што доводи до грешака у току рада и чини код мање одрживим, мање поузданим и тежим за разумевање за друге.

Због тога све треба да има тип. Дакле, ако креирате нови објекат са именом и презименом, требало би да креирате интерфејс или тип који садржи име и презиме и дефинише облик објекта којим манипулишете.
