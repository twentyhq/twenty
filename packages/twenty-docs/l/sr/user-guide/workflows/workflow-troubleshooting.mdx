## Отклањање грешака помоћу извршавања радних процеса

Користите интерфејс за извршавање радних процеса да бисте отклонили грешке:

- Приступите путем панела "Извршења" у уреднику радних процеса
- Кликните на појединачна извршења да бисте видели улазне/излазне податке за сваки корак
- Проверите статус извршења, поруке о грешци и ток података између корака

## Уобичајени проблеми и решења

### Радни процес се не покреће

**Проблем**: Радни процес се не извршава када се очекује.

**Решења**:

- Проверите да ли је радни процес **активан** (није у режиму нацрта)
- Проверите да ли се конфигурација покретача поклапа са вашом структуром података
- За покретаче записа, уверите се да су исправно одабрани објекат и поља
- За вебхук покретаче, проверите URL и очекивани формат података
- За заказане покретаче, проверите конфигурацију времена

### Обрасци нису доступни

**Проблем**: Радње са обрасцима су тешко доступне или компликоване за пронаћи у аутоматизованим радним процесима.

<Warning>
Обрасци су тренутно дизајнирани само за ручне покретаче. За радне процесе са другим покретачима (Забележено креирање, ажурирање итд.), обрасци су доступни само путем интерфејса за извршавање радног процеса, што није очекивано корисничко искуство. Центар за обавештења ће бити објављен 2026. године како би се исправно подржали обрасци у аутоматизованим радним процесима.
</Warning>

**Решење**: Користите ручне покретаче када су обрасци неопходни или реструктурирајте радне процесе да избегнете обрасце у аутоматизованим токовима.

### Висока потрошња кредита

**Проблем**: Радни процеси троше више кредита него што се очекивало.

**Уобичајени узроци и решења**:

- **Неефикасни API позиви**: Комбинујте API позиве када је могуће уместо појединачних захтева
- **Погрешна конфигурација ручних покретача**: Користите `Масовна` доступност уместо `Појединачна` да обрадите више записа у једном извршавању радног процеса
- **Недостају филтери**: Додајте условну логику да зауставите радне процесе када критеријуми нису испуњени
- **Непотребни кораци**: Уклоните сувишне радње и оптимизујте логику радног процеса
- **Реално време наспрам заказаног**: Користите заказане радне процесе за нетренутне процесе

### Паралелно ограничење броја радних процеса

**Проблем**: Достизање ограничења од 100 паралелних радних процеса по радном простору.

<Warning>
You cannot run more than 100 workflows in parallel at any given time per workspace.
</Warning>

**Решења**:

- Користите `Масовну` доступност за ручне покретаче да обрадите више записа у једном извршавању
- Примените закашњења између извршавања радног процеса користећи заказане покретаче
- Оптимизујте радне процесе да буду бржи и смањите време паралелног извршења
- Размотрите обраду у пакетима током периода смањеног оптерећења

### Ограничење брзине API позива

**Проблем**: Спољни API позиви не успевају због ограничења брзине.

**Решења**:

- Користите заказане радне процесе уместо покретача у реалном времену када је могуће
- Примените закашњења између API позива у активностима кода
- Пакетујте API захтеве када то спољна услуга подржава
- Надгледајте извршавање радних процеса због грешака у ограничењу брзине и прилагодите тајминг

### Проблеми са итерирањем

**Проблем**: Итеративне акције не функционишу како се очекује.

**Решења**:

- **Напомена**: Итератор је тренутно у бета фази. Активирајте га под Подешавања → Издања → Лаб
- Проверите да ли је улаз низ записа из претходног корака
- Проверите да ли су акције унутар Итератора правилно конфигурисане
- Користите Итератор са `Масовним` ручним покретачима за оптималне перформансе

### Missing Permissions

**Проблем**: Не могу приступити одељку радних процеса.

<Warning>
Ако не видите одељак радни процеси, то је због проблема са дозволама. Контактирајте администратора свог радног простора да вам да приступ радним процесима.
</Warning>

## Савети за оптимизацију

### Најбоље праксе за перформансе

- **Почните једноставно** и постепено додајте комплексност
- **Користите `Масовну` доступност** за ручне покретаче да ефикасно обрадите више записа
- **Додајте филтере рано** да зауставите радне процесе када критеријуми нису испуњени
- **Пакујте API позиве** и користите заказане радне процесе за нетренутне процесе
- **Сакријте извршавања радних процеса** са страница које приказују све радне процесе како бисте побољшали перформансу учитавања
- **Редовно надгледајте потрошњу кредита** у Подешавања → Наплата

### Превенција грешака

- **Тестирајте у нацрт режиму** пре активирања радних процеса
- **Проверите API одговоре** и примените резервне акције
- **Користите описна имена корака** ради лакшег одржавања
- **Документирајте комплексну логику** за чланове тима
