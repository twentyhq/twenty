## Шта су поља за однос?

Relation fields link records from one object to records in another object. На пример:

- **Људи** → **Компаније** (свака особа ради за компанију)
- **Прилике** → **Људи** (сваки договор има контакт особу)
- **Задаци** → **Прилике** (сваки задатак се односи на одређени договор)

## Creating Relation Fields

### 1. Додајте поље

Go to **Settings → Data Model → [Your Object]** and click **Add Field**.

### 2. Choose Relation Type

- **Врста поља**: Изаберите "Однос"
- **Циљни Објекат**: Изаберите објекат за повезивање
- **Relationship**: Currently supports **1-to-many** relationships only. Обавезно креирајте однос у правом смеру.

### 3. Configure Field Names

Требаће вам да подесите имена за обе стране односа:

- **Име изворног поља**: Како се поље приказује на вашем тренутном објекту
- **Име циљног поља**: Како се обрнуто поље приказује на циљном објекту

<Warning>
Имена поља не могу бити измењена када се однос сачува јер утичу на API структуру. Изаберите пажљиво.
</Warning>

## Повезивање са члановима тима

Можете креирати односе са било којим објектом, укључујући **члнове радног простора** (ваши корисници Twenty тима). Ово је корисно за креирање поља за власништво:

- **Власник налога**: Повежите компанију са чланом тима који је управља
- **Власник договора**: Доделите одређену прилику одређеном продавачу

Када креирате однос са `члнови радног простора`, видећете имена ваших чланова тима у изборима падајућег менија, олакшавајући доделу власништва и одговорности.

## Најбоље праксе

- **Планирајте своје односе** пре него што их направите
- **Користите јасна, описна имена** за обе стране поља
- **Тестирајте односе** са узорцима података пре потпуне имплементације
