## Управљање објектима

<AccordionGroup>
<Accordion title="Can I reorder objects in the left navigation bar?">
Још увек не. Поређање објеката у навигацији је тренутно фиксно, али је ова функција планирана за будуће издање.
</Accordion>

<Accordion title="Can I hide objects from the left navigation bar?">
Сви активни објекти се појављују у навигацији. Можете деактивирати објекте који вам нису потребни под **Подешавања → Модел података**.
</Accordion>

<Accordion title="Can I delete standard objects (People, Companies, etc.)?">
You can deactivate any standard objects but you cannot hard delete them.
</Accordion>
</AccordionGroup>

## Field Capabilities

<AccordionGroup>
<Accordion title="Can I create formula fields?">
Formula fields are coming in **Q1 2026**. У међувремену, можете користити радне токове за аутоматско израчунавање и ажурирање вредности поља.
</Accordion>

<Accordion title="Can I have nested fields in my objects?">
Угнежђена поља долазе у **К1 2026**. Тренутно, можете користити радне токове да донесете вредности поља из сродних објеката. For example, to display a company's industry on a Person record, create a custom field on People and use a workflow to synchronize the value.
</Accordion>

<Accordion title="Why can't I update relation field names?">
Називи релационих поља утичу на API структуру и не могу се мењати након креирања. Ако вам је потребно да преименујете релационо поље, мораћете да направите ново и избришите старо.
</Accordion>

<Accordion title="Why do I need different singular and plural names?">
Our GraphQL API uses both forms for different operations:
- `createPerson` (singular) for single record actions
- `createPeople` (plural) for bulk operations

Ово ствара ограничења када су једнина и множина исте, али побољшава искуство програмера. </Accordion>

<Accordion title="Why are some field names protected?">
Certain field names like `Type` or `Application` are reserved for system use. Изаберите алтернативна имена као што су `Категорија` или `Класификација`.
</Accordion>
</AccordionGroup>

## Напредне карактеристике

<AccordionGroup>
<Accordion title="Can I create many-to-many relationships?">
Односи много према многима долазе у **К1 2026**. Тренутно, направите посредни објекат са два односа један према многима као алтернативу.
</Accordion>

<Accordion title="What are Morph Many relationships?">
Morph Many односи омогућују да се један објекат односи према различитим типовима објеката. На пример, могућност би могла бити у релацији са особом или компанијом. Ова функција је сада доступна.
</Accordion>

<Accordion title="Can I reorder fields in objects?">
Редослед поља ће бити могућ са прилагођеним изгледима у **К4 2025**. Тренутно, поља се приказују по абецедном реду.
</Accordion>
</AccordionGroup>

## Приступ и дозволе

<AccordionGroup>
<Accordion title="Where can I see and edit my data model?">
You can access your Data Model under **Settings → Data Model**.
</Accordion>

<Accordion title="Why can't I see the Data Model under Settings?">
Контактирајте свог администратора радног простора. Приступ моделу података је обично ограничен само на администраторе.
</Accordion>

<Accordion title="How many custom objects or fields can I create?">
You can create as many custom objects and fields as you need - the price won't change.
</Accordion>
</AccordionGroup>
