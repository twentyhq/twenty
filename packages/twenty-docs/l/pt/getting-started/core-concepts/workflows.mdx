Os fluxos de trabalho permitem automatizar tarefas repetitivas e conectar o Twenty a ferramentas externas — sem escrever código (embora você possa, se quiser).

<VimeoEmbed videoId="1185511711" title="Workflow" />

## Como os fluxos de trabalho funcionam

Cada fluxo de trabalho tem três partes:

1. **Disparador** — O que inicia o fluxo de trabalho
2. **Etapas** — O que acontece em seguida (uma ou mais ações em sequência)
3. **Variáveis** — Dados que fluem entre as etapas

## Disparadores

| Disparador             | Quando é acionado                                                                       |
| ---------------------- | --------------------------------------------------------------------------------------- |
| **Evento de registro** | Um registro é criado, atualizado, excluído ou inserido/atualizado (upsert)              |
| **Manual**             | Um usuário clica em um botão (em um único registro, em vários registros ou globalmente) |
| **Agendamento**        | Em um intervalo recorrente (sintaxe cron)                                               |
| **Webhook**            | Um sistema externo envia um HTTP POST                                                   |

## Ações

Os fluxos de trabalho podem encadear qualquer combinação de:

* **Operações de registro** — Criar, atualizar, localizar, excluir ou inserir/atualizar registros (upsert)
* **Enviar e-mail** — Enviar ou rascunhar e-mails a partir de contas conectadas
* **Solicitação HTTP** — Chamar qualquer API externa
* **Código** — Executar JavaScript personalizado para lógica complexa
* **Ramificações** — Condições if/else para dividir o caminho do fluxo de trabalho
* **Iterador** — Percorrer arrays de dados
* **Agente de IA** — Deixar um agente de IA processar dados de forma autônoma
* **Atraso** — Aguardar antes de continuar
* **Formulário** — Coletar entrada do usuário no meio do fluxo de trabalho

## O que você pode criar

* Enviar alertas no Slack quando um negócio atingir um determinado estágio
* Enriquecer automaticamente novos contatos com dados de APIs externas
* Detectar oportunidades estagnadas e notificar o responsável
* Sincronizar dados entre o Twenty e o seu sistema de faturamento
* Gerar PDFs ou faturas a partir de dados de registros
* Responder automaticamente a e-mails recebidos que correspondam a certos critérios
