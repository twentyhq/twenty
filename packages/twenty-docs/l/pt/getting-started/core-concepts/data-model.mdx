Tudo no Twenty é construído em torno de **objetos** e **campos** — os blocos de construção do seu modelo de dados.

## Objetos

Objetos são as tabelas que armazenam seus dados. O Twenty vem com objetos padrão prontos para uso:

* **Companies** — Organizações com as quais você faz negócios
* **People** — Contatos individuais
* **Opportunities** — Negócios no seu funil
* **Tasks** — Itens de ação para sua equipe
* **Notes** — Texto livre vinculado a registros

Você também pode criar **objetos personalizados** para tudo o que a sua empresa precisar — projetos, tíquetes de suporte, produtos, contratos ou qualquer outra coisa.

## Campos

Campos são as propriedades de cada objeto. O Twenty é compatível com uma ampla variedade de tipos de campo:

| Categoria    | Tipos                                                                         |
| ------------ | ----------------------------------------------------------------------------- |
| **Básico**   | Texto, Número, Booleano, Data, Moeda, Avaliação, Seleção                      |
| **Composto** | Endereço (rua, cidade, estado, CEP), Nome completo, Links, Telefones, E-mails |
| **Especial** | Relação, Anexo de arquivo, JSON, Ator (quem criou/modificou)                  |

Cada objeto também recebe campos de sistema automáticos: `id`, `createdAt`, `updatedAt`, `createdBy` e `position`.

## Relações

Os objetos se conectam entre si por meio de relações. Uma Company tem muitas People, uma Opportunity pertence a uma Company e assim por diante. Você pode criar relações personalizadas entre quaisquer objetos, incluindo relacionamentos muitos-para-muitos.

## O que torna isso poderoso

Ao contrário dos CRMs tradicionais, em que você fica limitado a campos predefinidos em objetos predefinidos, o Twenty permite modelar seus dados exatamente da forma como o seu negócio funciona. Objetos personalizados recebem o mesmo tratamento de primeira classe que os nativos — incluindo endpoints de API, visualizações, permissões e gatilhos de fluxo de trabalho.
