## O que são Campos de Relação?

Campos de relação conectam registros de um objeto a registros em outro objeto. Por exemplo:

- **Pessoas** → **Empresas** (cada pessoa trabalha para uma empresa)
- **Oportunidades** → **Pessoas** (cada negócio tem uma pessoa de contato)
- **Tarefas** → **Oportunidades** (cada tarefa está relacionada a um negócio específico)

## Criando Campos de Relação

### 1. Adicionar o Campo

Vá para **Configurações → Modelo de Dados → [Seu Objeto]** e clique em **Adicionar Campo**.

### 2. Escolha o Tipo de Relação

- **Tipo de Campo**: Selecione "Relação"
- **Objeto Alvo**: Escolha qual objeto conectar
- **Relacionamento**: Atualmente suporta apenas relações **1-para-muitos**. Certifique-se de criar a relação na direção certa.

### 3. Configurar Nomes dos Campos

Você precisará definir nomes para ambos os lados da relação:

- **Nome do campo de origem**: Como o campo aparece no seu objeto atual
- **Nome do campo alvo**: Como o campo reverso aparece no objeto alvo

<Warning>
Os nomes dos campos não podem ser editados uma vez que a relação é salva, pois isso impacta a estrutura da API. Escolha com cuidado.
</Warning>

## Relating to Team Members

You can create relations to any object, including **Workspace Members** (your Twenty team users). This is useful for creating ownership fields:

- **Proprietário da Conta**: Vincule uma Empresa a um membro da equipe que a gerencia
- **Proprietário do Negócio**: Atribua uma Oportunidade a um vendedor específico

When you create a relation to`Workspace Members`, you'll see your team members' names in dropdown selections, making it easy to assign ownership and responsibilities.

## Melhores Práticas

- **Planeje seus relacionamentos** antes de criá-los
- **Use nomes claros e descritivos** para ambos os nomes de campos
- **Teste os relacionamentos** com dados de amostra antes da implementação completa
