## Objetos Padrão

Objetos padrão são entidades predefinidas no seu espaço de trabalho para ajudá-lo a começar. Eles fazem parte de um modelo de dados compartilhado acessível a todos os usuários do Twenty. Você pode usá-los como estão, personalizá-los ou desativá-los.

<img src="/images/user-guide/objects/standard-objects.png"style={{width:'100%'}}/>

### Pessoas

O objeto `Pessoas` armazena seus contatos. Inclui detalhes de contato e histórico de interações, para que você possa ver todas as suas interações com clientes em um só lugar.

### Empresa

O objeto `Empresas` armazena suas contas de negócios. Inclui detalhes como setor, tamanho e localização. Empresas se conectam aos objetos `Pessoas` e `Oportunidades`.

### Oportunidades

O objeto `Oportunidades` armazena dados relacionados a negócios. Acompanha a progressão de vendas potenciais, desde a prospecção até o fechamento, registrando etapas, tamanhos de negócio, conta associada e data esperada de fechamento. Você pode visualizar seu pipeline de vendas em um layout kanban.

## Objetos personalizados

Custom objects let you store information that's unique to your organization and that standard objects can't handle. Por exemplo, se você é a SpaceX, pode querer criar um objeto personalizado para Foguetes e Lançamentos.

<img src="/images/user-guide/objects/custom-objects.png"style={{width:'100%'}}/>

### Criando um novo objeto personalizado

Para criar um novo objeto personalizado:

1. Vá para Configurações na barra lateral à esquerda.
2. Em Espaço de trabalho, vá para Modelo de dados. Aqui você poderá ver uma visão geral de todos os seus objetos Padrão e Personalizados existentes (ativos e desativados).

<VimeoEmbed videoId="926288174" title="Video demonstration" />

3. Clique em `+ Novo objeto` no topo. Digite o nome (singular e plural), escolha um ícone e adicione uma descrição para seu objeto personalizado e clique em Salvar (no canto superior direito). Usando Listagem como exemplo de objeto personalizado, o singular seria "listagem" e o plural seria "listagens" junto com uma descrição como "Listagens que os anfitriões criaram para exibir sua propriedade."

<Warning>
Os nomes no singular e no plural precisam ser diferentes. Isto é necessário para que nossa API GraphQL funcione corretamente.
</Warning>

<VimeoEmbed videoId="926293493" title="Video demonstration" />

4. Assim que criar seu objeto personalizado, poderá gerenciá-lo. Você pode editar o nome, ícone e descrição, visualizar os diferentes campos e adicionar mais campos.

<img src="/images/user-guide/objects/customize-fields.png"style={{width:'100%'}}/>

**Note:** Se você não tem certeza se um novo objeto ou campo é necessário, consulte [este artigo](/l/pt/user-guide/data-model/customize-your-data-model) para orientação sobre como projetar seu modelo de dados.
