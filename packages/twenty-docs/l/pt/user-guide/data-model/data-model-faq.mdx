## Gerenciamento de Objetos

<AccordionGroup>
<Accordion title="Can I reorder objects in the left navigation bar?">
Ainda não. A ordenação de objetos na navegação atualmente é fixa, mas essa funcionalidade está prevista para um lançamento futuro.
</Accordion>

<Accordion title="Can I hide objects from the left navigation bar?">
Todos os objetos ativos aparecem na navegação. Você pode desativar os objetos que não precisa em **Configurações → Modelo de Dados**.
</Accordion>

<Accordion title="Can I delete standard objects (People, Companies, etc.)?">
You can deactivate any standard objects but you cannot hard delete them.
</Accordion>
</AccordionGroup>

## Field Capabilities

<AccordionGroup>
<Accordion title="Can I create formula fields?">
Os campos de fórmula estão previstos para o **1º trimestre de 2026**. Enquanto isso, você pode usar fluxos de trabalho para calcular e atualizar valores de campo automaticamente.
</Accordion>

<Accordion title="Can I have nested fields in my objects?">
Os campos aninhados estão previstos para o **1º trimestre de 2026**. Atualmente, você pode usar fluxos de trabalho para trazer valores de campo de objetos relacionados. Por exemplo, para exibir o setor de uma empresa em um registro de Pessoa, crie um campo personalizado em Pessoas e use um fluxo de trabalho para sincronizar o valor.
</Accordion>

<Accordion title="Why can't I update relation field names?">
Os nomes de campos de relação afetam a estrutura da API e não podem ser alterados após a criação. Se você precisar renomear um campo de relação, precisará criar um novo e excluir o antigo.
</Accordion>

<Accordion title="Why do I need different singular and plural names?">
Our GraphQL API uses both forms for different operations:
- `createPerson` (singular) for single record actions
- `createPeople` (plural) for bulk operations

Isso cria limitações quando as formas singular e plural são as mesmas, mas melhora a experiência do desenvolvedor. </Accordion>

<Accordion title="Why are some field names protected?">
Certos nomes de campo como `Type` ou `Application` são reservados para uso do sistema. Escolha nomes alternativos como `Categoria` ou `Classificação`.
</Accordion>
</AccordionGroup>

## Recursos Avançados

<AccordionGroup>
<Accordion title="Can I create many-to-many relationships?">
As relações muitos-para-muitos estão previstas para o **1º trimestre de 2026**. Atualmente, crie um objeto intermediário com duas relações 1-para-muitos como solução alternativa.
</Accordion>

<Accordion title="What are Morph Many relationships?">
As relações Morph Many permitem que um objeto se relacione com vários tipos de objetos diferentes. Por exemplo, uma Oportunidade pode se relacionar tanto a uma Pessoa quanto a uma Empresa. Este recurso já está disponível.
</Accordion>

<Accordion title="Can I reorder fields in objects?">
A reorganização de campos estará disponível com layouts personalizados no **4º trimestre de 2025**. Atualmente, os campos são exibidos em ordem alfabética.
</Accordion>
</AccordionGroup>

## Acesso e Permissões

<AccordionGroup>
<Accordion title="Where can I see and edit my data model?">
You can access your Data Model under **Settings → Data Model**.
</Accordion>

<Accordion title="Why can't I see the Data Model under Settings?">
Entre em contato com o administrador do seu espaço de trabalho. O acesso ao modelo de dados geralmente é restrito apenas a administradores.
</Accordion>

<Accordion title="How many custom objects or fields can I create?">
You can create as many custom objects and fields as you need - the price won't change.
</Accordion>
</AccordionGroup>
