## Visão Geral

Os créditos alimentam suas automações de fluxo de trabalho no Twenty. Cada ação de fluxo de trabalho consome créditos com base em sua complexidade.

## Alocação de créditos

Os créditos são baseados no seu ciclo de faturamento, não no seu plano:

| Ciclo de faturamento | Créditos       |
| -------------------- | -------------- |
| Mensal               | 5 milhões/mês  |
| Anual                | 50 milhões/ano |

<Note>
  Os 5 milhões de créditos mensais foram projetados para permitir que você execute automações sem se preocupar com custos. Para a maioria dos fluxos de trabalho que usam ações padrão, isso é mais do que suficiente. Você só precisará de créditos adicionais ao executar nós de código avançados ou recursos com tecnologia de IA.
</Note>

## Consumo de créditos

Ações diferentes consomem quantidades diferentes de créditos:

| Tipo de ação                                                      | Uso de Créditos          |
| ----------------------------------------------------------------- | ------------------------ |
| **Operações básicas** (pesquisar, atualizar, criar registros)     | Mínimo                   |
| **Operações complexas** (nós de código, chamadas de API externas) | Mais créditos            |
| **Prompts de IA** (em breve)                                      | Variável com base no uso |

Os créditos são deduzidos em tempo real à medida que os fluxos de trabalho são executados.

## Monitoramento do uso

Acompanhe seu consumo de créditos:

1. Vá para **Configurações → Faturamento**
2. Veja seu uso atual e os créditos restantes
3. Monitore tendências para planejar créditos adicionais, se necessário

## Compra de Créditos Adicionais

Precisa de mais créditos?

1. Vá para **Configurações → Faturamento**
2. Clique na opção para comprar pacotes de créditos adicionais
3. Selecione a quantidade de que você precisa
