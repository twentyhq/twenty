## Relatar Erros

Para relatar um erro, por favor [crie um problema no GitHub](https://github.com/twentyhq/twenty/issues/new).

Você também pode pedir ajuda no [Discord](https://discord.gg/cx5n4Jzs57).

## Solicitações de Funcionalidades

Se você não tem certeza se é um erro e sente que está mais próximo de uma solicitação de funcionalidade, então você provavelmente deve [abrir uma discussão em vez disso](https://github.com/twentyhq/twenty/discussions/new).

## Envie um Pull Request

Contribuir com código para o Twenty começa com um pull request (PR).

### Antes de Começar

1. Confira as [issues existentes](https://github.com/twentyhq/twenty/issues) para trabalhos relacionados
2. Para novas funcionalidades, abra primeiro uma issue para discutir
3. Consulte nosso [Código de Conduta](https://github.com/twentyhq/twenty/blob/main/CODE_OF_CONDUCT.md)

### Fork e clone

1. Crie um fork do repositório no GitHub
2. Clone seu fork:

```bash
git clone https://github.com/YOUR_USERNAME/twenty.git
cd twenty
```

3. Adicione o remoto upstream:

```bash
git remote add upstream https://github.com/twentyhq/twenty.git
```

### Crie uma branch

```bash
git checkout -b feature/your-feature-name
```

Use nomes de branch descritivos:

* `feature/add-export-button`
* `fix/login-redirect-issue`
* `docs/update-api-guide`

### Faça suas alterações

1. Escreva código limpo e bem documentado
2. Siga o estilo de código existente
3. Adicione testes para a nova funcionalidade
4. Atualize a documentação se necessário

### Envie seu PR

1. Faça push da sua branch:

```bash
git push origin feature/your-feature-name
```

2. Abra um PR no GitHub
3. Preencha o modelo do PR
4. Vincule as issues relacionadas

### Checklist de PR

* [ ] O código segue as diretrizes de estilo do projeto
* [ ] Os testes passam localmente
* [ ] A documentação está atualizada
* [ ] A descrição do PR explica as alterações
